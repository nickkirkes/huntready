# Implementation Plan: S02.2 — Hunting District Ingestion

## Context

E02 ingests Montana big-game geometries from MT FWP's ArcGIS MapServer. S02.0 added the `gis_layer` document type and `geometry.verbatim_rule` column; S02.1 built the shared `arcgis` library. S02.2 is the first state-adapter story: it consumes those primitives to load Hunting District geometries from layers #3 (Antelope), #10 (Black Bear), and #11 (Deer/Elk/Lion) into the `geometry` table with deterministic species-prefixed IDs, MultiPolygon WKT, ArcGIS source citations, and verbatim `REG` text where present. This is also the first DB-write code in the project — there is no pre-existing INSERT/UPSERT pattern to follow.

S02.2 introduces a shared `ingestion.lib.db` UPSERT helper (will be reused by S02.3, S02.4, S02.5) and the first per-state adapter under `ingestion/states/montana/`.

## Tasks

### T1: Shared `ingestion.lib.db` UPSERT helper (~5 min)
**Files:** ingestion/ingestion/lib/db.py
**Action:** Create the first DB-write module. Provide a connection helper (reads `DATABASE_URL` from env via psycopg3) and `upsert_geometry(conn, geom: Geometry) -> None` plus `upsert_geometries(conn, geoms: Iterable[Geometry]) -> int` (returns rows affected). Both functions accept a caller-provided connection so a single transaction can wrap all three layers. UPSERT shape per epic line 256:

```sql
INSERT INTO geometry (id, name, kind, geom, state, license_year, source, verbatim_rule)
VALUES (%s, %s, %s, ST_GeomFromText(%s, 4326)::geography, %s, %s, %s, %s)
ON CONFLICT (id) DO UPDATE SET
    geom = EXCLUDED.geom,
    name = EXCLUDED.name,
    source = EXCLUDED.source,
    license_year = EXCLUDED.license_year,
    verbatim_rule = EXCLUDED.verbatim_rule
```

**Details:**
- Use `psycopg.connect(os.environ["DATABASE_URL"])`. Raise a clear error if env var is missing.
- Serialize `source` via `Json(geom.source.model_dump(exclude_none=True))` (psycopg3's `psycopg.types.json.Json` adapter).
- `geom.geom` (the WKT string from the Pydantic field) is passed directly to `ST_GeomFromText(%s, 4326)::geography`.
- `upsert_geometries` iterates the input and delegates to `upsert_geometry` per row. Does NOT commit — the caller controls the transaction (so all three Montana layers commit atomically in `main()`).
- Module-level docstring documents that this is the canonical write path for the `geometry` table; future stories should extend rather than parallel-implement.
- `from __future__ import annotations`; type-hint everything; no `Any`.

**Verify:** `cd ingestion && .venv/bin/ruff check ingestion/lib/db.py && .venv/bin/mypy ingestion/lib/db.py`
**UI:** no

### T3: Montana HD load script (~5 min)
**Files:** ingestion/states/montana/load_hds.py
**Depends on:** T1
**Action:** Implement the Montana adapter for layers #3, #10, #11. Structure:

```python
from dataclasses import dataclass
from ingestion.lib import arcgis, db
from ingestion.lib.schema import Geometry

MT_FWP_HUNTING_DISTRICTS_URL = "https://fwp-gis.mt.gov/arcgis/rest/services/admbnd/huntingDistricts/MapServer"
MT_FIXTURE_DIR = Path(__file__).parent / "fixtures"  # CWD-independent
MT_LAYER_SLUG = "huntingDistricts"

@dataclass(frozen=True)
class HDLayerConfig:
    layer_id: int
    species_slug: str   # "antelope", "bear", "deer-elk-lion"
    id_prefix: str      # "MT-HD-antelope", etc.

HD_LAYERS = (
    HDLayerConfig(3,  "antelope",       "MT-HD-antelope"),
    HDLayerConfig(10, "bear",           "MT-HD-bear"),
    HDLayerConfig(11, "deer-elk-lion",  "MT-HD-deer-elk-lion"),
)
```

Pure helpers (each easily unit-testable):
- `_extract_district(props: dict, metadata: LayerMetadata) -> str`: looks for `DISTRICT` in `props`; if absent, raises `ArcGISError(f"layer {metadata.name} feature missing DISTRICT field; available={metadata.out_fields}")`. Future: if #3/#10 reveal a different field name, extend here. Fail-loud per ADR-008.
- `_extract_verbatim_rule(props: dict) -> str | None`: reads `REG` field. If missing, `None`, or whitespace-only after `.strip()` → return `None`. Per known-pitfall: never write `""` to `verbatim_rule`.
- `_extract_license_year(props: dict) -> int | None`: reads `REGYEAR` if present; coerces to `int`. If missing or `None` → return `None`.
- `_feature_to_geometry(feature, layer_config, layer_metadata, service_url, fetch_year) -> Geometry`: builds a `Geometry` instance with `id=f"{layer_config.id_prefix}-{district}-geom"`, `name=` per-layer naming convention (use `f"{species_slug.replace('-', ' ').title()} HD {district}"` or similar — pick one and document), `kind="hunting_district"`, `geom=geojson_to_multipolygon_wkt(feature)`, `state="US-MT"`, `license_year=_extract_license_year(...)`, `verbatim_rule=_extract_verbatim_rule(...)`, and a per-feature `source=` constructed via `arcgis.build_source_citation(license_year=REGYEAR or fetch_year, ...)` per the epic example at line 175-188.
- `_load_layer(conn, service_url, layer_config, fixture_dir, fetch_year, *, session=None) -> list[Geometry]`: orchestrates `fetch_layer_metadata` → `fetch_features` → loop building `Geometry` instances → `db.upsert_geometries(conn, geoms)`. Returns the list for caller-side verification.
- `main()`: parses `--service-url` and `--fetch-year` (default: today's UTC year), opens a single psycopg3 connection, iterates `HD_LAYERS`, logs counts per layer, commits.

**Details:**
- `SourceCitation.license_year` is **per-feature**: `REGYEAR` if the feature provides it, else `fetch_year`. Per the epic at line 175-188, the citation answers "which annual cycle does this evidence belong to?" — a question the source data should answer when it can. Per-row `Geometry.license_year` stays REGYEAR-or-NULL (NULL is meaningful for year-invariant geometries). When REGYEAR is present, both fields end up as the same int; when REGYEAR is absent, citation falls back to fetch_year while the row stays NULL.
- Build a `requests.Session` once and reuse for all three layers (the `arcgis` lib accepts one).
- Use the `MT_FIXTURE_DIR` constant for fixture writes.
- `if __name__ == "__main__":` calls `main()`.
- Use `logging.getLogger(__name__)`; log per-layer feature count.
- Failure policy: any feature that fails `geojson_to_multipolygon_wkt` (GeometryCollection, empty, etc.) propagates the `ArcGISError` — fail loud per ADR-008.

**Verify:** `cd ingestion && .venv/bin/ruff check states/montana/load_hds.py && .venv/bin/mypy states/montana/load_hds.py`

## Blast Radius

- **Do NOT modify:** `ingestion/ingestion/lib/arcgis.py` (S02.1, just merged), `ingestion/ingestion/lib/schema.py` (S02.0, just merged), `supabase/migrations/*` (no migration needed), `mcp-server/`, `web/`, `plugin/`.
- **Watch for:** psycopg3 connection leaks (always use context manager); Pydantic `extra="forbid"` on `Geometry` rejects typos like `verbatim_text` instead of `verbatim_rule` — let it fail loudly; pyproj/geopandas are unrelated to this story.
- **First-of-kind concerns:** This is the first DB-write code in the project. Pattern choices (psycopg3 connect signature, JSON adapter usage, WKT cast syntax) become precedent for S02.3-S02.7 — pick conservatively and document any non-obvious choice in T1's docstring.

## Conventions

- Three-place type-sync was completed in S02.0; no schema changes here.
- ADR-001/008: source citations always present, verbatim text never paraphrased — `_extract_verbatim_rule` enforces this by writing `None` (not `""`) when source has nothing.
- ADR-005: shared lib lives at `ingestion/ingestion/lib/`; state-specific code under `ingestion/states/<state>/`. The `db.py` module is shared (will be reused by S02.3-S02.5).
- ADR-014: `SourceCitation.publication_date = f"{license_year:04d}-01-01"` — already enforced by `arcgis.build_source_citation`. S02.2 just passes the right `license_year` int.
- Known pitfall (`verbatim_rule` accepts `""`): guarded in `_extract_verbatim_rule`. Tested in T4.
- No comments explaining what code does. Comments only for non-obvious WHY (e.g., the "citation `license_year` ≠ row `license_year`" subtlety in `_load_layer` may warrant one short line).
