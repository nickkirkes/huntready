# Implementation Plan: S02.3 Montana Portions Ingestion

**Story spec:** `docs/planning/epics/E02-geometry-ingestion.md` lines 287-326.

**Summary:** Build a Python adapter that ingests four ArcGIS portion layers from MT FWP (#4 Antelope, #12 Mule Deer, #13 Whitetail, #14 Elk Portions) into the `geometry` table with `kind='portion'`. Mirrors the just-merged S02.2 hunting-district adapter (`load_hds.py`). All shared infrastructure (`lib/arcgis.py`, `lib/db.py`, `lib/schema.py`) is reusable as-is — `"portion"` is already a valid `Geometry.kind` Literal value.

**Net-new behavior vs S02.2:**
1. ID format `MT-HD-{species}-{DISTRICT}-portion-{slug}-geom` requires a slug derived from `SHAPECODE` (preferred) or slugified `PORTIONNAME` (fallback).
2. Per-layer collision detection on the assembled IDs (fail loudly if `len(ids) != len(set(ids))`).
3. New inline `_slugify` helper (no slugify dep in `pyproject.toml`).

### T2: Extraction helpers — district, verbatim_rule, license_year, slugify, portion_slug (~4 min)

**Files:** `ingestion/states/montana/load_portions.py`

**Depends on:** T1

**Action:** Add five module-level helper functions. Three are copy-equivalents of `load_hds.py` helpers (same field semantics for portion layers per spec line 306). Two are new.

**Details:**

- `_extract_district(props: dict[str, Any], metadata: LayerMetadata) -> str` — identical to `load_hds.py` lines 61-69 (returns `str(value)` for non-None `DISTRICT`; raises `ArcGISError` with `available=` listing if absent).
- `_extract_verbatim_rule(props: dict[str, Any]) -> str | None` — identical to `load_hds.py` lines 72-79 (returns `None` for missing/whitespace-only `REG`; preserves original non-empty string without stripping).
- `_extract_license_year(props: dict[str, Any]) -> int | None` — identical to `load_hds.py` lines 82-90 (coerces `REGYEAR` to int; raises `ArcGISError` on `ValueError`/`TypeError`).
- `_slugify(text: str) -> str` — pure helper. Implementation: lowercase, replace any run of non-alphanumeric characters with a single hyphen, strip leading/trailing hyphens. Use `re.sub(r"[^a-z0-9]+", "-", text.lower()).strip("-")`. Empty result raises `ArcGISError("cannot slugify empty/whitespace text: {text!r}")`.
- `_extract_portion_slug(props: dict[str, Any], metadata: LayerMetadata) -> str` — try `SHAPECODE` first; if non-empty, return `str(SHAPECODE)` verbatim (no slugify — SHAPECODE is already a code). If `SHAPECODE` is missing/empty/whitespace, try `PORTIONNAME`; return `_slugify(str(PORTIONNAME))`. If neither yields a non-empty value, raise `ArcGISError("layer {metadata.name!r} feature missing both SHAPECODE and PORTIONNAME; available={list(metadata.out_fields)}")`.

### T3: Feature normalization + load_layer with collision detection (~4 min)

**Files:** `ingestion/states/montana/load_portions.py`

**Depends on:** T2

**Action:** Add `_feature_to_geometry` (mirroring `load_hds.py` with the new ID format) and `_load_layer` (with a post-normalization collision check before calling `db.upsert_geometries`).

**Details:**

- `_feature_to_geometry(feature, layer_config, layer_metadata, service_url, fetch_year) -> Geometry` — mirrors `load_hds.py` lines 93-126 with these specific changes:
  - Compute `district = _extract_district(props, layer_metadata)` and `slug = _extract_portion_slug(props, layer_metadata)`.
  - `id = f"{layer_config.id_prefix}-{district}-portion-{slug}-geom"` (note: matches spec line 308 exactly).
  - `name = f"{layer_config.species_display} HD {district} portion {slug}"`.
  - `kind = "portion"` (NOT `"hunting_district"`).
  - All other behavior identical: `geojson_to_multipolygon_wkt`, per-feature `build_source_citation` with `license_year=regyear if regyear is not None else fetch_year`, `verbatim_rule=_extract_verbatim_rule(props)`, `state=MT_STATE_CODE`, `license_year=regyear` (geometry-level value preserves NULL).

- `_load_layer(conn, service_url, layer_config, fixture_dir, fetch_year, *, session=None) -> list[Geometry]` — mirrors `load_hds.py` lines 129-162 with one addition: AFTER building `geoms` and BEFORE calling `db.upsert_geometries(conn, geoms)`, perform collision detection:
  ```python
  ids = [g.id for g in geoms]
  if len(ids) != len(set(ids)):
      from collections import Counter
      duplicates = sorted(i for i, n in Counter(ids).items() if n > 1)
      msg = (
          f"layer {layer_config.layer_id} ({layer_config.species_slug}) produced "
          f"{len(ids) - len(set(ids))} duplicate geometry id(s): {duplicates[:5]}"
          + (f" ... ({len(duplicates) - 5} more)" if len(duplicates) > 5 else "")
      )
      raise ArcGISError(msg)
  ```
  Move the `Counter` import to the top of the file with the other stdlib imports rather than inline (keep the function body clean).

  - `ingestion/states/montana/fixtures/huntingDistricts-{3,10,11}-metadata-*.json` — S02.2 fixtures

- **Watch for:**
  - Don't write `verbatim_rule=""` to the DB — must be `None` when `REG` is empty (known pitfall).
  - SourceCitation built per-feature with REGYEAR (commit `e78a1c4` pattern), not once per layer.
  - `_build_session()` and `db.connect()` must both be context managers in a single `with` block (commit `4ab7cfc`).
  - Don't add a `slugify` library to `pyproject.toml` — inline regex is sufficient.
  - The metadata fetch for layers #4, #12, #13 happens during live load (post-build); do not block the build pipeline on it.

## Conventions

- ADR-014: `SourceCitation.document_type='gis_layer'` — handled inside `arcgis.build_source_citation`.
- ADR-015: `geometry.verbatim_rule` populated from `REG`; NULL when absent.
- Imports use `from ingestion.lib import arcgis, db` (matches `load_hds.py` lines 33-35).
- Test imports use `from states.montana.load_portions import ...` (matches `test_load_hds.py` line 13).
- Adapter scripts have no `__init__.py` (per README "Import path" section).
