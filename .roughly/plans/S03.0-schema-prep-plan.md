## Scope summary

S03.0 ships ADR-018's three coordinated schema additions in one migration:

1. New link table `license_season` (composite PK + secondary index)
2. New column `geometry.legal_description text` (nullable)
3. Extend `geometry.kind` CHECK constraint with `'state'`

…plus three-place schema sync (Pydantic + TypeScript + architecture.md), the canonical write path update in `db.py` (so `legal_description` round-trips), one Montana state geometry row written via a new load script (source choice + SHA recorded in calibration findings), and the two E03 working-artifact directory READMEs.

**No regulation data loaded.** Schema-prep only. The single `geometry` row is reference data.

### T1: Migration SQL — `license_season`, `geometry.legal_description`, `geometry.kind='state'` (~5 min)

**Files:** `supabase/migrations/20260504032424_e03_schema_additions.sql` (new)

**Action:** Create the single E03 schema-prep migration containing all three additions.

**Details:**

Mirror the header style from `supabase/migrations/20260428000000_geometry_verbatim_rule.sql:1-18` (`-- ===` separator, project name, story line, summary, ADR references). Use a single `BEGIN; ... COMMIT;` block — three additions ship atomically.

Body in three blocks, in this exact order (so `legal_description` and the `kind` extension can reference the same `geometry` table cleanly):

1. **`license_season` link table** — exact DDL from ADR-018 §1, plus the secondary index on `season_definition_id`:

   ```sql
   CREATE TABLE IF NOT EXISTS license_season (
       license_tag_id          text    NOT NULL
           REFERENCES license_tag(id),
       season_definition_id    text    NOT NULL
           REFERENCES season_definition(id),

       PRIMARY KEY (license_tag_id, season_definition_id)
   );

   CREATE INDEX IF NOT EXISTS license_season_season_definition_id_idx
       ON license_season (season_definition_id);
   ```

   Add a `--` comment block above the `CREATE TABLE` mirroring the explanatory style at `20260425000000_initial_schema.sql:227-230` for `regulation_season`. State explicitly: this is per-license season coverage (NOT per-regulation), it does not replace `regulation_season`, both coexist (per ADR-018 §1 and §"Consequences").

2. **`geometry.legal_description` column** — add via `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`. Mirror the canonical style at `20260425000000_initial_schema.sql:207` (the anchor named by the `.roughly/known-pitfalls.md` "Style anchor for adding a nullable text column" entry). `20260428000000_geometry_verbatim_rule.sql:20-21` uses the same pattern and is a valid secondary reference:

   ```sql
   ALTER TABLE geometry
       ADD COLUMN IF NOT EXISTS legal_description text;    -- NULLABLE — null = no FWP-published prose boundary description for this geometry
   ```

3. **`geometry.kind` CHECK constraint extension** — exact DDL from ADR-018 §3:

   ```sql
   ALTER TABLE geometry DROP CONSTRAINT geometry_kind_check;
   ALTER TABLE geometry ADD CONSTRAINT geometry_kind_check
       CHECK (kind IN (
           'hunting_district', 'gmu', 'portion', 'bmu',
           'cwd_zone', 'restricted_area', 'bma', 'state', 'other'
       ));
   ```

   The auto-generated constraint name `geometry_kind_check` is correct because the original CHECK at `20260425000000_initial_schema.sql:168-172` is a column-level inline CHECK (no explicit name → Postgres uses `<table>_<column>_check`). Confirm by reading those lines before writing — if the name diverges, fix the migration to match.

Header summary line should reference: ADR-006 (three-place sync), ADR-010 (decomposed entities), ADR-018 (this migration).

### T7: Investigate Montana state-boundary source + select + pin SHA (~10 min)

**Files:** `ingestion/states/montana/load_state_boundary.py` (created in T8 — this task only investigates and produces the URL + SHA values that T8 hardcodes)

**Action:** Per ADR-018 §3, investigate whether MT FWP publishes a state boundary; if it does, use it (with `document_type='gis_layer'`); if not, fall back to US Census TIGER 2020.

**Details:**

This task is pure investigation — it should not write code, only produce the four values T8 needs:

- `STATE_BOUNDARY_URL` — the chosen source URL
- `STATE_BOUNDARY_SHA256` — the file's SHA-256
- `STATE_BOUNDARY_SOURCE_TYPE` — `'gis_layer'` (FWP) or one of the existing literal values (TIGER 2020 → use `'annual_regulations'` is wrong; the existing Literal has no neutral fallback. **Open question for the implementer:** if TIGER 2020 is chosen, the `SourceCitation.document_type` Literal at `schema.py:73-76` has only `annual_regulations`, `rule_change`, `emergency_order`, `correction`, `gis_layer`. None semantically fits a federal Census boundary file. Two options: (a) extend the Literal in T2 to add `'reference_boundary'`; (b) use `'gis_layer'` as a shoehorn since TIGER ships boundary geometries even though it's not "an ArcGIS layer." **Default to (a) — extend the Literal — and update T2/T3/T4 accordingly.** Document the choice in T11.)
- `STATE_BOUNDARY_PUBLICATION_DATE` — for FWP-sourced, the pattern at `arcgis.py:986` (`f"{license_year:04d}-01-01"`); for TIGER, the TIGER 2020 release year (e.g., `"2020-01-01"`).

**Investigation steps:**

1. Check `ingestion/states/montana/sources.yaml` (if it exists yet) for any prior state-boundary entry. (Note: per the E03 spec at line 152, sources.yaml is created in S03.1 — it likely doesn't exist yet. Skip if absent.)
2. Curl the MT FWP on-prem ArcGIS catalog: `curl -s "https://fwp-gis.mt.gov/arcgis/rest/services?f=json"` and grep folder names + service descriptions for "state" / "boundary" / "border". Per the `.roughly/known-pitfalls.md` "MT FWP services" entry, also check the AGOL org: `curl -s "https://www.arcgis.com/sharing/rest/search?q=montana+state+boundary&f=json"` and filter to `owner=MtFishWildlifeParks`.
3. If a credible FWP state-boundary endpoint is found, use it. Document URL.
4. If not, use TIGER 2020. Confirm exact filename for the Montana state shapefile: it's typically `tl_2020_us_state.zip` (national, then filter to STATEFP=30 = Montana) at `https://www2.census.gov/geo/tiger/TIGER2020/STATE/`. Pin to that exact ZIP URL.
5. Download the chosen source ONCE: `curl -s -o /tmp/mt-boundary.<ext> <URL>`. Compute SHA-256: `shasum -a 256 /tmp/mt-boundary.<ext>`. Record both URL + SHA.
6. Verify the file decodes to a polygon covering Montana (rough bbox: longitude -116.05 to -104.04, latitude 44.36 to 49.0 — visible in `ingestion/tests/conftest.py:39-43` as the existing MT FWP test extent).

This task returns four values for T8. **No code is written here.** Findings are documented in T11.

**Verify:** SHA computed and recorded; URL responds 200; downloaded file is non-empty and decodes to a polygon (or a zip containing a shapefile that decodes to a polygon).

### T8: Build `load_state_boundary.py` — fetch, verify, parse, write Montana state geometry (~15 min)

**Files:** `ingestion/states/montana/load_state_boundary.py` (new)

**Depends on:** T2, T5, T7

**Action:** Build the deterministic, SHA-pinned, idempotent load script that writes `MT-STATEWIDE-geom`. Pure orchestration over `db.upsert_geometry`, `shapely.make_valid`, and the source-fetch path.

**Details:**

Mirror `ingestion/states/montana/load_hds.py:14-19` for the run-instructions docstring (`Run from the repo root: ... Required env: DATABASE_URL.`).

Module-level constants from T7:

```python
STATE_BOUNDARY_URL: str = "<from T7>"
STATE_BOUNDARY_SHA256: str = "<from T7>"
STATE_BOUNDARY_PUBLICATION_DATE: str = "<from T7>"
STATE_BOUNDARY_DOCUMENT_TYPE: Literal["gis_layer", "reference_boundary"] = "<from T7>"
MT_STATE_CODE = "US-MT"
MT_AGENCY = "Montana Fish, Wildlife & Parks" if FWP else "U.S. Census Bureau"
GEOMETRY_ID = "MT-STATEWIDE-geom"
GEOMETRY_NAME = "Montana"
```

Pure functions (each independently testable):

1. `_fetch_source_bytes(url: str, *, session: requests.Session | None = None) -> bytes` — single GET, returns the raw payload. No retries / throttling here unless T7's source is FWP-ArcGIS — in which case route through `arcgis._build_session()` for the User-Agent + throttle conventions per the `HUNTREADY_INGESTION_CONTACT` env var convention in `CLAUDE.md`. For TIGER, plain `requests.get()` is fine (single one-off fetch, federal source).
2. `_verify_sha256(payload: bytes, expected: str) -> None` — `hashlib.sha256(payload).hexdigest() != expected` → raise `RuntimeError` with both observed and expected SHAs. Fail loud.
3. `_parse_to_multipolygon_wkt(payload: bytes) -> str` — source-format-specific:
   - **FWP-ArcGIS path:** the payload is GeoJSON (or ArcGIS JSON if `?f=json`). Reuse `arcgis.geojson_to_multipolygon_wkt()` from `ingestion/ingestion/lib/arcgis.py:308` — it already handles the `make_valid()` repair-and-MultiPolygon-coerce contract. Pass the feature dict directly.
   - **TIGER 2020 path:** the payload is a ZIP containing a shapefile. Use `geopandas.read_file("zip://...")` (already a project dependency per `pyproject.toml`), filter to `STATEFP == "30"` (Montana = FIPS 30), extract the geometry. Then run `shapely.make_valid()` and the same Polygon → MultiPolygon coercion `arcgis.geojson_to_multipolygon_wkt` does internally. The function returns WKT.
4. `_build_source_citation() -> SourceCitation` — construct the `SourceCitation` with `id=f"mt-state-boundary-<source-slug>-2020"` (or similar deterministic string), `agency`, `title`, `url=STATE_BOUNDARY_URL`, `publication_date=STATE_BOUNDARY_PUBLICATION_DATE`, `document_type=STATE_BOUNDARY_DOCUMENT_TYPE`, `supersedes=None`, `page_reference=None`. Do not reuse `arcgis.build_source_citation` because that helper is hardcoded to `gis_layer` and to the ArcGIS service-URL slug pattern; T8's needs are different even if T7 chooses FWP.
5. `main()` — argparse with one optional flag `--service-url` defaulting to `STATE_BOUNDARY_URL` (so tests can override). Open a `db.connect()` connection, build the `Geometry` instance:

   ```python
   Geometry(
       id=GEOMETRY_ID,
       name=GEOMETRY_NAME,
       kind="state",
       geom=wkt,
       state=MT_STATE_CODE,
       license_year=None,        # year-invariant per ADR-018 §3
       verbatim_rule=None,
       legal_description=None,   # state-level boundary; no FWP prose description for state shape
       source=citation,
   )
   ```

   Call `db.upsert_geometry(conn, geom)` then `conn.commit()`. Single atomic commit (per the `.roughly/known-pitfalls.md` "Single atomic commit" entry — even though there's only one row, the pattern stays consistent: build everything inside the `with conn:` block, commit at the end).

   Logging: function-level `logger = logging.getLogger(__name__)` (per the `Logger Naming Split` known-pitfall — state adapters use function-level logger). Log: source URL, observed SHA, geometry area in km² (sanity check — Montana ≈ 380,800 km²; outside 350k–410k → WARN).

**Verify:**
- `cd ingestion && .venv/bin/ruff check ingestion/states/montana/load_state_boundary.py`
- `cd ingestion && .venv/bin/mypy ingestion/states/montana/load_state_boundary.py`
