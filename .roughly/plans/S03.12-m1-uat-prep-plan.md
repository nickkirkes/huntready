## Discovery-surfaced corrections

These deviations from the literal PRD/AC text are intentional and must be footnoted in the deliverables:

1. **`jurisdiction_code` format** — PRD says `HD-262`; actual DB value is `MT-HD-deer-elk-lion-262`. UAT queries use the DB format with a footnote pointing to `_dea_jurisdiction_code()` in `load_regulation_records.py:244-276`.
2. **`verbatim_text` column does not exist on `regulation_record`** — PRD criterion #1 says "non-empty verbatim_text"; Q15/OQ1 decomposed verbatim to `season_definition.verbatim_rule` + `license_tag.verbatim_rule`. Reframe criterion #1 to verify `source` jsonb populated + `additional_rules` populated + `confidence IN ('high','medium')`.
3. **HD 262 vs HD 170 split for criterion #2** — HD 262 elk lacks the asymmetric A/B pattern; HD 170 (Flathead River) is the asymmetric demonstrator per `TestBuildDeaLinkRows::test_license_season_asymmetric_coverage_m1_criterion`. UAT shows HD 262 for the "basic regulation_record lookup" (still satisfies criterion #1 + #3) and HD 170 for the "asymmetric license_season coverage" demonstration.
4. **`make ingest STATE=montana` does not exist** — no Makefile at repo root. Criterion #6 idempotency check uses the individual `python ingestion/states/montana/load_*.py` batch sequence from the S03.10 working note operator runbook (extracted into `M1-uat.md` before the working-notes directory is deleted).
5. **`license_season` RLS gap** — `license_season` was added by `20260504032424_e03_schema_additions.sql` AFTER the RLS migration `20260425000001_rls_deny_all.sql`; the deny-all policies cover only the original 10 tables. Criterion #7 query must check `license_season` and flag the gap if no policies exist for it. (Not fixing in S03.12 — flag in handoff as a potential M2-first-week or pre-M2-blocker depending on findings.)
6. **ADR-017 status** — file unmodified per S03.11 FINALIZE. Criterion #8 satisfied as-is.

   **Criterion #2 — A/B asymmetric license_season coverage**

   Two queries: (a) basic regulation_record → license_tag join for HD 262 elk (shows the seasons HD 262 elk has at all); (b) the asymmetric demonstrator for HD 170 elk:

   ```sql
   -- (b) HD 170 elk A/B asymmetric license_season coverage (M1 criterion #2 demonstrator)
   --
   -- Join chain explanation for operators new to the schema:
   --   regulation_record    ── the anchor entity (per HD × species × year)
   --   regulation_license   ── many-to-many link table: which license_tags apply to
   --                          this regulation_record (one row per applicable license)
   --   license_tag          ── the license/permit instrument (e.g., "General Elk License",
   --                          "Elk B License: 170-00")
   --   license_season       ── many-to-many link table per ADR-018: which seasons
   --                          this specific license is valid in (A and B may differ)
   --   season_definition    ── named date range with weapon/residency constraints
   SELECT lt.license_code, lt.name AS license_name, lt.kind AS license_kind,
          sd.id AS season_id, sd.name AS season_name,
          sd.opens, sd.closes, sd.weapon_type
   FROM regulation_record rr
   JOIN regulation_license rl
     ON (rl.state, rl.jurisdiction_code, rl.species_group, rl.license_year)
      = (rr.state, rr.jurisdiction_code, rr.species_group, rr.license_year)
   JOIN license_tag lt ON lt.id = rl.license_tag_id
   JOIN license_season ls ON ls.license_tag_id = lt.id
   JOIN season_definition sd ON sd.id = ls.season_definition_id
   WHERE rr.state = 'US-MT'
     AND rr.jurisdiction_code = 'MT-HD-deer-elk-lion-170'
     AND rr.species_group = 'elk'
     AND rr.license_year = 2026
   ORDER BY lt.license_code, sd.name;
   ```
   Expected: General Elk License covers {archery_only, general, heritage_muzzleloader}; Elk B License: 170-00 covers {archery_only, general, late}. A-only = {heritage_muzzleloader}; B-only = {late}. Footnote: "Spec named HD 262; S03.7 closure shifted target to HD 170 because HD 262 elk lacks asymmetry. Locked by `TestBuildDeaLinkRows::test_license_season_asymmetric_coverage_m1_criterion`."

   -- Idempotency baseline: capture before re-run, capture after, diff
   SELECT 'regulation_record' AS tbl, COUNT(*) FROM regulation_record WHERE state='US-MT'

   UNION ALL SELECT 'license_tag',           COUNT(*) FROM license_tag           WHERE id LIKE 'MT-%'
