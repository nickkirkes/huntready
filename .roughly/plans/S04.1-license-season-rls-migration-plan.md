# Implementation Plan: S04.1 — license_season RLS migration

## Summary

Add one new Supabase SQL migration `supabase/migrations/20260530<HHMMSS>_rls_license_season.sql` that closes the M1 UAT criterion #7 leak surface on `license_season`. The migration enables + forces RLS, declares deny-all policies for `authenticated` (first) and `anon` (second), and revokes table privileges from both roles — matching the per-table block shape used by the base RLS migration at `supabase/migrations/20260425000001_rls_deny_all.sql` for the 10 original tables.

- Base migration uses **unqualified** table names (`ALTER TABLE regulation_record`). (Grep-parity with the base migration is sacrificed for AC-string-parity; AC3's "parity with the 10 original tables" clause refers to structural parity — ENABLE + FORCE + two deny-alls + REVOKE — not to identifier qualification.)
- Header banner pattern is `-- ===` block with title, story line, three-section description, "Relevant ADRs" trailer.
- No transaction wrapper (no `BEGIN;` / `COMMIT;`).
- Policy name strings byte-identical: `"Deny all access for authenticated"` and `"Deny all access for anon"`.

```sql
-- =============================================================================
-- HuntReady: license_season RLS Deny-All (M1 carry-forward)
-- S04.1 — Row Level Security policies for the license_season link table
-- =============================================================================
-- license_season was added by 20260504032424_e03_schema_additions.sql (ADR-018)
-- after the base RLS deny-all migration (20260425000001_rls_deny_all.sql).
-- The base migration uses a flat per-table enumeration that does not auto-extend
-- to subsequently-added tables, leaving license_season with 14 privilege leaks
-- (every SELECT/INSERT/UPDATE/DELETE/REFERENCES/TRIGGER/TRUNCATE x {anon,
-- authenticated}) and zero RLS policies. M1 UAT 2026-05-28 confirmed the gap
-- (criterion #7 FAIL). This migration closes it.
--
-- Three security layers (matches every block in 20260425000001_rls_deny_all.sql):
--   Layer 1: ENABLE + FORCE RLS (applies even to table owner)
--   Layer 2: Deny-all policies for authenticated and anon roles
--   Layer 3: REVOKE ALL from anon and authenticated (defense-in-depth)
--
-- Relevant ADRs: 002, 004, 018
-- =============================================================================


-- -----------------------------------------------------------------------------
-- license_season
-- -----------------------------------------------------------------------------
ALTER TABLE public.license_season ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.license_season FORCE ROW LEVEL SECURITY;

CREATE POLICY "Deny all access for authenticated" ON public.license_season
  FOR ALL TO authenticated USING (false) WITH CHECK (false);
CREATE POLICY "Deny all access for anon" ON public.license_season
  FOR ALL TO anon USING (false) WITH CHECK (false);

REVOKE ALL ON TABLE public.license_season FROM anon, authenticated;
```

Conventions to preserve byte-exactly:
- **`public.license_season` qualifier on every reference** — matches the spec ACs at epic L124-128 verbatim. (Functional semantics are identical to the base migration's unqualified form with default `search_path`; the explicit qualifier is a spec-text-parity choice.)
- Policy declaration order: **authenticated FIRST, anon SECOND** — matches base migration lines 29-31 for every block.
- Policy text: `FOR ALL TO authenticated USING (false) WITH CHECK (false)` on a continuation line beginning with two spaces — matches base format at line 30.
- `REVOKE ALL ON TABLE` (explicit `ON TABLE` form) — matches base line 34, 48, 62 et al.
- Banner block uses `-- ===…` (77 `=` characters) — count matches base migration line 1 width.
- Two blank lines between the banner block and the first table section (matches base line 21-22 → line 23).
- Trailing newline at end of file (matches base behavior).
