- **Style parity with sibling loaders.** Discovery surveyed all 9 other Montana loaders; all call `basicConfig` inside `main()`, all use the `%(asctime)s %(name)s %(levelname)s %(message)s` format. The multi-line call form matches the majority (cleaner for ruff).
- **No new ADRs.** Operator-ergonomics hygiene per epic L295.
- **No new tests.** Epic L291 is explicit: "A guard like '`main()` calls `logging.basicConfig`' would lock implementation detail rather than behaviour." AC #5's 1166+2-skipped baseline is the only test-side gate.
- **basicConfig idempotency.** `logging.basicConfig` is documented as a no-op when the root logger already has handlers. If a parent process or test harness has already configured root logging, the new call is silently skipped — no side effect.
