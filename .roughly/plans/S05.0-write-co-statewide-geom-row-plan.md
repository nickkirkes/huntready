## Source-tier decision (locked at plan time)

**Source:** US Census TIGER 2024 nationwide state shapefile, filtered to STATEFP=08 (Colorado).

**URL:** `https://www2.census.gov/geo/tiger/TIGER2024/STATE/tl_2024_us_state.zip`

**Rationale:**
- Tier 1 (CPW ArcGIS state-boundary layer) — not confirmed to exist; investigation cost > deferral benefit.
- Tier 2 (Colorado Geospatial Portal) — not confirmed; same.
- Tier 3 (US Census TIGER) — guaranteed to exist; federal authoritative state-boundary source; `document_type='gis_layer'` per ADR-014; `geopandas` in tech stack supports shapefile-from-ZIP cleanly.
- MT precedent (Montana State Library MSDI) was preferred for 1:24,000 GCDB-alignment specific to MT survey markers; for HuntReady's M2 use case (state-level overlay anchoring `CO-STATEWIDE-{species}` regulation_records), 1:500,000 federal scale is sufficient.
- If a richer Colorado-published source surfaces during E05 implementation, the constants-block + `--service-url` override architecture supports a swap without breaking downstream stories.

**SHA-256:** Computed at T2 implementation time via `curl -s -L <url> | sha256sum`; pinned as `STATE_BOUNDARY_SHA256` constant. ADR-001 drift fails loud thereafter.

**SourceCitation.id:** `co-census-tiger-state-2024` (pattern: `<state>-<agency-short>-<product>-<year>`; MT analog was `mt-msdi-framework-boundaries-9-2026`).

**Agency / title:** `agency="US Census Bureau"`, `title="TIGER/Line State Shapefile 2024 (filtered to Colorado, STATEFP=08)"`.

**`publication_date`:** `"2026-01-01"` (REGYEAR-anchored Jan 1 of current regulation year per ADR-014; NOT the TIGER vintage year `"2024-01-01"` which would imply data publication date).

- **`_verify_sha256(payload, expected) -> str`:** Exact copy of MT's function. Computes `hashlib.sha256(payload).hexdigest()`; returns the observed digest on match; raises `RuntimeError` with both digests in the message on mismatch. Mention `STATE_BOUNDARY_SHA256` constant in the error message so operators know where to re-pin.

- **`_parse_to_multipolygon_wkt(payload) -> str`:** The TIGER-specific branch (structurally DIFFERENT from MT's GeoJSON path).
  - Wrap `payload` in `io.BytesIO`; open as `zipfile.ZipFile`.
  - Extract the `.shp/.shx/.dbf/.prj` files to a `tempfile.TemporaryDirectory` (geopandas needs disk-resident sidecar files). Use `zf.extractall(tmpdir)`.
  - Find the `.shp` file inside tmpdir: `shp_path = next(p for p in pathlib.Path(tmpdir).glob("*.shp"))`. If no `.shp` found, raise `RuntimeError("TIGER ZIP missing .shp file")`.
  - `gdf = geopandas.read_file(shp_path)`.
  - **Reproject unconditionally:** `gdf = gdf.to_crs(4326)`. TIGER shapefiles are EPSG:4269 (NAD83) by default; reprojection is required for the resulting WKT to round-trip cleanly through Postgres `ST_GeomFromText(wkt, 4326)::geography`. **Critical:** `geopandas.GeoDataFrame.to_crs(4326)` returns a **new** GeoDataFrame — it does NOT mutate `gdf` in place. The result must be reassigned to `gdf` (as shown). A bare `gdf.to_crs(4326)` discards the reprojection silently; the subsequent filter and `geom.wkt` would carry NAD83 coordinates that Postgres would mis-interpret as WGS84, producing a ~100m systematic offset that the AC #8 area sanity check (~269,837 km² ±1%) cannot catch.
  - Filter: `colorado = gdf[gdf["STATEFP"] == CO_STATEFP]`. Assert `len(colorado) == 1` (raise on 0 or >1, naming the filter).
  - Extract geometry: `geom = colorado.iloc[0].geometry`. Run `shapely.make_valid(geom)` (or use `geopandas`'s built-in `.make_valid()` if exposed in the version on disk).
  - Coerce singleton `Polygon` → `MultiPolygon([poly])` (Colorado is single-part). If geom is already `MultiPolygon`, no wrap needed.
  - Return `geom.wkt`. **Do NOT call** `arcgis.geojson_to_multipolygon_wkt` — that helper expects GeoJSON-feature dict input and would not handle shapefile-derived shapely objects.

- **`_build_source_citation() -> SourceCitation`:** Direct construction (do NOT use `arcgis.build_source_citation` — TIGER URL is not an ArcGIS service URL; mirror MT pattern lines 184-200).

  ```python
  def _make_tiger_zip(geometries_with_statefp: list[tuple[str, shapely.geometry.base.BaseGeometry]]) -> bytes:
      """Build a synthetic TIGER-shaped ZIP containing a shapefile.

      Each element is (STATEFP, geometry). The function writes the records to a temp
      shapefile via geopandas, then ZIPs the .shp/.shx/.dbf/.prj sidecars."""
      import geopandas as gpd
      with tempfile.TemporaryDirectory() as tmpdir:
          tmpdir_path = pathlib.Path(tmpdir)
          gdf = gpd.GeoDataFrame(
              {"STATEFP": [statefp for statefp, _ in geometries_with_statefp]},
              geometry=[g for _, g in geometries_with_statefp],
              crs="EPSG:4269",  # NAD83, matches TIGER native CRS
          )
          shp_path = tmpdir_path / "tl_2024_us_state.shp"
          gdf.to_file(shp_path)
          # ZIP all sidecar files
          buf = io.BytesIO()
          with zipfile.ZipFile(buf, "w", zipfile.ZIP_DEFLATED) as zf:
              for p in tmpdir_path.iterdir():
                  zf.write(p, arcname=p.name)
          return buf.getvalue()
  ```
