# Implementation Plan: S05.2 — GMU ingestion (CPW FeatureServer layer 6)

Plan-format-version: 1

## Scope summary

New Colorado loader `ingestion/states/colorado/load_gmus.py` that fetches all ~186 CPW
Game Management Unit polygons from CPWAdminData FeatureServer layer 6 and writes them to
the `geometry` table with `kind='gmu'`, deterministic IDs from `GMUID`, per-feature
provenance, validity guards, and a pre-`db.connect()` count-band guard. New mocked test
suite `ingestion/tests/test_load_gmus.py`. Mirrors `ingestion/states/montana/load_hds.py`
structure with single-layer simplifications.

**Code-now vs operator-deferred (Group A / Group B split, per S05.0 precedent):**
- **Group A (this build):** `load_gmus.py`, `test_load_gmus.py` (mocked), all quality gates green.
  No live network, no live DB write.
- **Group B (operator-deferred):** committing the live `CPWAdminData-6-metadata-<ts>.json` +
  `CPWAdminData-6-manifest-<ts>.json` fixtures, `multipart-gmus.json`, the live row-count
  cross-check against `returnCountOnly=true`, and post-insert `ST_IsValid` verification all
  require a live fetch + DB write, which is operator-driven (mirrors S05.0 Group B and
  S02.1→S02.2). The loader CODE produces all of these when run; the build delivers the code
  + mocked tests. This split is surfaced at the Stage 5 completion gate.

## Resolved spec-vs-codebase conflicts (from discovery)

1. **`SourceCitation` has no `notes` field and `extra="forbid"`.** AC line 235 ("capture
   `EDIT_DATE`/`INPUT_DATE` in `geometry.source` jsonb's free-form `notes`/extension field")
   CANNOT be satisfied — passing `notes=` raises `ValidationError`, and the spec forbids
   schema migrations in E05. **Resolution: log-only.** The loader emits `EDIT_DATE` and
   `INPUT_DATE` per feature as INFO log output; they are not persisted to the DB row. Documented
   in a load-bearing comment + the closure note. This is the conventional V1 disposition.
2. **`_check_and_fix_projection` is at `arcgis.py:444`**, not the spec's `:397-408`. Reference
   by name only; no line-number citation in code/docstrings (S05.1 plan-review precedent).
3. **Citation id is case-preserved**: `build_source_citation` builds
   `id=f"{state_slug}-arcgis-{service_slug}-{layer_id}-{license_year}"` where `service_slug`
   comes from `_slug_from_service(service_url)` (case-preserved URL segment → `"CPWAdminData"`).
   With `state_slug="co-cpw"` the produced id is `co-cpw-arcgis-CPWAdminData-6-2026` — NOT the
   lowercase `co-cpw-arcgis-cpwadmindata-6-...` in sources.yaml's `id_template` (which is
   documentation only, never read at runtime). The test asserts against the value
   `build_source_citation` actually produces, not a hand-typed string.
4. **`ColoradoGeometryError` does not exist** — define it new in `load_gmus.py`. The shared lib's
   `geojson_to_multipolygon_wkt` raises `ArcGISError` for unrecoverable geometry; the loader
   catches that in the per-feature loop and re-raises as `ColoradoGeometryError` with `GMUID` +
   geometry-type tally context.

### T2: `_feature_to_geometry` pure function (~5 min)
**Files:** ingestion/states/colorado/load_gmus.py
**Depends on:** T1
**Action:** Add the per-feature normalization function converting one GeoJSON feature into a
`Geometry` instance, with GMUID-derived id and ArcGISError→ColoradoGeometryError wrapping.
**Details:**
- Signature: `def _feature_to_geometry(feature: dict[str, Any], layer_metadata: LayerMetadata,
  service_url: str, fetch_year: int, *, logger: logging.Logger) -> Geometry:` (logger passed in
  so EDIT_DATE/INPUT_DATE forensic INFO lines are emitted during normalization; the function
  stays otherwise pure/testable).
- Body:
  - `props = feature["properties"]`
  - `gmuid = _extract_gmuid(props, layer_metadata)`
  - Convert geometry with ArcGISError→ColoradoGeometryError wrapping:
    ```python
    try:
        geometry_wkt = arcgis.geojson_to_multipolygon_wkt(feature)
    except ArcGISError as exc:
        msg = f"GMU {gmuid}: geometry could not be coerced to MultiPolygon ({exc})"
        raise ColoradoGeometryError(msg) from exc
    ```
    (The shared helper already raises `ArcGISError` with a geometry-type tally on
    GeometryCollection-after-make_valid; wrapping adds the GMUID. The spec AC line 230 wants
    GMUID + type tally in the message — `exc` carries the tally, the wrapper prepends GMUID.)
  - Forensic logging (log-only per resolved conflict #1): if `props.get("EDIT_DATE")` or
    `props.get("INPUT_DATE")` present, `logger.info("GMU %s provenance: EDIT_DATE=%s INPUT_DATE=%s",
    gmuid, props.get("EDIT_DATE"), props.get("INPUT_DATE"))`. Add a comment: these are NOT written
    to the row — SourceCitation carries no forensic field and ADR-014 anchors publication_date to
    REGYEAR.
  - `county = _extract_county(props)`; build name:
    `name = f"GMU {gmuid} ({county})" if county else f"GMU {gmuid}"`
  - `citation = arcgis.build_source_citation(service_url=service_url, layer_id=CPW_GMU_LAYER_ID,
    metadata=layer_metadata, license_year=fetch_year, state_slug=CO_STATE_SLUG, agency=CO_AGENCY)`
    (Pass `fetch_year` directly — no per-feature REGYEAR exists. Citation gets
    `publication_date=f"{fetch_year:04d}-01-01"` = `"2026-01-01"`.)
  - `return Geometry(id=f"{CO_GMU_ID_PREFIX}-{gmuid}-geom", name=name, kind="gmu",
    geom=geometry_wkt, state=CO_STATE_CODE, license_year=None, source=citation)`
    (`license_year=None` on the Geometry row per AC line 233; `verbatim_rule`/`legal_description`
    default to None and are omitted.)

### T3: Multipart-GMU collection + pre-connect guards (~5 min)
**Files:** ingestion/states/colorado/load_gmus.py
**Depends on:** T2
**Action:** Add the duplicate-id guard, the count-band guard, and the multipart-GMU collection +

**Details:**

- `def _check_count_band(n: int) -> None:` — raise `RuntimeError` if `n` outside
  `_GMU_COUNT_GUARD_BAND` (inclusive), message naming observed count + band + the

- `def _collect_multipart_gmus(geoms: list[Geometry]) -> list[dict[str, Any]]:` — for each
  Geometry, parse its `geom` WKT via `wkt.loads(...)` (the `from shapely import wkt` import is
  already in T1's import list), inspect
  `.geoms` length; for parts > 1, append `{"gmuid": <int parsed from id>, "part_count": len(parts),

  store `part_count` and a planar-area proxy is misleading — instead derive `gmuid` from the

  `total_area_sq_km` to `None` with a comment that authoritative area comes from the post-load

  faithfully). Keep the key present (value `None`) so the fixture schema matches the spec AC's
  field list. *(If the reviewer prefers a populated value, the operator-run S05.7 query fills it;
  do not fabricate a planar approximation.)*
- `def _write_multipart_fixture(rows: list[dict[str, Any]], path: Path) -> None:` — write
  `json.dumps(rows, indent=2, sort_keys=True) + "\n"` to `path` (deterministic ordering; sort the
  rows list by gmuid before dumping). Create parent dir with `path.parent.mkdir(parents=True,
  exist_ok=True)`.

### T4: `_fetch_and_build` + `_write_geometries` + `main()` (~5 min)
**Files:** ingestion/states/colorado/load_gmus.py
**Depends on:** T3
**Action:** Add the fetch→build orchestration, the DB-write half, and the CLI entrypoint. The
fetch+build and the DB write are **two separate functions** so the guards can fire BEFORE
`db.connect()` per OQ7 discipline — there is no combined `_load_layer`.
**Details:**
- `def _fetch_and_build(service_url: str, fixture_dir: Path, fetch_year: int, *,
  session: requests.Session | None = None, logger: logging.Logger) -> list[Geometry]:`
  (network + normalization; no DB connection)
  - `metadata = arcgis.fetch_layer_metadata(service_url, CPW_GMU_LAYER_ID, fixture_dir,
    layer_slug=CO_LAYER_SLUG, session=session)`
  - `features = arcgis.fetch_features(service_url, CPW_GMU_LAYER_ID, metadata, fixture_dir,
    layer_slug=CO_LAYER_SLUG, session=session)`
  - `geoms = [_feature_to_geometry(f, metadata, service_url, fetch_year, logger=logger) for f in features]`
  - `return geoms`
- `def _write_geometries(conn: psycopg.Connection[Any], geoms: list[Geometry]) -> None:` —
  `db.upsert_geometries(conn, geoms)` (caller owns the commit). Add a one-line comment that the
  count-band + duplicate-id guards run in `main()` before this is ever reached.
- `def main(argv: list[str] | None = None) -> int:`
  - argparse with `--fetch-year` (default `_FETCH_YEAR`); **no `--service-url` flag** (S05.0
    cubic-fix precedent: a service-url override creates a silent-lie citation surface). Use the
    module constant `CPW_SERVICE_URL` directly.
  - `logging.basicConfig(level=logging.INFO, format="%(asctime)s %(name)s %(levelname)s %(message)s")`
    then `logger = logging.getLogger(__name__)` (function-level logger per state-adapter convention).
  - `CO_FIXTURE_DIR.mkdir(parents=True, exist_ok=True)`
  - Fetch + build inside the session block, BEFORE db.connect():
    ```python
    with arcgis._build_session() as session:
        geoms = _fetch_and_build(CPW_SERVICE_URL, CO_FIXTURE_DIR, args.fetch_year,
                                 session=session, logger=logger)
    ```
  - Guards (pre-connect): `dupes = _duplicate_ids(geoms)`; if dupes, raise `RuntimeError` listing
    up to 5. `_check_count_band(len(geoms))`.
  - Multipart fixture: `multipart = _collect_multipart_gmus(geoms);
    _write_multipart_fixture(multipart, MULTIPART_FIXTURE_PATH);
    logger.info("multi-part GMUs: %d", len(multipart))` and an INFO line per multipart GMU.
  - DB write phase: `with db.connect() as conn: _write_geometries(conn, geoms); conn.commit()`
    then `logger.info("upserted %d GMU geometries", len(geoms))`.
  - `return 0`
- `if __name__ == "__main__": raise SystemExit(main())`

## Conventions
- Import form `from ingestion.lib import arcgis, db` (NOT `from ingestion.lib.arcgis import ...`) —
  obs #2660, S05.1 precedent.
- Function-level `logger = logging.getLogger(__name__)` inside `main()`; lib uses module-level
  `_logger` — obs #1049.
- `logging.basicConfig(...)` inside `main()` (S04.3 / load_hds.py pattern).
- No `--service-url` CLI flag (S05.0 cubic-fix: silent-lie citation surface).
- Run scripts as `python <path>`, not `python -m` (editable-install namespace-package pitfall).
- Test monkeypatch string paths use `states.colorado.load_gmus.<attr>`.
- Fixtures: `CPWAdminData-6-metadata-<ts>.json` + `-manifest-<ts>.json` written automatically by
  `fetch_layer_metadata`/`fetch_features`; `multipart-gmus.json` fixed-path, written by the loader.
- ADRs: ADR-001 (fail-loud, authority), ADR-005 (state isolation), ADR-010 (MultiPolygon/geography),
  ADR-014 (gis_layer + REGYEAR-anchored publication_date).
