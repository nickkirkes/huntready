# Implementation Plan: S05.4 — Colorado restricted-area / no-hunt-zone overlay ingestion

Plan-format-version: 1

## Summary

Ingest **10 V1 federal no-hunt zones** into the `geometry` table from the USGS PAD-US 4.1 Federal Fee
Managers Authoritative FeatureServer: 4 National Parks (Rocky Mountain, Mesa Verde, Great Sand Dunes,
Black Canyon of the Gunnison) + 5 National Monuments (Dinosaur, Colorado NM, Florissant Fossil Beds,
Hovenweep, Yucca House) + 1 DOD installation (United States Air Force Academy). All rows
`kind='restricted_area'`, id `CO-restricted-{slug}-geom`, `geography(MultiPolygon, 4326)`, valid.

**Scope correction baked into this plan:** the research doc
(`docs/research/colorado-restricted-areas-evaluation.md`) says "9 zones" in prose but "4 NPs + 5 NMs +
AFA" = **10**, its enumerated named list (lines 223–232) has **10** entries, and the fetch-and-drop
math (11 fetched − Curecanti = 10) confirms **10**. The "9" is an arithmetic slip. This plan uses
**10** as the authoritative V1 count; Group B's `returnCountOnly` empirically confirms.

**Group A (this session):** loader code + mocked tests + investigation report + `sources.yaml` entry +
Group-B runbook. No live network, no DB write. **Group B (operator-pending):** live PAD-US fetch +
production DB write + `returnCountOnly` cross-check + `ST_IsValid` round-trip + GIS_Acres ±10% sanity.
Mirrors S05.0 / S05.2 exactly.

**Confirmed decisions (user-approved at Stage 1):**
- `geometry.verbatim_rule = None` for V1 (PAD-US carries no regulatory text; CPW Big Game brochure URL
  unresolved/404). The brochure is documented as a precondition for the future text-population step
  (E06 / Group B), NOT populated from a guess (ADR-001 / ADR-008 discipline).
- `source.document_type='gis_layer'` — geometry sourced from PAD-US over ArcGIS REST per AC line 450.
  The single `source` field cites PAD-US, not the brochure.
- S05.4 writes **geometry rows only** — no `jurisdiction_binding` (E06 territory; no `role` column on
  `geometry`). The `role='no_hunt_zone'` semantics are documented for E06, not written here.

## Conventions

- **Structural template:** mirror `ingestion/states/colorado/load_gmus.py` (S05.2) — pure extractors,
  `_feature_to_geometry`, fetch+build split from DB write, guards fire **before** `db.connect()` (OQ7
  discipline), `main()` with `logging.basicConfig` and function-level logger, **no `--service-url`
  flag** (S05.0 cubic-fix silent-lie-citation precedent).
- **Zero `ingestion/lib/` edits** (ADR-005; locked by the CO-leak guard tests). The custom WHERE-filtered
  fetch composes existing **private** arcgis primitives (`_build_session`, `_request_with_retry`,
  `_check_and_fix_projection`, `_write_features_fixture`, `_write_manifest_fixture`,
  `compute_feature_hash`, `geojson_to_multipolygon_wkt`, `fetch_layer_metadata`). The private-call
  convention is established (S05.2 observation 2698; `_build_session` already called by load_gmus.py).
- **Rejected alternative (documented for reviewers):** adding an optional `where` param to
  `arcgis.fetch_features`. Rejected because (a) every prior E05 story held the zero-lib-edit line and
  there is an explicit CO-leak guard, and (b) the V1 set is 10 features — well under one page — so the
  fragile pagination loop in `fetch_features` is unnecessary; a bounded single-page fetch with an
  explicit transfer-limit-not-exceeded assert is simpler and lower-risk than either replicating
  pagination or mutating a shared primitive exercised by MT loaders.
- **ADR lineage:** ADR-001 (fail-loud / no invented data), ADR-005 (state isolation), ADR-008
  (verbatim — empty for V1, no fabrication), ADR-010 (MultiPolygon/geography), ADR-014 (`gis_layer`
  document_type; `publication_date` = fetch-year Jan 1, NOT `editingInfo.lastEditDate`), ADR-015
  (verbatim_rule nullable). ADR-021 context only (E06 uses `no_hunt_zone`).
- **CO conventions:** `CO_STATE_CODE = "US-CO"`; `CO_FIXTURE_DIR = Path(__file__).parent / "fixtures"`;
  function-level `logger = logging.getLogger(__name__)` inside `main()`; single `conn.commit()`.

### T5: Custom WHERE-filtered fetch + build (~5 min)
**Files:** ingestion/states/colorado/load_restricted_areas.py

  layer_id=PADUS_LAYER_ID, metadata=metadata, license_year=fetch_year, state_slug=CO_STATE_SLUG,
  agency=CO_AGENCY)`.
- `host = urlparse(service_url).hostname or ""`; `query_url = f"{service_url}/{PADUS_LAYER_ID}/query"`.
- Count cross-check: `count_resp = arcgis._request_with_retry(session, query_url,
  params={"where": _V1_WHERE, "returnCountOnly": "true", "f": "json"}, host=host)`; require `"count"`
  key (raise `ArcGISError` if absent, naming the url); `expected_fetched = int(count_resp["count"])`.
- Single feature page: `page = arcgis._request_with_retry(session, query_url, params={"where": _V1_WHERE,
  "outFields": ",".join(_RA_OUT_FIELDS), "returnGeometry": "true", "outSR": 4326, "f": "geojson",
  "returnTrueCurves": "false", "geometryPrecision": 7}, host=host)`.
- `features = page.get("features") or []`.
- **Transfer-limit assert (belt-and-suspenders):** if `page.get("exceededTransferLimit")` is truthy OR
  `len(features) != expected_fetched`, raise `ArcGISError` explaining the V1 set should be a single small
  page and a mismatch means the WHERE clause or layer changed (cross-check `returnCountOnly`). This
  replaces the pagination loop safely because the V1 set is ~11 rows, far under `max_record_count`.
- **Projection guard:** `checked = arcgis._check_and_fix_projection(features,
  declared_native_crs_wkid=metadata.spatial_reference_wkid)`.

- Build: `geoms = [_feature_to_geometry(f, metadata, citation, logger=logger) for f in kept_features]`.
- Return `geoms`.
