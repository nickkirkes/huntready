The runbook + working note carry a documented generator step (`extensions.ST_PointOnSurface` over live geometry) that the operator runs during the batched live-write session and commits. This honors AC #2's "real `shapely.representative_point()` … no invented points" requirement (Option B — authoring named coordinates at-merge — would violate "no invented points" and is rejected).

### T1: CO geometry verification runbook (~8 min)
**Files:** docs/runbooks/E05-colorado-geometry-verification.md
**Action:** Create the Colorado operator runbook, CO analog of `docs/runbooks/E02-geometry-verification.md` (~264 LOC target).
**Details:**
Mirror the E02 runbook's structure (read `docs/runbooks/E02-geometry-verification.md` in full first) with these MANDATORY CO substitutions:

- **Title + intro:** "E05 Colorado Geometry Verification Runbook". Verifies E05 CO geometry rows (S05.0–S05.5) loaded correctly. Reference ADR-004 + ADR-016; CO overlay artifact at `ingestion/states/colorado/fixtures/geometry-overlays.json` (note: operator-generated per S05.5 Group B).
- **Every `ST_*` call MUST be `extensions.`-prefixed** (Supabase quirk per `.roughly/known-pitfalls.md` — Integration / Supabase / PostGIS). This is the explicit departure from the E02 runbook, which used bare `ST_*` because the quirk was learned later. Examples: `extensions.ST_Covers(geom, extensions.ST_GeogFromText('SRID=4326;POINT(<lng> <lat>)'))`; `extensions.ST_IsValid(extensions.ST_GeomFromText(extensions.ST_AsText(geom), 4326))`; `extensions.ST_NumGeometries(...)`; `extensions.ST_Envelope(extensions.ST_Collect(geom::geometry))`.
- **Every spot-check / topology / query SQL block MUST include `AND state = 'US-CO'`** in the WHERE clause (PRD 002 SC#4). The E02 runbook omitted this because MT was the only state.
- **Prerequisites section:** service-role `DATABASE_URL`; CO geometry rows loaded (S05.0 + S05.2 + S05.4 — note S05.3 produced zero CWD rows; expected CO total in band `[~197, ~250]`: 1 statewide + ~186 GMUs + 10 restricted areas); the `spatial-test-points.json` fixture (note it is **operator-generated** per Section 0 below — it is NOT committed at-merge, unlike MT's). Loaded `ingestion/.venv/`.
- **NEW Section 0 — Generate `spatial-test-points.json` (Group B, run first):** Document the generator. After CO geometry rows exist live, run a query to derive real representative points per kind:
  ```sql
  SELECT id, kind, extensions.ST_AsText(extensions.ST_PointOnSurface(extensions.ST_GeomFromText(extensions.ST_AsText(geom), 4326))) AS pt
  FROM geometry WHERE state = 'US-CO' ORDER BY kind, id;
  ```
  Operator selects ≥3 GMUs (incl. the first entry of `multipart-gmus.json` as the multi-part anchor), all/representative restricted areas, the statewide point, and adds ONE outside-CO negative control (a real point just over the WY or KS line, e.g. `lng=-104.0, lat=41.5` in WY — documented as the only non-`ST_PointOnSurface` coordinate, used purely to confirm the `state='US-CO'` filter returns zero rows). Write the fixture in the exact MT schema: `{"version": 1, "points": [{name, lat, lng, expected_kind, expected_id_pattern, expected_role_for_e03, notes}]}`. For CO, `expected_role_for_e03` is `null` for ALL points (no `jurisdiction_binding` rows exist in E05 — bindings are E06). Commit the fixture.
- **Section 1 — Spot-check via `extensions.ST_Covers`:** drive from `spatial-test-points.json`; include the one-shot Python loop variant (adapt MT's, reading `states/colorado/fixtures/spatial-test-points.json`, with `AND state = 'US-CO'` in the SQL). Expected outputs by entry: GMU/restricted-area entries return one row matching `expected_id_pattern`; the negative control returns zero rows. Note CO restricted areas are ALL orphans (no parent GMU) per S05.5 `EXPECTED_CO_RA_ORPHAN_IDS` — `ST_Covers` still returns the zone's own row. **Two-row clarification (per plan review):** a point inside a restricted area that geographically overlaps a GMU may return BOTH rows (one `restricted_area`, one `gmu`) — both are correct results since CO restricted areas are spatially adjacent to / overlapping with GMUs; document this so the operator does not read the extra row as an error.
- **Section 2 — Topology validity:** `SELECT id FROM geometry WHERE NOT extensions.ST_IsValid(extensions.ST_GeomFromText(extensions.ST_AsText(geom), 4326)) AND state = 'US-CO';` → zero rows. (PRD 002 SC#6.)
- **Section 3 — Multi-part GMU verification (named):** read first entry of `ingestion/states/colorado/fixtures/multipart-gmus.json` as the named anchor; `SELECT id, extensions.ST_NumGeometries(extensions.ST_GeomFromText(extensions.ST_AsText(geom), 4326)) AS parts FROM geometry WHERE id = '<anchor>' AND state = 'US-CO';` → `parts > 1`. **Fallback callout:** if `multipart-gmus.json` is empty (CO has zero multi-part GMUs), document as N/A and surface as a data observation — do not silently skip.
- **Section 4 — `EXPLAIN ANALYZE`:** representative point-in-polygon query (a real CO interior point); document the chosen plan inline. Note the geography GiST is the existing `geometry_geom_gix` from E01; both Index Scan and Seq Scan are acceptable for the ~197-row CO subset (cost-driven). Carry MT's "overlay computation is NOT a SQL operation" callout (Supabase 2-min `statement_timeout`).
- **NEW Section 5 — CO-bounds `ST_Envelope` check** (validation-triad resolution): `SELECT extensions.ST_Envelope(extensions.ST_Collect(geom::geometry)) FROM geometry WHERE state = 'US-CO';` → bbox within `[-109.06, -102.04] × [36.99, 41.00]`. Note this is the analytical-layer CO-specific check the shared lib's fetch-layer guard does NOT perform.
- **Section 6 — Reproducibility (wipe + re-ingest):** snapshot (CO-scoped), then the **FK-cascade-correct wipe sequence**:
  ```sql
  DELETE FROM jurisdiction_binding WHERE state = 'US-CO';  -- MUST run first (FK: jurisdiction_binding.geometry_id REFERENCES geometry(id))
  DELETE FROM geometry WHERE state = 'US-CO';              -- then geometry
  -- DO NOT use TRUNCATE: it wipes EVERY state (MT included) and breaks reproducibility.
  ```
  Re-ingest commands (`load_state_boundary.py`, `load_gmus.py`, `load_restricted_areas.py`). Topological-equality compare (`extensions.ST_Equals` with WKT round-trip on both sides), row-count + id-set parity (both directions), and the idempotency assertion (re-run = zero net new rows — PRD 002 SC#7). Note: in E05 no CO `jurisdiction_binding` rows exist yet (E06), so the first DELETE is a no-op today but the **sequence MUST be documented** for when E06 lands.
- **Section 7 — Overlay architecture (informational):** adapt MT's; CO builder at `ingestion/states/colorado/build_overlay_fixture.py`; same ADR-016 three-band discriminator; CO has GMU parents (not HD), no portions; `cwd_zone` vacuously empty; the 10 CO restricted areas are all `EXPECTED_CO_RA_ORPHAN_IDS` orphans. CO overlay fixture + dropped-audit are operator-generated (S05.5 Group B).
- **Cross-state filter regression note:** add a short subsection documenting that `test_co_binding_loader_sql_filters_by_state_co_pollution_guard` (S05.6) passes as the cross-state UAT step.
- **Cleanup section:** drop `geometry_snapshot` if created; confirm CO row count restored.
**Verify:** `grep -c "extensions.ST_" docs/runbooks/E05-colorado-geometry-verification.md` returns a positive count AND `grep -n "[^.]ST_[A-Z]" docs/runbooks/E05-colorado-geometry-verification.md` returns no bare (non-`extensions.`-prefixed, non-`_ST_`) `ST_*` calls in SQL blocks (allow prose mentions like "ST_Covers predicate"); `grep -q "DELETE FROM jurisdiction_binding WHERE state = 'US-CO'" docs/runbooks/E05-colorado-geometry-verification.md` AND `grep -q "DELETE FROM geometry WHERE state = 'US-CO'" docs/runbooks/E05-colorado-geometry-verification.md` both succeed, with the jurisdiction_binding DELETE appearing first; `grep -q "TRUNCATE" docs/runbooks/E05-colorado-geometry-verification.md` succeeds (the guard text).
