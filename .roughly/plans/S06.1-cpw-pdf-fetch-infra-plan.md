# Implementation Plan: S06.1 — CPW PDF fetch infrastructure + CO sources.yaml `pdfs:` section

Plan-format-version: 1

## Summary

Stand up Colorado's PDF fetch infrastructure by mirroring Montana's S03.1 pattern: a
state-adapter orchestrator (`states/colorado/fetch_pdfs.py`) that reuses the locked
shared-lib primitive `ingestion.lib.pdf_fetch.fetch_pdf` **without modifying the lib**,
a new `pdfs:` section in CO `sources.yaml` carrying the two operator-resolved CPW PDFs
(both fully pinned: URL + SHA-256 + page count), a gitignore update so raw PDFs and
drift markers stay local, and an additive CO test file. No live network fetch occurs in
the build session — first-fetch manifests are operator-pending (S03.1 / E05 Group-B
posture). Both `document_type` values (`annual_regulations`, `correction`) are already in
the shared lib's `_VALID_DOCUMENT_TYPES`, so no lib change is needed.
