## KI#18 SQL corrections (exactly 3) + PostGIS idiom (apply in every runbook query)
1. `regulation_season` / `regulation_license` filter by `state='US-CO'` (composite FK `(regulation_record_state, regulation_record_id)`; NO bare `regulation_record_id` prefix filter).
2. `draw_spec.inactive_forfeit_years` is a `point_system` jsonb field → `WHERE point_system->>'inactive_forfeit_years' IS NULL` (NOT a column).
3. `jurisdiction_binding` / `regulation_reporting` FK to `regulation_record` via composite `(state, jurisdiction_code, species_group, license_year)` (NOT a single `regulation_record_id`); `jurisdiction_binding` has NO `state` column — scope by `geometry_id IN (SELECT id FROM geometry WHERE state='US-CO')` or `regulation_record_state`.
**PostGIS idiom (a general query convention — NOT one of the 3 KI#18 corrections above; keep it labeled separately per F1):** All PostGIS `extensions.`-prefixed; geometry-only funcs use WKT round-trip `extensions.ST_GeomFromText(extensions.ST_AsText(geom),4326)` (Supabase rejects bare `geom::geometry`, SQLSTATE 42846); `extensions.ST_Covers(geom, <geography point>)` needs no round-trip (still `extensions.`-prefixed — only the WKT round-trip is what it skips).
