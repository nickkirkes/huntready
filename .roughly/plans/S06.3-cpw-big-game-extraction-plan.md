# Implementation Plan: S06.3 — CPW Big Game brochure extraction (deer + elk + pronghorn)

Plan-format-version: 1

## Goal

Build `ingestion/states/colorado/extract_big_game.py` — a state-agnostic-clean extractor that parses the
84-page 2026 CPW Big Game brochure into a deterministic `ingestion/states/colorado/extracted/big-game-2026.json`
artifact (one row per per-GMU per-species regulation row, verbatim text + per-row season windows + ADR-017
confidence), plus `ingestion/tests/test_extract_co_big_game.py` (30+ tests). Structural analog of MT's
`extract_dea.py`. Downstream S06.6/S06.7/S06.8 consume the artifact.

## Live-PDF ground truth (from Stage-2 discovery; probed against the real bytes)

- PDF: `ingestion/states/colorado/fixtures/co-cpw-big-game-2026-brochure-2026-03-04.pdf` (84 pp, fetched, SHA-256 `38cf26e1…3582b3`).
- Manifest: `…-2026-03-04-pdf-manifest.json` present; `fetched_at` = `2026-06-09T14:39:44.025982+00:00` (thread as `extracted_at`).
- **Method-organized tables** (NOT unit-organized like MT). Each table stacks all GMUs for one method-of-take.
- Page map (1-based, CONFIRM exact bounds at T1 against the live PDF): deer ≈ 30–43, elk ≈ 45–62, pronghorn ≈ 63–67.
- **Dual-column-block tables**: deer-rifle / pronghorn ≈ 12 cols (two 6-col blocks); elk-archery ≈ 10 cols (no Season-Dates column — date in the section header). Column count varies by method → dispatch a parser per method.
- **Hunt code** `{Species}-{Sex}-{GMU}-{Season}-{Method}` e.g. `D-M-001-O1-A`. Species `D|E|P`; Sex `M|F|E`; Method `A|M|R`.
- **Whitetail sub-table** embedded in the deer section → tag rows `species_group="whitetail"` vs `"mule_deer"`.
- Anomalies: `See Unit N` cross-reference rows; `■` footnote rows; resident/nonresident separate elk-archery pages; season-date-in-header (elk archery); map pages (zero tables / garbled text); youth-hunt pages 27–29 (skip with INFO unless T1 shows they belong to V1 tables).
- List A/B/C / `OTC` values carried verbatim (feed S06.8 draw_spec).
- **S06.2 NOT triggered** — M1 `lib/pdf.py` primitives suffice; ZERO `ingestion/ingestion/lib/` edits.
