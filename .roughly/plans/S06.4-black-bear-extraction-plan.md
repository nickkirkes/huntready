# Implementation Plan: S06.4 — CPW Black Bear extraction (Colorado)

Plan-format-version: 1

## Summary

Ship a new self-contained extractor `ingestion/states/colorado/extract_black_bear.py` that parses the CPW
Big Game brochure's **bear section (PDF pages 72–77)** into a deterministic JSON artifact, plus the MT-style
**three-pass correction-merge** machinery as the spec-mandated inert-confirmation pathway for the 2-page
correction PDF. Downstream S06.6 / S06.7 / S06.9 consume the artifact.

### Structural decisions (resolved at discovery + live-PDF probe — do NOT re-litigate)

1. **Separate sibling module** `extract_black_bear.py` (NOT an extension of the merged `extract_big_game.py`).
   Rationale: S06.3's `big-game-2026.json` SHA is pinned; bear needs its own 3-pass correction `main()`;
   per-state extractors are self-contained units (verified: MT `extract_black_bear.py` and CO
   `extract_big_game.py` import only `ingestion.lib.pdf`, never a sibling extractor).
2. **Self-contained helpers, reimplemented inline** (ADR-005 / per-state-extractor convention). The bear
   extractor mirrors `extract_big_game.py`'s parsing patterns (`_parse_hunt_code`, `_normalize_cell`,
   `_block_slices`, `_parse_table_block`, `_assign_row_confidence`, `_parse_season_window`,
   `_parse_header_date`, R14 `_undouble_text`) but does **not import** them. Bear is simpler (one species,
   no whitetail/season-choice fan-out), so the reimplementation is bounded.
3. **MUST use `ingestion.lib.pdf.write_extraction_artifact(records, path)`** for ALL artifact writes — NOT
   MT's legacy `_write_deterministic_json` (`indent=2`, which blew past cubic's 50k-line cap). This forces a
   **list-of-records** artifact shape (`Sequence[object]`), matching `big-game-2026.json`, NOT MT's single
   top-level dict. Closure/reporting/statewide candidates become their own record lists (see T3/T4 decision).
4. **Correction is inert for bear, NOT bypassed.** Live-PDF probe confirmed the 2-page correction PDF carries
   **moose (page 1) + elk muzzleloader (page 2)** content — NOT "moose-only" as the S06.1 forward-note
   claimed. **Neither touches bear.** The extractor still parses the full correction and produces an explicit
   bear-targeted `operations=[]` artifact + a logged INFO line; it does not return early or skip the parse.
   (The moose+elk-not-moose-only discrepancy is a flag-to-PM finding — recorded in the closure note, not acted
   on; a possible latent gap for the already-closed S06.3 elk extraction is PM territory, out of S06.4 scope.)

### Bear section map (from live-PDF probe, 1-based PDF pages; content-page offset = +10)

| PDF page | Content | Tables |
|----------|---------|--------|
| 72 | Bear intro | "Bear Season Dates" summary (2-col) + "List A, B & C" explanation prose |
| 73 | Mandatory Inspections prose + **Archery — Limited** | dual side-by-side blocks `Unit \| Valid GMUs \| Hunt Code \| List` |
| 74 | **Muzzleloader — Limited** + **Rifle — Limited** (start) | muzz: 4-col blocks; rifle: 5-col `Unit \| Valid GMUs \| Dates \| Hunt Code \| List` |
| 75 | **Rifle — Limited** (continued) | 5-col blocks |
| 76 | **Add-On OTC** (Archery / Muzzleloader / Rifle) | `Valid GMUs \| Hunt Code \| List` (no Unit col) |
| 77 | standalone **OTC** Rifle / Rifle Private-Land / Rifle Plains | `Valid GMUs \| Dates \| Hunt Code \| List` |

Hunt-code grammar: `B-{sex}-{gmu}-{season}-{method}` e.g. `B-E-050-O1-M`. Species letter **`B`**; sex usually
`E` (either); method `A`/`M`/`R`. Season dates come from the **method-banner header** for archery/muzzleloader
("Season dates: Sept. 2–30 — Sex: Either") and **per-row Dates column** for rifle. `license_tag.kind` maps:
limited-draw tables → `limited_draw`; add-on + standalone OTC tables → `over_the_counter`. Page running-headers
are scrambled rotated sidebars ("HuBnet aPrlan") — ignore; the actual hunt codes extract clean.
