# Implementation Plan: S06.6 — Colorado `regulation_record` ingestion

Plan-format-version: 1

## Summary

First E06 DB-write story. New loader `ingestion/states/colorado/load_regulation_records.py`,
structural analog of MT's S03.6 `load_regulation_records.py` but **simpler** (no legal-descriptions
artifact, no fan-out dict, no statewide anchors in V1, no jurisdiction_binding writes). Reads the two
committed CO extraction artifacts (`big-game-2026.json` + `black-bear-2026.json`) and writes
`regulation_record` rows in one atomic transaction (build → guards pre-`db.connect()` → upsert loop → commit).
Plus 50+ tests in `ingestion/tests/test_load_co_regulation_records.py`.

**Group A only** (loader code + mocked/dry-run tests + quality gates) — the live DB write is Group-B
operator-pending (depends on outstanding E05 operator geometry writes). This build targets the build/dry-run
path; no live DB run.

## Load-bearing facts established by discovery (do not re-derive)

- **Big-game artifact** (`extracted/big-game-2026.json`): a JSON **array** of 736 section dicts (one record per
  line, loads via `json.load`). Species are **pre-separated** by the extractor: `mule_deer` (307), `whitetail` (34),
  `elk` (260), `pronghorn` (135). **No `deer` label → NO fan-out dict** (divergence from MT). Single `source_id`:
  `"co-cpw-big-game-2026-brochure"`. **Zero** `NOTE:` lines in `verbatim_text`. **Zero** `STATEWIDE` gmu_codes.
  **One** section with blank `gmu_code` (`""`) — index 415, elk archery, confidence `low`, the documented S06.3.1
  heading-absorption residual (its GMU is 020, which already has 3 proper non-blank elk sections).
- **Sections are per (gmu_code, species_group, method_group).** `regulation_record` is the **anchor entity keyed by
  `(state, jurisdiction_code, species_group, license_year)`** — it carries NO method_group. So the loader **collapses**
  all sections sharing a `(gmu_code, species_group)` into **one** `regulation_record`. Method/season/license detail
  decomposes to child entities in S06.7. Collapsed counts: **352** big-game (`(gmu,species)` pairs, non-blank) + **46**
  bear (unique gmu) = **398** records. (The 906 section count would be wrong — collapse is the intended design; matches
  the epic's "~300–500" estimate.) 207 big-game pairs span >1 method-group section (max 4); 35 bear gmus span >1 section.
- **Bear artifact** (`extracted/black-bear-2026.json`): a flat JSON **list** of 173 dicts with a `record_type`
  discriminator: 170 `section` (species_group `"black_bear"` → DB **`"bear"`**), 2 `statewide_rule`, 1
  `reporting_obligation`. **No top-level `sources`/`rows`/`statewide_rules` keys** (divergence from MT — do NOT port MT's
  `bear_artifact.get("sources")` dict-lookup pattern). Sections share the same single `source_id`
  `"co-cpw-big-game-2026-brochure"`. Section dicts have NO `source_publication_date` field (no per-section provenance
  cross-check). Zero NOTE lines. The 1 `reporting_obligation` record is S06.9 territory (ignored here).
- **The 2 `statewide_rule` records** have `rule_hint` ∈ {`"season_dates_summary"`, `"list_abc_explanation"`} — both
  **informational summaries, NOT pre-purchase coursework / Bear ID Test prerequisites**. Per the epic criterion
  (write `CO-STATEWIDE-bear` only for "statewide bear coursework / Bear ID Test / pre-purchase prerequisites analog of
  MT's Bear ID Test"), neither triggers a statewide anchor. **0 statewide anchors written in CO V1.** These two hints
  are the **known-informational allowlist**; any *other* `rule_hint` (or any big-game `STATEWIDE` gmu_code) is an
  undeclared statewide candidate that **fails loud** (flag-and-discuss per AC #549).
- **Geometry id format** (`load_gmus.py:191`): `CO-GMU-{gmuid}-geom` where `gmuid = str(int(props["GMUID"]))`
  (no leading zeros). Artifact `gmu_code` is zero-padded (`"001"`, `"020"`, `"082"`). To map: `int(gmu_code)` → e.g.
  `001 → 1`, `020 → 20`. CO statewide geometry id is `CO-STATEWIDE-geom` (S05.0).
- **`regulation_record` has no `verbatim_rule` column** (Q15/ADR-018 §3). Section `verbatim_text` is NOT written.
  Only the anchor (source + confidence + schema_version) + `additional_rules` (NOTE lines, empty for CO).
- **State constant:** the 4 existing CO loaders use `CO_STATE_CODE = "US-CO"`; the S05.6 scaffold uses
  `_STATE: Final[str] = "US-CO"`. S06.0/D3 picked `_STATE` (rename of the 4 loaders bundled with S06.10). **S06.6 uses
  `_STATE: Final[str] = "US-CO"`** (matches MT precedent + scaffold; do not add a 6th `CO_STATE_CODE` site) with a brief
  naming-note comment mirroring `colorado/load_jurisdiction_bindings.py:36`.
- **No new `db.py` helpers.** Reuse `db.upsert_regulation_record(conn, record)` (db.py:331) and `db.connect()` as-is.
  No bindings, no legal_description in S06.6.

## Design decisions (locked in this plan)

1. **`jurisdiction_code` format = `CO-GMU-{int(gmu_code)}`** (e.g. `"CO-GMU-1"`, `"CO-GMU-20"`, `"CO-GMU-82"`). The
   `regulation_record` PK already carries `species_group` separately, so the jurisdiction_code only identifies the GMU
   (unlike MT's `MT-HD-deer-elk-lion-{n}` which shares a code across species). S06.10 maps `jurisdiction_code → geometry_id`
   by appending `-geom` → `CO-GMU-{int}-geom` (matches `load_gmus.py`). Implemented as a pure helper `_co_gmu_jurisdiction_code(gmu_code: str) -> str`.
2. **Collapse by `(gmu_code, species_group)`** for big-game and by `gmu_code` (species fixed `"bear"`) for bear. One
   `regulation_record` per group. Within a group: `confidence = pdf.min_tier([ConfidenceTier(r["extraction_confidence"]) for every row in every section in the group])` (most-uncertain-wins; NOT bare `min()` on strings); `additional_rules` = NOTE lines extracted across all sections' `verbatim_text` in deterministic order (empty for CO); `page_reference` = bound from the group's **representative section** = the section with the smallest `(page_num_1based, method_group)` tuple (deterministic), via `citation.model_copy(update={"page_reference": ...})`.
3. **Blank `gmu_code` sections are skipped with a `logger.warning`** (count + log them; exclude from records). No
   jurisdiction_code is derivable without a GMU, and the one occurrence (GMU 020 elk) is already covered by 3 proper
   non-blank elk sections — **no anchor is lost**. Never fabricate a GMU.
4. **0 statewide anchors** in CO V1; main() writes only `regulation_record` rows. The `_JURISDICTION_BINDING_ID_FORMAT`
   constant is defined (byte-identical to MT) purely for S06.10 to import; no bindings built/written here.
5. **`document_type` guard:** after loading the citation, assert `citation.document_type in {"annual_regulations", "correction"}`
   else `raise ValueError(...)` (ADR-019 §Decision item 5 / AC #548). CO's brochure citation is `annual_regulations` → passes.
6. **Count guard band:** baseline `_SPEC_ESTIMATE_TOTAL = 398`, ±30% via `int()` truncation → **[278, 517]**. Fires
   pre-`db.connect()`. Document the band in the closure note.
7. **Determinism:** group iteration order is sorted (sort group keys; for big-game sort by `(int(gmu_code), species_group)`;
   for bear sort by `int(gmu_code)`). Record list order is deterministic and stable across runs.
