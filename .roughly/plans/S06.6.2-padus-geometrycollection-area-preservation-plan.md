# Implementation Plan: S06.6.2 — PAD-US GeometryCollection area-preservation handling

Plan-format-version: 1

## Summary

Second consecutive PAD-US 4.1 republish-drift carve-out. Discovery + Phase A live characterization (already complete) established:

- **9 of 10 V1 zones pass** the current strict path. **Only Rocky Mountain NP raises** — its ring has a self-intersection that `make_valid` repairs into `GeometryCollection([MultiPolygon, 19m-LineString])`. `parsed.area` is **inflated** by the self-intersection double-count; `recovered.area` is the *correct* area (matches PAD-US `GIS_Acres` to 0.04%). The "discrepancy" is the artifact being removed, not real data loss.
- **Max relative area-loss across all GC-branch zones: 0.0676% (RMNP only). No zone > 0.1%.**
- **Phase B decision = Outcome (a): relax `rel_tol` `1e-6` → `1e-3`** in `geojson_to_multipolygon_wkt`. Clears RMNP (0.0676%) with margin; the existing raise-test (overlapping polys, −12.5%) still raises by 125×. Outcome (b) NOT triggered (no re-source, no ADR, no PM flag-and-discuss). Outcome (c) buffer(0) rejected empirically (changes area for non-GC zones, e.g. Dinosaur −0.576%).

The fix is one numeric literal + an inline rationale comment. The rest is tests, the Phase D manifest/metadata fixtures (per **Option A** — features payload stays gitignored), and the closure memo.

### T2: Phase C — unit tests locking the epsilon boundary (~6 min)
**Files:** ingestion/tests/test_arcgis.py
**Action:** Add two boundary tests to `TestGeojsonToMultipolygonWkt` (the class starting at line 74) that lock the new `rel_tol=1e-3` threshold on both sides. Mirror the existing `test_geometry_collection_*` tests' style (inline GeoJSON `GeometryCollection` of overlapping polygons; controlled overlap sets the discrepancy).
**Details:**
- Mechanism for a controlled discrepancy: a GeometryCollection of two unit squares (each 1.0 × 1.0 = 1 deg², `parsed.area` = 2.0). An overlap of area `O` makes `recovered.area = 2.0 − O`, so relative discrepancy = `−O / 2.0`.
- **Test A — passes just under threshold (`~ −5e-4`, recovers with WARNING):** overlap `O = 0.001 deg²` → discrepancy `−5e-4`, comfortably between `1e-6` and `1e-3`.
  - poly1 ring: `[[0,0],[1,0],[1,1],[0,1],[0,0]]`
  - poly2 ring: `[[0.999,0],[1.999,0],[1.999,1],[0.999,1],[0.999,0]]` (overlaps poly1 in the strip x∈[0.999,1], y∈[0,1] = 0.001 deg²)
  - Name e.g. `test_geometry_collection_recovers_when_area_loss_within_1e3_tolerance`.
  - Assert: returns `MULTIPOLYGON` WKT; a WARNING with `OBJECTID=<n>` and `"non-polygonal artifacts"` is emitted (use `caplog.at_level("WARNING", logger="ingestion.lib.arcgis")` like the existing recover-test). Use a distinct OBJECTID (e.g. 6062).
  - Add a comment noting these coordinates are a synthetic stand-in for the RMNP ring-self-intersection case (real RMNP discrepancy −0.0676%; this test uses −0.05% to sit safely inside the band) — keep the comment factual and tied to the Phase A finding.
- **Test B — raises just over threshold (`~ −2e-3`, genuine loss):** overlap `O = 0.004 deg²` → discrepancy `−2e-3`, above `1e-3`.
  - poly1 ring: `[[0,0],[1,0],[1,1],[0,1],[0,0]]`
  - poly2 ring: `[[0.996,0],[1.996,0],[1.996,1],[0.996,1],[0.996,0]]` (overlap strip width 0.004 → 0.004 deg²)
  - Name e.g. `test_geometry_collection_raises_when_area_loss_exceeds_1e3_tolerance`.
  - Assert: `pytest.raises(ArcGISError, match=r"OBJECTID=<n>.*do not preserve area")`. Use a distinct OBJECTID (e.g. 6063).
- Before finalizing, sanity-check the arithmetic of both overlaps by reasoning (or a scratch shapely calc) so Test A truly lands in (1e-6, 1e-3) and Test B truly lands above 1e-3. If a chosen overlap doesn't produce the intended side of the boundary, adjust the strip width and update the comment with the actual discrepancy.
- Do NOT modify or delete the existing `test_geometry_collection_recovers_when_polygon_area_preserved` (L87) or `test_geometry_collection_raises_when_area_not_preserved` (L114) — they are the MT-regression guards and must stay green.
**Verify:** `cd ingestion && .venv/bin/pytest tests/test_arcgis.py::TestGeojsonToMultipolygonWkt -q`
