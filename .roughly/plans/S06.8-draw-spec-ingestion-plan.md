# Implementation Plan: S06.8 — `draw_spec` ingestion (Colorado)

Plan-format-version: 1

## Summary

New loader `ingestion/states/colorado/load_draw_specs.py` (state-agnostic-clean, canonical **three-phase** shape: build → guards pre-`db.connect()` → write). Reads the committed `draw-mechanics-2026.json` (S06.8.0 artifact) + the CO big-game/black-bear extraction artifacts, writes one `draw_spec` row per unique limited-draw CO hunt code, and backfills `license_tag.draw_spec_key` (per ADR-012). Plus `ingestion/tests/test_load_co_draw_specs.py` (80+ tests).

**Deviation from MT analog:** MT's `load_draw_specs.py` has a Phase-1 license_tag *re-upsert* (to correct OTC mis-classification from its `apply_by` heuristic) before upserting draw_specs and backfilling. CO needs NO such re-upsert — CO's kind classification was already correct at S06.7 from `list_value`/section `license_kind`. So the write phase here is just: upsert draw_specs → backfill `draw_spec_key` → single commit. The module remains the canonical three-phase (build / guards / write) per AC #969.

## Discovery-confirmed facts (load-bearing)

- **draw_spec count = 1917** unique limited-draw hunt codes (113 hybrid / 1804 non-hybrid). `_EXPECTED_DRAW_SPEC_COUNT = 1917`; ±30% band `[1342, 2492]`.
- **Limited-draw universe** = big-game rows where `_co_big_game_license_kind(row["list_value"]) == "limited_draw"` (A/C/None) + bear rows in sections where `_co_bear_license_kind(section["license_kind"]) == "limited_draw"`. OTC rows get NO draw_spec. (Bear kind is **section-level** per S06.7 `load_seasons_and_licenses.py:1398` — match it so backfill ids align.)
- **Hybrid set** = `{r["hunt_code"] for r in mechanics if r["record_type"]=="hybrid_code"}` (116 codes). A limited-draw hunt code in the hybrid set → 2-pool + 0.20 NR cap; else → 1-pool + 0.25 NR cap.
- **No Q17 quota conflicts**: zero limited-draw rows carry a non-null `quota` → `quota=None` on every draw_spec; the Q17 STOP guard never fires on 2026 data (still implemented, defensive).
- **Species letters present**: A (pronghorn), B (bear), D (deer), E (elk). All 4 have a `point_only_code` in the artifact → `purchase_only_code` set per species letter; never null for V1.
- **Artifact has no per-code `draw_phase`/`successor`/quota** → all rows `draw_phase="primary"`, `successor_hunt_code_key=None`, `quota=None`, `application_deadline=important_dates.primary_draw_deadline` (2026-04-07), `choices=ChoiceConfig(count=4, points_used_in_choices=[1])`.
- **Known upstream residual (handle as WARNING + closure-note flag, NOT a blocker — out of S06.8 loader scope per ADR-022):** 3 ` +`-suffixed bear codes at GMU 851 (`B-E-851-O1-M +`, `B-E-851-O2-R +`, `B-E-851-O5-R +`) are a `extract_black_bear.py` row-fusion residual; they are loaded faithfully as-is (corrupted hunt_code, non-hybrid). They relate to 3 orphan hybrid codes (`B-E-851-O1-R/O2-R/O5-R` — in the hybrid set but matching no clean limited-draw code): the ` +` corruption directly explains the `O2-R`/`O5-R` orphans; `O1-R` (rifle) is a genuinely-absent variant (per-unit bear at GMU 851 has `O1-A` archery + `O1-M +` muzzleloader but no clean `O1-R`). 1 empty big-game hunt_code (GMU 020, the known S06.6 skip) is skipped-with-WARNING. Surface all via WARNING + new Known-Issue candidate in the closure note. The orphan-detection logic (`orphans = hybrid_set - set(draw_specs.keys())`) is sound regardless of the per-code cause.
