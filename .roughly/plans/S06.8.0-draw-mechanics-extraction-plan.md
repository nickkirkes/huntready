## Artifact shape (flat record list тАФ `write_extraction_artifact` writes one record per line w/ `sort_keys=True`)

A LIST of typed records discriminated by `record_type` (mirrors the bear artifact's `record_type` pattern S06.6 already consumes). Records, deterministically sorted (by `record_type`, then `species`, then `hunt_code`):

- `{"record_type": "hybrid_code", "species": "deer|elk|pronghorn|bear", "hunt_code": "<code without *>", "low_availability": true|false}` тАФ one per hybrid code
- `{"record_type": "point_only_code", "species": "deer|elk|pronghorn|bear", "hunt_code": "D-P-999-99-P"}` тАФ one per species (4)

All records also carry `"source_id": "co-cpw-big-game-2026-brochure"` and `"extracted_at": <manifest timestamp>` for provenance parity with the other CO artifacts.

### T2: Hybrid hunt-code extraction (deer/elk/pronghorn tables + bear bbox crop) (~5 min)
**Files:** ingestion/states/colorado/extract_draw_mechanics.py
**Depends on:** T1
**Action:** Implement `_extract_hybrid_codes(pdf) -> list[dict]` returning one `hybrid_code` record per code.
**Details:**
- Access page 29 (1-indexed) via the lib's page access: `iter_pages` yields `(page_num_1based, page)` TUPLES, so unpack тАФ `for _, page in iter_pages(pdf, 29, 29):` (match `extract_big_game.py:2627`). Or `pdf.pages[28]` for a bare page object.
- **Deer/Elk/Pronghorn:** call `extract_tables(page)`. Walk the returned `TableMatch` list. A species sub-table is identified by a header/cell matching `re.compile(r"(Deer|Elk|Pronghorn)\s*[тАФтАУ-]\s*Hybrid Draw Hunt Codes", re.I)`. pdfplumber splits each species block into a header table plus 1тАУ2 continuation tables with no header (deer = 3 tables, elk = 3 tables, pronghorn = 1 wide table per the probe). Strategy: once a species header is found, attribute all subsequent header-less tables to that species until the next species header. Collect every cell whose stripped text (after removing a leading `*`) matches `_HUNT_CODE_RE`.
- **Bear:** `extract_text(page, bbox=(330.0, 415.0, 590.0, 435.0))` then `_HUNT_CODE_EMBEDDED_RE.findall` on the result (3 codes). Verify the crop returns the `"Bear тАФ Hybrid Draw Hunt Codes"` header; if the bbox drifts, find the bear header's bbox via `page.search`/`find_section` and crop from there тАФ do NOT hard-fail on the literal coordinates without first trying an anchor-based fallback, but DO fail loud if no bear codes are recovered at all.
- For each code: `low_availability = code_raw.startswith("*")`; `hunt_code = code_raw.lstrip("*")`; validate `_HUNT_CODE_RE.match(hunt_code)` (raise `ColoradoDrawMechanicsError` naming the offending string if not).
- **Fail loud** if any of the 4 species yields zero hybrid codes (missing anchor / layout drift) тАФ raise with the species name + page. Do NOT silently emit an empty set (S05.4 pitfall).
- Sanity band: total hybrid codes across species must be within `_HYBRID_COUNT_BAND = (80, 160)` (probe found 116); raise if outside. Per-species non-zero is already enforced above.
- Dedup within species (a code listed twice тЖТ one record), preserving deterministic order.
- Return records sorted by `(species, hunt_code)`.
**Verify:** Add a tiny `if __name__` smoke or run `.venv/bin/python -c "from states.colorado import extract_draw_mechanics as m; import ingestion..."` тАФ but primary verify: `cd ingestion && .venv/bin/ruff check states/colorado/extract_draw_mechanics.py && .venv/bin/mypy states/colorado/extract_draw_mechanics.py`
