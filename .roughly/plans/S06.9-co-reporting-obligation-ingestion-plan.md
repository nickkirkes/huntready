# Implementation Plan: S06.9 — Colorado `reporting_obligation` ingestion

Plan-format-version: 1

## Summary

Build `ingestion/states/colorado/load_reporting_obligations.py` — the CO analog of MT's
`load_reporting_obligations.py`. It reads the committed CO bear extraction artifact
(`states/colorado/extracted/black-bear-2026.json`, a **flat list** with `record_type`
discriminators), filters to the **single** `record_type == "reporting_obligation"` record,
and writes **exactly 1 `reporting_obligation` row** in one atomic transaction.

**Key structural divergences from MT (load-bearing):**
1. **CO writes ONLY the `reporting_obligation` entity row. NO `regulation_reporting` link
   rows** — those are S06.10's job (epic L1017/L1024/L1094). So NO `_build_regulation_reporting_links`,
   NO `db.write_regulation_reporting` call, NO `RegulationReporting` import.
2. **CO bear artifact is a flat list**, not a dict with `sources`/`rows`/`reporting_obligations`
   keys. Filter `record_type == "reporting_obligation"` from the list (mirror
   `load_regulation_records.py`'s `_KNOWN_BEAR_RECORD_TYPES` fail-loud-then-filter pattern).
3. **Citation comes from `sources.yaml`** via a CO-local `_load_citation_from_sources_yaml`,
   not from an inline artifact `sources` list. `page_reference` is built from the artifact
   record's `page_reference` dict via `pdf.page_reference_to_str()`.
4. **One dispatch entry**: `("STATEWIDE", "mandatory_inspection")`.

**The single row written:**
- `id` = `co-bear-mandatory-check-5day-statewide`
- `kind` = `"mandatory_check"` (the closest of the 6 enum values for `kind_hint="mandatory_inspection"`
  — a post-harvest in-person presentation/sealing duty; NOT a silent enum widen)
- `deadline` = `"5 working days"`; `deadline_hours` = `120` (5×24 working-day proxy; representable
  in the slug encoding as a multiple of 24)
- `submission_method` = `"agency_office"` (present to "any CPW office")
- `submission_url` = `None`; `submission_phone` = `None`
- `applies_to_regions` = `None` (STATEWIDE); `what_to_present` = `["bear head", "hide"]`
- `verbatim_rule` = the artifact's verbatim_rule value verbatim (currently the heading
  "Mandatory Bear Inspections & Seals" — see Blast Radius note; faithful artifact consumption
  per Path A, the extraction fix is a separate flagged carve-out)
- `source` = `co-cpw-big-game-2026-brochure` (from `sources.yaml`; `document_type="annual_regulations"`)
  with `page_reference="co-cpw-big-game-2026-brochure-2026-03-04.pdf:p73"`

**Q18 = option (c) (S06.0/D1): 0 typed CWD `reporting_obligation` rows.** CWD text stays in
`regulation_record.additional_rules` (S06.6). The `cwd_sample` enum value stays defined-but-unused.

**ADR-020 drift guard:** `assert_dispatch_dict_drift_free(...)` at **module top-level** (after the
dispatch dict + derivation fn, before any other function), `helper_name="_REPORTING_ROW_SPEC"`,
`id_field="id_suffix"`.

**Count guard:** exact `(1, 1)` band (S03.6.1 single-row precedent). The literal "±30% band" AC
wording yields `(0, 1)` for an expected count of 1, which would silently pass a zero-write run —
so an exact-match band is the disciplined single-row analog. Documented deviation; reviewer should
expect it.

### T3: `_build_reporting_obligations` — flat-list read, dispatch, fail-loud guards (~5 min)
**Files:** ingestion/states/colorado/load_reporting_obligations.py
**Depends on:** T2
**Action:** Add `_build_reporting_obligations(bear_artifact: list[dict]) -> list[ReportingObligation]`
that reads the flat-list artifact, validates/filters `record_type`, dispatches on
`(region_scope, kind_hint)`, builds the citation + page_reference, constructs the `ReportingObligation`,
and guards against duplicate ids.
**Details:**
- Signature takes the parsed flat list (`list[dict]`), matching `load_regulation_records.py`'s
  `_build_co_bear_records(bear_artifact: list[dict], ...)` style. (mypy: use the same
  `# type: ignore[type-arg]` annotation convention the CO regulation_records loader uses for the
  bear flat list if needed.)
- **Record-type validation then filter** (mirror `load_regulation_records.py`): iterate the list with
  `enumerate`; fail loud (RuntimeError naming the index + actual type) on a non-dict entry; fail loud
  on a missing `record_type` key; fail loud if `record_type` not in `_KNOWN_BEAR_RECORD_TYPES` (catches
  misspellings); collect entries where `record_type == "reporting_obligation"`.
- Load the base citation ONCE: `citation_base = _load_citation_from_sources_yaml(_BIG_GAME_CITATION_ID)`
  then `_assert_document_type_allowed(citation_base)`. (The page_reference is per-row, so build the
  final per-row citation inside the loop with the row's page_reference — see below.)
- For each filtered reporting_obligation entry (`enumerate` for diagnostics):
  - Extract required keys with a try/except KeyError → RuntimeError naming the index + missing key +
    present keys: `region_scope`, `kind_hint`, `source_id`, `page_reference`, `verbatim_rule`
    (mirror MT `load_reporting_obligations.py:433-445`).
  - `spec = _REPORTING_ROW_SPEC.get((region_scope, kind_hint))`; if `None`, RuntimeError enumerating
    the valid dispatch keys (this is the **fail-loud gate for an unknown `kind_hint`** — satisfies the
    AC's "do NOT silently widen the kind enum").
  - Fail loud if `source_id != _BIG_GAME_CITATION_ID` (RuntimeError naming the bad id + the expected id).
  - `page_ref_str = pdf.page_reference_to_str(page_reference)` (the artifact's page_reference dict has
    `pdf_filename` + `page_num_1based` — matches the `PageReference` TypedDict `page_reference_to_str`
    reads).
  - Build the per-row citation: `citation = _load_citation_from_sources_yaml(_BIG_GAME_CITATION_ID, page_reference=page_ref_str)`.
  - Construct `ReportingObligation(id=f"{_ID_PREFIX}-{spec['id_suffix']}", kind=spec["kind"],
    deadline=spec["deadline"], deadline_hours=spec["deadline_hours"], submission_method=spec["submission_method"],
    submission_url=spec["submission_url"], submission_phone=spec["submission_phone"],
    applies_to_regions=spec["applies_to_regions"], what_to_present=spec["what_to_present"],
    verbatim_rule=verbatim_rule, source=citation)`. Pydantic `frozen/extra=forbid/non-empty verbatim_rule`
    validates here.
  - Append to `result`.
- **Duplicate-id guard** (port MT `load_reporting_obligations.py:517-533`): scan `result` for duplicate
  `.id` values; RuntimeError if any (an artifact with 2+ entries collapsing to the same dispatch key
  would otherwise be silently overwritten by `ON CONFLICT (id) DO UPDATE`). Do NOT add an in-builder
  exact-count check — the count guard in `main()` is the canonical count authority (MT's removed-the-count-check
  comment lesson).
- Return `result`.
