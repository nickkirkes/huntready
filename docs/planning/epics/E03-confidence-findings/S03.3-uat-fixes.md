## Confirmed decisions

**Fix 2 (D2) — Region 7 / "Portions of HDs ..." emission strategy: option (a).**

Each "Portions of HDs A, B, C ..." sub-section emits **one section per listed HD**, with the same set of rows duplicated under each. License codes (e.g., `007-21`) provide the disambiguator. S03.6 dedups at ingestion via license_code matching. Rationale: simple slicer change; preserves geometric truth (the regulation does apply to those HDs); duplicate rows in the artifact are harmless because they normalize to a single regulation_record per (state, jurisdiction_code, species_group, license_year).

The HD-listing parse must accept all observed forms: `"Portions of HDs 700, 701, and 703 North of the Yellowstone River"`, `"Portions of HDs 701, 702, 703, 704, and 705 South of the Yellowstone River"`. Section-level metadata should preserve the directional qualifier (e.g., "North of the Yellowstone River") in `verbatim_text` so S03.10's binding generation can decide whether the qualifier matters.

## Fix directives

### Fix 1 (D1) — Per-row `season_windows` extraction

Replace the section-level first-observation-wins helper with per-row date-cell extraction. Each row's `season_windows[<key>].window` reads from that row's corresponding column cell, not the section's first-observed value.

- The "WARNING on divergence" path becomes obsolete; divergence is the data, not an anomaly. Remove the warning.
- `season_coverage[<key>]` semantics stay the same: `true` when the row has a non-`-` value in that column, `false` otherwise.
- `season_windows[<key>]` is only emitted when `season_coverage[<key>]` is `true` (preserves existing contract; downstream code checks coverage before reading window).
- `weapon_type_override` continues to be derived from the column header, not the date string.

**Test (must lock against regression to first-observation-wins):**
- `test_per_row_general_window_distinct_in_hd_124_deer` — HD 124 deer artifact must have row 1 General `Oct 24-Oct 30`, row 2 General `Oct 24-Nov 29`, row 3 General `Oct 31-Nov 29`, row 4 General `Oct 31-Nov 29`, row 5 General `Oct 24-Nov 29`. All five distinct (or at least: row 1 ≠ row 2 ≠ row 3, asserting the artifact preserves divergence).
- Existing tests that asserted section-level uniform windows must be inverted to assert per-row windows. If a test was named like `test_section_season_windows_first_observation_wins` or similar, it tests the wrong contract — rewrite it.

### Fix 2 (D2) — Region 7 / Portions sub-section handling

Add antelope-table section-delimiter detection for two patterns in addition to the existing `HD NNN - Name` delimiter:

1. **Region N header rows** (e.g., `Region 5`, `Region 6`, `Region 7`) — these are not section delimiters themselves but signal that the next non-HD-numbered delimiter is a Portions sub-section. Skip the row; do NOT absorb its content.
2. **`Portions of HDs A, B, C ... <directional qualifier>` rows** — parse the HD list (regex to extract the integers from "HDs 700, 701, and 703"). Emit one `DeaSectionExtraction` per listed HD with:
   - `hd_number`: each HD as a string (`"700"`, `"701"`, `"703"`)
   - `hd_name`: empty string `""` (the row text is not a per-HD name; it's a multi-HD descriptor)
   - `species_group`: `"antelope"`
   - `verbatim_text`: full text of the Portions sub-section, INCLUDING the directional qualifier and the delimiter row itself, so downstream stories have access to the geometric scope language
   - `rows`: same set of rows duplicated under each section (license_code disambiguates at ingestion)
   - `page_reference`: page of the first row in the Portions sub-section

**Test:**
- `test_region_7_portions_north_yellowstone_emits_three_sections` — input PDF has "Portions of HDs 700, 701, and 703 North of the Yellowstone River" with rows {`007-21`, `007-31`, `900-20`}. Artifact must contain three antelope sections keyed `hd_number=700`, `hd_number=701`, `hd_number=703` (after STATEWIDE 900-20 dedup). Each non-statewide row appears once per section.
- `test_hd_690_does_not_absorb_007_series_rows` — HD 690 must have exactly 2 rows (`690-20`, `690-30`) after the fix.
- `test_portions_section_preserves_directional_qualifier_in_verbatim_text` — `verbatim_text` for a Region 7 portions section must contain the substring "North of the Yellowstone River" (or equivalent for South).

**Open implementation question:** the directional qualifier might matter for S03.10 binding (you don't bind the regulation to all of HD 700, only to "the portion north of the Yellowstone River"). For S03.3 the contract is just "preserve the qualifier in verbatim_text"; the binding decision is S03.10's. Don't attempt to encode geometric scope in S03.3.
