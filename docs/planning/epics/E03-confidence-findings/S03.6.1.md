## 1. Artifact-shape decision

A new top-level field on `black-bear-2026-base.json` and `black-bear-2026.json` (the merged artifact) named `statewide_rules`, typed as `list[StatewideRuleCandidate]`. `corrections-2026-03-18.json` does NOT carry this field (the 2026-03-18 correction PDF is BMU-table-only — no page-2 prose touched).

`StatewideRuleCandidate` shape (TypedDict, declared alongside the existing six TypedDicts in `extract_black_bear.py`):

```python
class StatewideRuleCandidate(TypedDict):
    rule_hint: str                   # V1 value: "pre_purchase_prerequisite"
    verbatim_text: str               # post-cleanup-rule (single-space-collapsed) paragraph
    page_reference: PageReference    # pdf_filename + page_num_1based + bbox + extracted_at
    source_id: str                   # "mt-fwp-black-bear-2026-booklet"
    source_publication_date: str     # "2026-04-27"
    extraction_confidence: str       # ConfidenceTier value: "medium" for V1
```

**Rationale (forward-compat):** a `list[StatewideRuleCandidate]` accommodates additional pre-purchase prerequisites in M2 (e.g., hunter education completion, archery certificate, MOSS conservation license) without schema migration. The `rule_hint` is intentionally a plain `str` (not a `Literal`) so adapters can introduce new hints without an exhaustive enum maintenance burden across states; the loader side maps `rule_hint` to unstructured prose in `VerbatimRule.text` and does not branch on the value.

**Merge-pass impact:** none beyond a single pass-through line `statewide_rules=base["statewide_rules"]` in the `BlackBearMergedExtraction(...)` return of `_merge_with_corrections`. The Pass-2/3 correction algorithm operates exclusively on `rows`, `closures`, `reporting_obligations` and is not modified.

**Determinism:** SHA-256 of `black-bear-2026.json` is byte-identical across two consecutive extraction runs (verified 2026-05-22 with `shasum -a 256`: `a09aefdb845257c13a85ba6ed5c6e81191e6ab34e5ea16d2e59d7ef2b99a8fb8`).

**Cleanup rule (S03.3 docstring discipline):** the new `re.sub(r"\s+", " ", body)` applied to the captured Bear ID Test paragraph is documented in the module-level "Cleanup rules" docstring in `extract_black_bear.py`, with scope (page-2 right-column paragraph between the two `_BEAR_ID_*_RE` anchors only), rationale (extras-only whitespace collapse, ADR-008-safe), and test pointer (`TestExtractStatewideRules::test_whitespace_collapse_extras_only`).

## 2. `id` encoding — locked format

```python
_JURISDICTION_BINDING_ID_FORMAT = (
    "{state}-{jurisdiction_code}-{species_group}-{license_year}-{role}-{geometry_id}"
)
```

For V1 bear: `"US-MT-MT-STATEWIDE-bear-bear-2026-primary_unit-MT-STATEWIDE-geom"`.

Locked by `TestBuildStatewideBearBinding::test_id_encoding_is_deterministic` (asserts the exact string).

**Q19 risk discussion (slug-drift on UPSERT):** encodes only PK-equivalent fields (`state`, `jurisdiction_code`, `species_group`, `license_year`) plus `role` and `geometry_id`. The two mutable-on-re-ingest columns — `verbatim_rule` and `source` — are NOT encoded; they can update freely without producing a new `id`. The encoded fields are all identity fields that must not change between runs without representing a different binding (e.g., a re-roled binding produces a new row, not a silent UPSERT-mismatch). This contains the Q19 drift risk locally for jurisdiction_binding; the project-wide pre-M2 fix (derive-and-assert via generalized helper) is tracked in `docs/open-questions.md` Q19.

## 3. S03.10 carry-over implications

1. **Helper inheritance.** `db.upsert_jurisdiction_binding(conn, binding)` is introduced in `ingestion/ingestion/lib/db.py` by this story (mirrors S03.6's introduction of `update_legal_description` ahead of S03.8 reuse). S03.10 reuses this helper as-is — same signature, same fail-loud-on-rowcount-0 contract, same no-commit caller-controlled-transaction shape.

2. **Encoding contract.** S03.10 MUST use the exact `_JURISDICTION_BINDING_ID_FORMAT` defined by this story. When S03.10 derives `MT-STATEWIDE-bear → MT-STATEWIDE-geom` from its overlay-fanout logic, the encoded `id` must be byte-identical to the value already in the DB (`"US-MT-MT-STATEWIDE-bear-bear-2026-primary_unit-MT-STATEWIDE-geom"`), which the UPSERT-by-`id` then resolves as a no-op against the row this story wrote.

3. **Row-count baseline.** S03.10's `_JURISDICTION_BINDING_COUNT_GUARD` band starts from a baseline of 1 (this story's bear binding already present) plus its derived bindings. The story-level count band is exact-match `[1, 1]` for S03.6.1; S03.10 broadens this to cover HD-level + portion + restricted_area + CWD-zone + `MT-STATEWIDE-antelope → MT-STATEWIDE-geom` bindings.

4. **MT-STATEWIDE-antelope binding.** Verified 2026-05-22: `MT-STATEWIDE-antelope`'s `additional_rules` carries exactly **2 NOTE entries** (Boundary Change + New Hunting District) from `_extract_note_lines(section["verbatim_text"], …)` against the DEA `900-20` section. There is currently **no** `jurisdiction_binding` row for `MT-STATEWIDE-antelope → MT-STATEWIDE-geom` (the table contained zero rows pre-S03.6.1; S03.10 will derive that binding via overlay-fanout symmetrically and use the same `_JURISDICTION_BINDING_ID_FORMAT` to produce `"US-MT-MT-STATEWIDE-antelope-pronghorn-2026-primary_unit-MT-STATEWIDE-geom"`). The S03.6.1 bear binding intentionally lands earlier than S03.10's broader generation to keep the bear anchor self-contained at story close.

5. **FK ordering.** S03.6.1's atomic transaction inserts regulation_record rows before jurisdiction_binding rows (composite-FK requirement). S03.10 must preserve this ordering — its broader binding insertion must happen after all referenced regulation_records exist in the same transaction.

## 5. Q20 status (rule prefix convention)

No new question opened. The V1 decision is to carry the Bear ID Test paragraph as raw verbatim text in `VerbatimRule.text` (no NOTE: or REQUIREMENT: prefix added by the adapter), because:
- `VerbatimRule` has no `kind` field on the Pydantic model (`schema.py:81-94`) — "kind" cannot be expressed structurally.
- ADR-008 forbids paraphrase; a prefix added by the adapter is paraphrase.
- The prose is self-describing ("A free Black Bear Identification Test Certificate is required to obtain a license.") — a prefix adds no information.

If M2 hunter-ed prerequisite extraction shows a recurring pattern that benefits from a structured discriminator across states, Q20 can be opened at that time.
