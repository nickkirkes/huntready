## Per-record aggregation outcomes

The MIN-tier aggregation in `_build_dea_records` (via `pdf.ConfidenceTier.min_tier`)
surfaces two mixed-confidence patterns in V1 data:

- **Elk: 2 sections of 112 emit `high` confidence; 110 emit `medium`.** Each
  section's confidence reflects the MIN over its per-row extraction confidences.
  The two `high` elk sections are HDs where every license row extracted cleanly
  (no medium-confidence rows). Worth a spot-check in S03.12 UAT to confirm the
  extractor is consistently strict.
- **Pronghorn: 30 of 31 sections emit `high`; 1 emits `medium`.** The single
  `medium` pronghorn section is the candidate for UAT spot-check — identify
  which HD (or STATEWIDE) and which row dragged the section to MEDIUM.

No HIGH→LOW or MEDIUM→LOW demotions surfaced; the lowest tier observed is
`medium`. No bear demotions occur at this stage either — S03.4 already demoted
correction-touched rows HIGH→MEDIUM per ADR-017 §4; the MIN aggregation here
passes them through unchanged.

The `('high', 'low') → 'low'` lexicographic-min trap case is locked by the
unit test `test_confidence_min_tier_trap_case_high_low_returns_low` in
`tests/test_load_regulation_records.py`.

## Confidence-assignment inputs (F12 per-record traces)

This section provides per-record provenance traces for `regulation_record.confidence`
as materialized by the MIN-aggregation in `_build_dea_records`
(`load_regulation_records.py:306-310`) and the direct pass-through in
`_build_bear_records` (`load_regulation_records.py:405-406`). These are F12
inputs for the S03.11 confidence-calibration audit.

### Deer fan-out symmetry guarantee

The MIN over per-row extraction tiers is computed **once per DEA section** at
`load_regulation_records.py:306-310`, before the fan-out loop at `:315`. Both
`mule_deer` and `whitetail` records emitted from the same section share
**identical** `confidence` values. There is no separate MIN call per fan-out
target — `confidence` is captured before the `for target_species_group in
target_species:` loop and reused unchanged for both records. This is a
structural guarantee: any change that adds a per-species re-computation would
need a test to lock fan-out parity.

### MT-STATEWIDE-antelope path

The `900-20` STATEWIDE antelope row enters `dea-2026.json` via
`is_statewide_overlay=True` in `extract_dea.py`, emitted as a 1-row section
with `hd_number="STATEWIDE"`. The MIN over a 1-element list is identity: the
single row's `extraction_confidence="medium"` passes through unchanged. The
resulting `MT-STATEWIDE-antelope` regulation_record therefore carries
`confidence="medium"`. The artifact path is:
`dea-2026.json → section[species_group="antelope", hd_number="STATEWIDE"] → rows[0].extraction_confidence="medium"`.

### Section-level MIN-aggregation worked example — HD 124 deer

HD 124 is a representative deer section with a mixed-tier row set, illustrating
how the section-level MIN collapses to record confidence (this is distinct from
S03.3.md's row-level traces, which document per-row extraction decisions within
the PDF):

- Section: `hd_number="124"`, `species_group="deer"`, page 48 of
  `mt-fwp-dea-2026-booklet-2026-04-27.pdf`.
- Per-row extraction tiers (5 rows): `[MEDIUM, MEDIUM, MEDIUM, MEDIUM, HIGH]`
- `pdf.min_tier([MEDIUM, MEDIUM, MEDIUM, MEDIUM, HIGH])` → `MEDIUM`
  (lowest-wins; HIGH does not promote; the lexicographic-min trap case is locked
  by `test_confidence_min_tier_trap_case_high_low_returns_low` in
  `tests/test_load_regulation_records.py`).
- Resulting `regulation_record.confidence`: `medium` for both the
  `MT-HD-deer-elk-lion-124 / mule_deer` and
  `MT-HD-deer-elk-lion-124 / whitetail` records (fan-out symmetry: same MIN
  applied to both; see §above).

**Column key:**
- `entity_id` — composite-PK shorthand: `{jurisdiction_code} / {species_group}`. All rows are V1 `license_year=2026`, `state="US-MT"` (implicit).
- `confidence` — the final `regulation_record.confidence` value written to the DB, after MIN-aggregation (DEA) or direct pass-through (bear).
- `source_format` — how the source data was physically encoded in the PDF.
- `extraction_op` — which extractor module assigned per-row `extraction_confidence`.
- `transformation_steps` — logic applied between per-row tier and final record confidence.
- `correction_touched` — whether ADR-017 §4 single-step demotion (HIGH→MEDIUM) was applied by S03.4 before this loader ran.
- `MIN aggregation evidence` — the per-row tiers fed into `pdf.min_tier(row_tiers)` and the resulting value.
