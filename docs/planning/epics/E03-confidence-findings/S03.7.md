## Real-data findings baked into the implementation

1. **`"Sept"` 4-letter month abbreviation** — antelope STATEWIDE 900-20 row's date uses `"Aug. 15-Nov. 08"` (dotted), but the antelope per-HD A-licenses use `"Sept"` form.  `_parse_window` normalizes `"Sept" → "Sep"` before `strptime("%b ...")`.
2. **`license_tag.kind` heuristic** — initial 3-branch heuristic (general/B/statewide) missed 121 rows.  Expanded to 5 branches to cover `"Deer Permit: …"`, `"Elk Permit: …"` (91 rows), and `"Antelope License: XYZ"` non-STATEWIDE (30 rows).  Caught by plan-review pre-implementation.
3. **`ClosurePredicate.notification_channel`** is required (no Pydantic default).  Initial plan T8 didn't read it from the artifact — would have raised `ValidationError` at the first BMU.  Caught by plan-review pre-implementation.
4. **HD 309 has no `archery_only_season`** — 1-row absence; bear total season_definitions is 104, not 35×3=105.  The artifact's `None` skip-logic correctly omits the missing season.
5. **HD 170 elk has 2 `General Elk License` artifact rows** — same `license_code`, different `opportunity` cells ("Brow-tined Bull Elk" + "Brow-tined Bull or Antlerless Elk").  The builder emits both; the `ON CONFLICT (id) DO UPDATE` collapses them at insert.  202 sections share this duplicate-license_code-within-section pattern.  License-level fields are structurally identical for same-code rows; the semantic payload is the UNION of season_coverage handled per-row in T7's license_season builder.
6. **Per-row season-window divergence** within a section — S03.3's per-row windows can differ within a section (the "Elk B License: 699-01" pattern documented in S03.3's closure note).  T5's `_build_dea_season_definitions` emits a defensive WARN log when this occurs but applies first-occurrence-wins.  Expected, not a defect.

## Deferred items (M2 or later)

1. **Per-cell verbatim_rule precision** — V1 uses section-scoped `verbatim_text` for `license_tag.verbatim_rule`.  If a downstream consumer (e.g., a UI surface) needs cell-precision, M2 could decompose the row into a structured per-cell representation.  No current consumer demands this.
2. **`license_tag.species` granularity** — V1 uses artifact-level labels.  If MT ever ships a mule-deer-only license (no whitetail), the V1 "deer" shared license_tag wouldn't fit — revisit then.  Linked to Q16 in `docs/open-questions.md`.
3. **Bear `hound_training_season`** — all 35 V1 rows have this field set to None (correction-applied).  If a future correction reverses this, the builder will emit additional season_definitions automatically.
