## Probe findings

### Probe 1 — Bear ID coursework → EXTRACTION DEFECT (carve-out S03.6.1)

Text **is** in the bear booklet, page 2 right column under "Highlights/Reminders" / "Obtain a License":

> A hunter may purchase only one Black Bear License per year. A free Black Bear Identification Test Certificate is required to obtain a license. A hunter must take and pass a "Black Bear Identification test" before purchasing a Black Bear Hunting license. A hunter must present a certificate of completion issued by FWP at the time of purchase. The test is available online at: fwp.mt.gov/hunt/education/bear-identification

**Why S03.4 missed it:** `ingestion/states/montana/extract_black_bear.py:1167-1249` (`_extract_reporting_obligations`) is hard-scoped to `_CLOSURE_PROSE_PAGE = 7` right-column only. Page 2 was never read.

**Target-table reversal (PM 2026-05-19):** `reporting_obligation` is defined in CLAUDE.md as "post-harvest/in-season duties." The Bear ID Test is a **pre-purchase licensing prerequisite** — neither post-harvest nor in-season. Mapping to `kind="mandatory_check"` would conflate it with check-station physical inspection (the existing `tooth_submission`/`hide_skull_presentation` rows are the canonical use). Adding `kind="education"` would extend `reporting_obligation` to carry rules that aren't reporting or obligations.

**Correct target-table:** `MT-STATEWIDE-bear` regulation_record (new row owned by S03.6.1), mirroring `MT-STATEWIDE-antelope` (existing S03.6 anchor for DEA `900-20`).

**Carve-out as S03.6.1 (queued post-S03.9):** See epic § S03.6.1 placeholder.

### Probe 3 — General mandatory reporting → EPIC ASSUMPTION WRONG (line 807 diff)

DEA booklet (141 pages): zero mandatory-reporting text for deer/elk/pronghorn. Page 31 has "Harvest Survey" = voluntary phone-sampling program, NOT a reporting obligation. FWP website (`https://fwp.mt.gov/hunt/regulations`) enumerates mandatory-report species as ONLY: black bear (48-hour), wolf (10-day skull/hide), marten (24-hour), migratory swans (72-hour bill-card). Explicit "no mandatory reporting requirements listed for deer, elk, antelope, mountain lion, moose, sheep, or goat."

Bear artifact's `harvest_report` verbatim_rule is unambiguously bear-only ("All successful **black bear** hunters must personally report their **black bear** harvest within 48 hours…").

**Disposition:** Epic line 807 is the bug. Draft diff below, surface to PM at S03.9 close for explicit approval (do not silently edit).

## Pitfalls to file (at Stage 8 wrap-up)

1. **E03 story discovery — source-audit upstream artifacts for each epic-required row type.** Pattern bit twice (S03.8 B5, S03.9 three-blocker). The discovery agent's first action for any S03.X story should be: enumerate epic-required output rows, then grep the named upstream artifacts for keywords matching each. Cheap; catches scope mismatches before they become gate decisions.

2. **`reporting_obligation.kind` semantic boundary.** `reporting_obligation` models post-harvest / in-season duties only. Pre-purchase prerequisites (Bear ID Test, hunter education certifications) belong in `regulation_record.additional_rules` keyed by a STATEWIDE anchor, not as a `mandatory_check` row. Conflating the two would mis-answer consumer queries like "what mandatory checks does a successful hunter perform?".

3. **`submission_method` interpretation for multi-modal source text.** Verbatim text often lists multiple submission channels (e.g., "deliver in person or by mail," "call or use MyFWP"). `submission_method` accepts ONE Literal value. Strategy: pick the FIRST channel mentioned in the verbatim (the headlined modality); the full verbatim_rule preserves the source faithfully so the choice is non-lossy. Document the call in the working note.

4. **`id text`-PK UPSERTs update slug-encoded fields under the same id.** Surfaced 2026-05-21 by cubic-review round 3. All three project `id text`-PK upserts (`season_definition`, `license_tag`, `reporting_obligation`) currently update slug-encoded fields on conflict (kind/deadline/scope etc.) — drift between the hand-encoded slug and the structured fields would silently rewrite the meaning of existing rows that already have link-table references pointing at them. Q19 in `docs/open-questions.md` tracks the project-wide fix planned pre-M2; do NOT propagate this pattern to new helpers without addressing Q19.

## Q19 / drift-guard decision (2026-05-21)

Cubic-review round 3 surfaced a project-wide concern (db.py:201): the `_UPSERT_REPORTING_OBLIGATION_SQL` DO UPDATE clause refreshes slug-encoded fields (kind, deadline, applies_to_regions) under the same `id`. If a future dispatch-dict edit drifts the slug encoding from the structured fields, the meaning of existing rows would silently change. All three `id text`-PK UPSERT helpers in db.py have the same pattern (`_UPSERT_SEASON_DEFINITION_SQL` from S03.7, `_UPSERT_LICENSE_TAG_SQL` from S03.7, `_UPSERT_REPORTING_OBLIGATION_SQL` from S03.9), so this is a project-wide concern — not a S03.9 divergence.

**Decision:** Q19 filed as a pre-M2 blocker for the project-wide fix (will land across all three helpers in a single PR before first year-over-year re-ingestion run; ADR will be drafted at resolution; Option A "derive-and-assert" is the leading candidate). S03.9 ships a **local drift guard** as belt-and-suspenders for the V1 window: `_derive_expected_id_suffix(kind, deadline_hours, region_scope)` + `_assert_dispatch_dict_drift_free(_REPORTING_ROW_SPEC)` running at module load; raises `RuntimeError` with a diagnostic on drift. Scope is S03.9-LOCAL by design — does NOT touch db.py, does NOT touch S03.7 helpers, does NOT introduce shared infrastructure. Symmetry with S03.7 is preserved by keeping the assertion in the S03.9 module.

**Tests:** `TestDispatchDictDriftGuard` (9 tests) locks: canonical dispatch passes at module reload; drift in id_suffix/kind/deadline_hours raises with diagnostic naming the offending key; `_derive_expected_id_suffix` rejects unrepresentable deadlines and unknown regions; `_KIND_SLUG_OVERRIDES` shortening + `STATEWIDE`-vs-region ordering both exercised.
