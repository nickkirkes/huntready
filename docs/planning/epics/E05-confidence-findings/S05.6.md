## Review-driven change — distance is parameter-bound (not a hardcoded `5000`)

The Stage-6 review triad (code-reviewer Critical + silent-failure-hunter W1, static-analysis
concurring) flagged that the spec's illustrative reference SQL hardcoded the literal `5000`
while defining `_NO_HUNT_ZONE_NEARBY_DISTANCE_M = 5000` separately — a latent silent-drift
trap: E06's anticipated recalibration of the constant would not flow into the query. The fix
binds the distance as a third `%s` parameter, mirroring MT's actual precedent
(`montana/load_jurisdiction_bindings.py:588` binds the constant, not a literal). This is a
deliberate, more-faithful-to-MT departure from the spec's illustrative SQL snippet; it breaks
no AC substring assertion (no AC requires the literal `5000` in the SQL — AC-7's 5000 m
inheritance is documented via the constant + this note).

## Design decisions

- **Inline `_STATE`, no `_constants.py`.** Per the AC-1 `"or equivalent"` clause, `_STATE` lives
  inline in the `load_jurisdiction_bindings.py` scaffold, mirroring MT's actual precedent
  (MT keeps `_STATE` inline; the MT adapter has no `_constants.py`). A separate constants file
  for a single value would diverge from MT without benefit.
- **Module-level SQL constant — deliberate departure from MT.** MT inlines its SQL as a local
  variable inside `_query_nearby_hds_for_zone`. S05.6 promotes the SQL to a module-level named
  constant (`_QUERY_NEARBY_GMUS_FOR_ZONE_SQL`) so the regression test can import and inspect it
  directly without a MagicMock round-trip. The reference function still exists so E06 gets a
  faithful drop-in and so param-binding can be locked by test.

## AC-7 — 5000-meter threshold inheritance

The `_NO_HUNT_ZONE_NEARBY_DISTANCE_M = 5000` (meters) threshold is **inherited from Montana's**
`montana/load_jurisdiction_bindings.py` `_NO_HUNT_ZONE_NEARBY_DISTANCE_M: Final[int] = 5000`
(S03.10 Option A — boundary-to-boundary `ST_DWithin` on geography). It is the starting
calibration for CO's no-hunt-zone "nearby GMU" binding rule.

**Empirical recalibration for Colorado is deferred to E06.** CO's federal no-hunt zones
(National Parks / National Monuments / the Air Force Academy — the 10 S05.4 zones) differ in
size and GMU-adjacency from MT's 3 zones (Glacier NP / Sun River / Yellowstone NP), so the
5000 m value may want tuning once E06 runs the binding loader against the live CO geometry.
The final call — keep 5000 m or recalibrate — belongs to E06's binding-loader spec or its
closure note, NOT to S05.6.

## Naming tension (E06 cleanup candidate)

The four existing CO loaders (`load_gmus.py`, `load_state_boundary.py`,
`load_restricted_areas.py`, `build_overlay_fixture.py`) each define `CO_STATE_CODE = "US-CO"`
(public, no underscore, no `Final`). This scaffold introduces `_STATE: Final[str] = "US-CO"` to
mirror the MT binding-loader convention E06 inherits. That leaves two names for the same value
in the CO adapter. **Unifying them is an E06 cleanup candidate, deliberately out of S05.6's
scaffold-only scope** — S05.6 does not retrofit the 4 existing loaders.

## No `jurisdiction_binding` writes (AC-8)

The AC-8 invariant — zero `jurisdiction_binding` rows for any CO geometry throughout E05 —
holds vacuously: the scaffold has no executable write path, and no other E05 story writes
bindings. E06's S03.10-equivalent story is the first to write CO bindings.
