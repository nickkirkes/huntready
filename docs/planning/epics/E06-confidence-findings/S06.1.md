## Summary

Stood up Colorado's PDF fetch infrastructure by mirroring Montana's S03.1 pattern:

- **`ingestion/states/colorado/fetch_pdfs.py`** (new) — a CO-pathed copy of `states/montana/fetch_pdfs.py`. Logic mirrors MT; the diffs are the module docstring, the `_SOURCES_YAML` / `_FIXTURE_DIR` paths (`montana` → `colorado`), the argparse strings, and one post-review addition: it passes `expected_sha256=entry.get("expected_sha256")` to `fetch_pdf` so the operator pin is enforced on first fetch. Reuses the shared-lib primitives `ingestion.lib.pdf_fetch.fetch_pdf` + `PdfMetadata` + `PdfFetchError`. Both CO `document_type` values (`annual_regulations`, `correction`) are already in the lib's `_VALID_DOCUMENT_TYPES`.
- **`ingestion/ingestion/lib/pdf_fetch.py`** (post-review edit, 2026-06-09) — added a **state-agnostic** opt-in `expected_sha256` enforcement param to `fetch_pdf` (see Group B → "First-fetch SHA enforcement"). The lib stays state-agnostic — `TestNoColoradoLeakIntoSharedLib` + `TestFetchPdfNoStateAdapterImports` stay green — so this is consistent with ADR-005, but **S06.1 as shipped is NOT a zero-lib-edit story** (the *initial* build session made no lib edits; the post-review fix added this param).
- **`ingestion/states/colorado/sources.yaml`** — extended with a new top-level `pdfs:` key (alongside the existing `gis_layers:` from S05.1) carrying the two operator-resolved CPW PDFs.
- **`ingestion/states/colorado/fixtures/.gitignore`** — added `*.pdf` + `*-pending-reextraction.flag` rules (raw PDFs and drift markers stay local; per-PDF manifests are committed). Mirrors MT's PDF gitignore stanza.
- **`ingestion/tests/test_fetch_co_pdfs.py`** (new) — 24 tests (12 initial + 10 from 3 MT orchestrator-behavior classes ported during review + 2 pin-passthrough tests). **`ingestion/tests/test_pdf_fetch.py`** — +6 lib pin-gate tests (post-review, additive).

## Group A (at-merge) vs Group B (operator-pending) AC split

Per the E05/S04.1 "operator verifies live" pattern, S06.1's ACs split:

**Group A — satisfied at merge:**
- `fetch_pdfs.py` mirrors MT's shape; `TestNoColoradoLeakIntoSharedLib` + `TestFetchPdfNoStateAdapterImports` stay green. The initial build made zero `ingestion/ingestion/lib/` edits; the post-review pin fix then added one **state-agnostic** param to `pdf_fetch.py` (see Group B → "First-fetch SHA enforcement" and the Summary lib-scope bullet) — the lib stays state-agnostic, but the shipped story is not zero-lib-edit.
- `sources.yaml` `pdfs:` section added with the 2 HEAD-verified, fully-pinned entries; no `pending: true` entries (both URLs resolved). `expected_sha256` present per non-pending entry.
- gitignore policy in place (`*.pdf`, `*-pending-reextraction.flag`; manifests committable).
- Drift-marker / page-count-mismatch / empty-yaml / pending-blocks behaviors locked by `test_fetch_co_pdfs.py` (24 tests). SHA-pin and SHA-drift raise behaviors are owned by the shared lib and covered by `test_pdf_fetch.py` (incl. the 6 new pin-gate tests).

**Group B — operator-pending (deferred to first live run):**
- **Per-PDF manifest commits at first fetch** (epic AC). No live network fetch occurred in the build session, so the `*-pdf-manifest.json` files are not yet committed — this is the S03.1 AC #4 posture ("infrastructure-met, fixture-deferred to first ops-run"). The committed manifest is the artifact the SHA-256 drift gate compares against; it is created when the operator runs `ingestion/.venv/bin/python ingestion/states/colorado/fetch_pdfs.py`.
- **First-fetch SHA enforcement is now automated (post-review fix, 2026-06-09).** A cubic review flagged that `expected_sha256` was documentation-only and unenforced on first fetch (the bad bytes would silently become the baseline). Fixed at the root: `fetch_pdf` now takes an optional `expected_sha256` and, when given a real pin, enforces it on EVERY fetch — including the first, **before** any PDF/manifest is written — raising `PdfFetchError` on mismatch. CO's `fetch_pdfs.py` passes `entry.get("expected_sha256")`, so the operator's first live fetch is gated automatically against:
  - `co-cpw-big-game-2026-brochure` → `38cf26e1d0cdb930c38a9d18f04bbaced7c72a2573c86613c3ca5a9adb3582b3`  <!-- pragma: allowlist secret -->
  - `co-cpw-big-game-2026-correction-2026-02-19` → `8eddff7011d3c25ee3b119a3dacf46e5ff9028555051423972a80b1c9371dbf0`  <!-- pragma: allowlist secret -->
  A mismatch raises (stop-and-investigate per ADR-001: CDN served stale/corrupt bytes) and writes nothing — no manual SHA check needed. The change is state-agnostic in `ingestion/lib/pdf_fetch.py` (opt-in param; `"unknown"`/absent skips, so MT is unaffected). See the updated "`expected_sha256` … two SHA gates" entry in `.roughly/known-pitfalls.md`. The committed manifest from the first clean fetch remains the durable baseline for the separate re-fetch drift gate.
