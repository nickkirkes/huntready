**Group B ‚Äî operator-pending (deferred to first live run):**
- **Per-PDF manifest commits at first fetch** (epic AC). No live network fetch occurred in the build session, so the `*-pdf-manifest.json` files are not yet committed ‚Äî this is the S03.1 AC #4 posture ("infrastructure-met, fixture-deferred to first ops-run"). The committed manifest is the artifact the SHA-256 drift gate compares against; it is created when the operator runs `ingestion/.venv/bin/python ingestion/states/colorado/fetch_pdfs.py`.
- **‚öÝÔ∏è Operator first-fetch step ‚Äî verify the manifest SHA against the pin.** `expected_sha256` in `sources.yaml` is documentation/intent and is **NOT read by `fetch_pdf` at runtime** (inherited MT design): the runtime drift gate compares a re-fetch against the *committed manifest*, not against `expected_sha256`. Therefore on the **first** fetch (no prior manifest), `fetch_pdf` writes whatever bytes the CDN returns with no SHA check. So after the first run the operator MUST manually confirm the written manifest's `pdf_sha256` equals the `expected_sha256` pinned here before committing the manifest:
  - `co-cpw-big-game-2026-brochure` ‚Üí `38cf26e1d0cdb930c38a9d18f04bbaced7c72a2573c86613c3ca5a9adb3582b3`  <!-- pragma: allowlist secret -->
  - `co-cpw-big-game-2026-correction-2026-02-19` ‚Üí `8eddff7011d3c25ee3b119a3dacf46e5ff9028555051423972a80b1c9371dbf0`  <!-- pragma: allowlist secret -->
  A mismatch is a deliberate stop-and-investigate signal per ADR-001 (CDN served stale/corrupt bytes); do NOT commit the manifest until the bytes match the pin. Once the verified manifest is committed it becomes the durable drift baseline for all subsequent fetches.
