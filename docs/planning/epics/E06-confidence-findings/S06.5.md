## Summary

Populated `geometry.verbatim_rule` on the 10 S05.4 `kind='restricted_area'` rows (`CO-restricted-{slug}-geom`) from the CPW Big Game brochure, page 78 ("Land Closures & Use Restrictions"). Single-purpose targeted-UPDATE loader; no extraction-artifact dependency (the page-78 closure prose is not in `big-game-2026.json` — it is read from the raw PDF at runtime).

- **`ingestion/states/colorado/load_restricted_area_verbatim.py`** (new, ~275 LOC incl. docstrings) — state-agnostic-clean (AST guard). Reads page 78 via `lib/pdf.open_pdf` + `lib/pdf.extract_text(page, bbox=…)` (column crops), regex-anchors the two source spans, builds a 10-row verbatim map, runs pre-`db.connect()` guards (OQ7), and UPDATEs via `db.update_geometry_verbatim`. Three-phase `main()` (build → guards → conn/commit), `--dry-run`, no `--service-url` (S05.0 silent-lie-citation precedent), function-level logger.
- **`ingestion/ingestion/lib/db.py`** — new `update_geometry_verbatim(conn, geometry_id, text)` helper (S06.0/D6). Exact analog of `update_legal_description`: `UPDATE geometry SET verbatim_rule = %s WHERE id = %s`; fail-loud `RuntimeError` on `cur.rowcount == 0`; no commit (caller owns the txn). This is a state-agnostic write-path extension (the canonical extension point per the db.py module docstring), consistent with ADR-003/ADR-004 — **not** a state-specific lib edit.
- **`ingestion/tests/test_db.py`** — `TestUpdateGeometryVerbatim` (+4 tests).
- **`ingestion/tests/test_load_co_restricted_area_verbatim.py`** (new, 16 tests).

**Provenance — D5=(b) split-provenance.** Only `verbatim_rule` is written. `source` stays the PAD-US citation (`co-usgs-padus-arcgis-Federal_Fee_Managers_Authoritative_PADUS-0-2026`, `document_type='gis_layer'`) on all 10 rows. No schema / three-place-sync / migration change. The regulatory-text authority (CPW) lives in `verbatim_rule`; the geometry provenance (PAD-US) lives in `source`. (Option-(a) UPDATE-`source` was explicitly rejected at S06.0/D5; multi-source provenance remains a deferred PRD-002 ADR-candidate.)

## ADR-008 byte-equivalence (UPDATE surface)

Both written strings are verbatim substrings of `pdf.extract_text` output for their source span — no normalization, no rewording. Newlines (line-wrap) and the `man-\ndatory` soft-hyphen are preserved exactly as pdfplumber emits them. The parsers `raise RuntimeError` if their anchor is absent (brochure layout change) rather than hand-editing — flag-and-discuss, never silent rewrite. Locked by `TestParsers` (substring assertions), `TestNoLayoutTrueRegression` (no `layout=True` AST guard), and `TestLivePdfIntegration` (end-to-end byte-for-byte against the live PDF).

**Multi-column note:** page 78 is a dense 3-column layout; full-page `extract_text` scrambles columns (and a band crop with a `top` cut character-scrambles the AFA first line). Extraction crops by column over the **full page height**: NPS = right column `bbox=(392, 0, width, height)`; AFA = left column `bbox=(0, 0, 200, height)`. (Same family as the MT S03.5 legal-descriptions column-crop pitfall.)

## Phrasing case recorded (AC #485) — case (1) + a 9+1 split

CPW publishes **one generic sentence** for the National Park Service lands (case 1), shared across all **9 NPS rows** (4 National Parks + 5 National Monuments). The 10th zone — the **Air Force Academy** — is NOT NPS and gets its own distinct page-78 prose. So the loader writes a **9 + 1 split**, locked by `test_phrasing_case_locked_nine_nps_share_one_string_afa_distinct` and the `_check_map` guards (9 NPS values collapse to exactly one unique string; AFA differs).

- **9 NPS rows** (`verbatim_rule`, "Statewide Restrictions" item 5):
  > `National parks and monuments managed by the\nNational Park Service are closed to hunting. Check\npark websites for more information.`
- **AFA row** (`CO-restricted-united-states-air-force-academy-geom`, the AFA access-rules intro):
  > `AFA hunters must pay an AFA access fee and receive a man-\ndatory safety orientation. All rifle deer hunting is escorted,\nand is allowed only on the days, areas and by method of take\nauthorized by the AFA. Archery white-tailed deer hunting is\nallowed by limited permit only from Dec. 1-31, self-escorted\nin specific assigned areas. Call 719-333-3308 or visit usafa.\nisportsman.net for more details.`

REG + COMMENTS combination (ADR-015's `\n\n--- COMMENTS ---\n\n` separator): **N/A** — the source is a single brochure page, no REG/COMMENTS structure; the combiner was not added.

## 🚩 AFA classification flag — for S06.10 + PM (spec-revision-candidate, not a code defect)

The Air Force Academy is a regulated-**access** hunting area, **not** a no-hunt zone: GMU 512 carries CPW hunt codes (`D-M-512-S1-R`, limited cow elk, OTC whitetail `D-E-048-U6-R`); rifle deer hunts are escorted; access is by AFA fee + mandatory safety orientation. Its `verbatim_rule` now describes **how to hunt there**, which is faithful and useful data — but it would contradict a `role='no_hunt_zone'` binding.

**S06.10 must NOT bind the AFA geometry as `role='no_hunt_zone'`** (ADR-021). Recommended: bind it `role='other_overlay'` (restricted-access overlay), or revisit the S05.4 "federal no-hunt zone" classification of the AFA. The intent (user decision, 2026-06-17): **the AFA area must remain protected as a huntable zone in query results.** The other 9 (NPS) zones are genuine no-hunt zones and `role='no_hunt_zone'` is correct for them.

This contradiction first becomes visible at S06.5 (verbatim text population); resolving it is binding/classification territory (S06.10 + PM), out of S06.5's scope.

## Group A (at-merge) vs Group B (operator-pending)

Per the E05/S04.1 "operator verifies live" pattern (S06.6-onward posture; applies here because the live UPDATE targets operator-pending E05 Group B geometry rows):

**Group A — closed at merge:** loader code + `db.update_geometry_verbatim` helper + 20 additive tests + all quality gates (ruff, mypy on `lib/`, full suite 1701 + 4 skipped, cubic). `--dry-run` reads the local PDF, runs all guards, writes nothing. The live-PDF integration test runs in CI when the PDF is present and locks byte-equivalence.

**Group B — operator-pending (no DB write in the build session):**

1. Ensure the brochure PDF is fetched locally:
   `cd ingestion && .venv/bin/python states/colorado/fetch_pdfs.py`
2. Dry-run first (validates extraction + guards, no DB):
   `.venv/bin/python states/colorado/load_restricted_area_verbatim.py --dry-run`
   → expect exit 0 and `S06.5 verbatim map built: 9 NPS rows + 1 AFA row; NPS=130 chars, AFA=397 chars`.
3. Live UPDATE against the production service-role DSN (requires the 10 S05.4 geometry rows already written — E05 Group B precondition):
   `.venv/bin/python states/colorado/load_restricted_area_verbatim.py`
4. SQL spot-check (AC #482 / #483 / #486):
   ```sql
   SELECT id, (verbatim_rule IS NOT NULL) AS has_text, left(verbatim_rule, 50) AS preview,
          (source->>'document_type') AS source_doctype
   FROM geometry
   WHERE id LIKE 'CO-restricted-%' ORDER BY id;
   ```
   Verify: all 10 `has_text = true` (AC #482); `source_doctype = 'gis_layer'` on all 10 (AC #483, D5=b — `source` unchanged); the 9 NPS rows share the closure sentence; AFA shows `AFA hunters must pay…`. PM-run spot-check ≥3 zones (e.g. Rocky Mountain NP, Mesa Verde NP, AFA) against the brochure (AC #486).

Once captured, PM ticks the Group B ACs (#482, #483, #486) in a follow-up doc-only commit.
