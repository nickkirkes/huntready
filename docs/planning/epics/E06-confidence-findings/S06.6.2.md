## Phase B decision — Outcome (a): documented epsilon relaxation

**Locked: relax `rel_tol` `1e-6` → `1e-3` (0.1%).** Rationale:

- Max relative area-loss across all GC-branch zones is RMNP's **−0.0676%** (the other 9 never enter the
  GC branch). **No zone exceeds 0.1%.**
- The loss is a benign ring-self-intersection cleanup artifact (recovered matches published acreage to
  0.04%), not real data loss.
- `1e-3` clears 0.0676% with margin while still failing loud on genuine loss: overlapping polygons in a
  GeometryCollection drop real coverage and produce gaps orders of magnitude larger — the regression test
  `test_geometry_collection_raises_when_area_not_preserved` exercises ~−12.5%, which still raises at the
  new threshold by ~125×.

**Outcome (b) did NOT fire** — no zone > 0.1%, so no re-source, **no PM flag-and-discuss event, and no
ADR** were required. (Per spec, outcome (b) would have triggered a multi-source-provenance ADR decision
surfaced to the human before Phase C; it did not.)

**Outcome (c) buffer(0) rejected** — empirically, a universal `buffer(0)` pre-process changes the area of
zones that don't need it (e.g. Dinosaur NM −0.576%, which currently passes via a direct `make_valid`
MultiPolygon), so it is not a clean fix. The targeted epsilon relaxation touches only the GC-recovery
branch and leaves the 9 passing zones byte-for-byte unchanged.

## Cubic review disposition

3 cubic iterations during Stage 6:
- **Iter 1** (1 issue, P2): `TestPadusTenZoneIntegration` skips in CI (features gitignored) → the real-data path isn't CI-locked. **Addressed**: added `test_self_intersection_recovers_through_make_valid_geometrycollection`, a CI-run unit test that reproduces the real RMNP mechanism with a small synthetic self-intersecting Polygon (make_valid → `GeometryCollection([Polygon, LineString])`, 0.015% artifact — inside 1e-3, above the old 1e-6, so it would have raised pre-fix). The tolerance is now CI-locked through the genuine `make_valid` path, not just the GC-input area-comparison.
- **Iter 2** (2 issues, P2): (a) the new self-intersection test "doesn't obviously self-intersect" → **addressed** with an explicit `assert not shape(...).is_valid` precondition that fails loud if a future GEOS makes the input valid; (b) the relaxation can accept up-to-0.1% loss → see below.
- **Iter 3** (1 issue, P1): "GeometryCollection recovery now accepts materially lossy shapes" — restates the deliberate relaxation tradeoff.

**Disposition of the persistent finding (accepted by design — cubic-termination option (c)):** the `1e-6 → 1e-3` relaxation accepting up to 0.1% area discrepancy in the GC-recovery branch is the story's central, evidence-backed decision (Phase B Outcome (a)), endorsed at Stage-4 plan review and independently validated by the silent-failure-hunter ("sound and appropriately bounded"). It is **not silent** — every recovery emits a WARNING naming OBJECTID + attributes + artifact count + the threshold, for operator audit; ADR-001's fail-loud guard still raises for any gap > 0.1%. The spec explicitly sanctioned this: "PM expectation is outcome (a) with a documented small epsilon relaxation IF the area-loss is genuinely sub-percent" — and Phase A measured it at 0.0676% (a benign self-intersection cleanup artifact; recovered area matches published GIS_Acres to 0.04%). The only way to "resolve" the finding is to revert to 1e-6, which rejects RMNP's correct geometry and defeats the story. Accepted as documented-deferral; the rationale trail is the inline comment in `arcgis.py`, this memo, and the SFH sign-off.
