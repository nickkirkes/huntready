## Carve-out provenance

S06.8.0 was carved out 2026-06-24 at S06.8 build Stage-2 discovery + an independent Phase-A brochure probe against the committed CO extraction artifacts.

The committed artifacts (`extracted/big-game-2026.json`, `extracted/black-bear-2026.json`) cover per-unit hunt tables (brochure pp. 33–71) but carry **zero draw-mechanics data**: `apply_by`, `quota`, `quota_range` are null on all 2,762 big-game rows; no `draw_phase` / `successor_hunt_code` / hybrid-eligibility / residency-cap field appears anywhere. S06.8's 80/20-hybrid + 20/25-residency premise required the brochure draw-instructions front matter (pp. 8–32), which the per-unit extractor never captured.

Per user decision 2026-06-24 ("carve out re-extraction first") and the established mid-epic carve-out precedent (S06.3.1 / S06.6.1 / S06.6.2), S06.8.0 extracts that front matter into `extracted/draw-mechanics-2026.json` BEFORE S06.8 builds the `draw_spec` loader.

## Downstream (S06.8 Phase C)

S06.8's ACs are revised to consume `draw-mechanics-2026.json`:

- Hunt codes in the extracted hybrid set → 80/20 pools (`rank_ordered_by_points` 0.80 + `unweighted_random` 0.20) + `min_points=5` + `residency_cap.nonresident_max_share=0.20` (for applicants with ≥6 pts) or `0.25` (for <6 pts).
- All other limited-draw codes → single 1.0 `rank_ordered_by_points` pool + `nonresident_max_share=0.25`.
- `application_deadline` from the extracted `important_dates` singleton keyed by `draw_phase`.
- `purchase_only_code` per species from the extracted `point_only_code` records.
- **Coupling lock** (hybrid pool-count ⇔ residency cap) is now anchored to extracted data, not research-note assumptions; the 20%/25% split is read directly from the `nr_allocation` singleton.
