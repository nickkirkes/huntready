## Key decisions

1. **Constants are derive-and-asserted** from the artifact (`hybrid_mechanics` + `nr_allocation`) via `_validate_artifact_constants` (`math.isclose` for floats): `_HYBRID_RANDOM_POOL_MIN_POINTS=5`, `_HYBRID_PREFERENCE_POOL_SHARE=0.80`, `_HYBRID_RANDOM_POOL_SHARE=0.20`, `_HYBRID_ELIGIBILITY_POINT_LINE=6`, `_HIGH_DEMAND_NR_CAP=0.20`, `_STANDARD_NR_CAP=0.25` — single source of truth; any extractor re-run drift raises.
2. **`_HYBRID_ELIGIBILITY_POINT_LINE` is validate-only** — the hybrid-vs-non-hybrid decision is an UPSTREAM determination made by `extract_draw_mechanics.py` (which classified codes into `hybrid_code` records); the constant is a drift guard, intentionally NOT consumed in `AllocationPoolEligibility`. Its equality with `nr_allocation.high_demand_threshold_points` (both 6) is intentional per spec L928/L934 (the same upstream determination drives both the pool split and the residency cap).
3. **Coupling lock observed-at-extraction** (AC #973): `TestCouplingLock` reads the expected 0.20/0.25 caps from the artifact's `nr_allocation` record, not inline literals. On 2026 data: `len(pools)==2 ⇒ 0.20`, `len(pools)==1 ⇒ 0.25` holds for all rows.
4. **`drift_guard` NOT imported** (composite-PK `(state, hunt_code, year)`, no slug drift per ADR-020); AST-locked by `TestNoDriftGuardImport`.
5. **Q12 NOT fired** — `parameters=NULL` on every row. **Q17 NOT fired** — zero limited-draw rows carry a non-null `quota`; the cross-listing validator (`_validate_cross_listing_consistency`) is implemented defensively (Q17 STOP guard) but never raises on 2026 data.
6. **`successor_hunt_code_key` / leftover phase** — the artifact carries no successor/leftover data; all rows are `draw_phase="primary"`, `successor=NULL`. Documented absence (not a flag-and-discuss STOP — nothing contradicts).
7. **Backfill-id correctness** — both build walks mirror S06.7's `_co_license_tag_id` derivation byte-for-byte: bear `species_group="bear"` (not `"black_bear"`); both walks use `row["gmu_code"]` (row-level, no section fallback) matching `load_seasons_and_licenses.py:1024/1345`; blank-gmu rows skip-with-WARNING (S06.7 parity).

## Malformed bear hunt codes — human-ratified decision (skip-with-WARNING)

3 bear GMU-851 hunt codes carry an upstream `+`-suffix corruption (`B-E-851-O1-M +`, `B-E-851-O2-R +`, `B-E-851-O5-R +`) — an `extract_black_bear.py` row-fusion residual. **Decision (user-ratified 2026-06-27): skip them with WARNING** rather than write a malformed string as a `hunt_code` PRIMARY KEY. Rationale: a malformed PK is un-lookupable in the serving DB; "faithful-load" was rejected because it propagates the defect; "normalize-in-loader" is incoherent because S06.7 already wrote the matching license_tag with the *same* corrupted id (`CO-GMU-851-bear-B-E-851-O2-R +-2026`), so a clean-normalized backfill would target a nonexistent id and fail loud. The 3 corrupted bear license_tags keep `draw_spec_key=NULL` (a tracked gap). The 3-agent review triad concurred; cubic dissented (preferred faithful-load) across 2 iterations — escalated and ratified per Stage-6 option (c).

These 3 codes also surface as **orphan hybrid codes** (`B-E-851-O1-R/O2-R/O5-R` — in the hybrid set, no clean limited-draw match): the `+` corruption explains `O2-R`/`O5-R`; `O1-R` (rifle) is a genuinely-absent per-unit variant. Both surfaces are WARNING-logged (`_report_orphan_hybrid_codes` + the build-walk malformed skip).

**→ New Known-Issue candidate (post-E06 bear-extractor carve-out):** fix the `extract_black_bear.py` GMU-851 row-fusion residual so both S06.7 and S06.8 emit clean ids; then the 3 bear codes get proper draw_specs + backfills. Analog of S06.3.1 (big-game R16). Out of S06.8 (loader) scope per ADR-022.
