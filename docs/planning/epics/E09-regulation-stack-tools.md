## Objective

E09 implements the three regulation-stack MCP tools — `get_regulations`, `list_seasons`, `get_tag_requirements` — against the Montana corpus, built entirely on the E08 foundation. They share the most query/join logic (all read the regulation/season/license/draw-spec/reporting stack through `jurisdiction_binding` fan-out and the link tables), so **`get_regulations` is the flagship that designs the shared layer once** (jurisdiction resolution, the bounded composite query, coverage-signal logic, and the generalized response mechanisms); `list_seasons` and `get_tag_requirements` are narrower reads over the same tables that reuse it.

When E09 closes: each tool returns a spec-valid response validating against its **own declared `outputSchema`**, with `structuredContent` as the source of truth, read-only annotations set, `sources[]` on every response, and schema-version-gated rows surfaced in `meta.warnings` (never silent-dropped). `verbatim_rule` and `confidence` pass through byte-identically. Out-of-scope queries return a structured `coverage: "none"` (not empty, not error). Tools are schema-generic (no MT-specific branches) so Colorado works at `m2` without server changes.

See [PRD 003 §"E09 — Regulation-stack tools"](../prds/003-M3-canonical-interface.md) (outcome + exit criteria), `architecture.md` §"The MCP server" (tool signatures), §"Response shape: `GetRegulationsResponse`" (the Shape C envelope, amended 2026-06-30 — `data_freshness` nullable), and §"Data model" (the 6 entities + link tables). Load-bearing ADRs: [ADR-001](../../adrs/ADR-001-authority-preserved.md), [ADR-006](../../adrs/ADR-006-schema-versioned-from-day-one.md), [ADR-008](../../adrs/ADR-008-verbatim-regulation-text.md), [ADR-010](../../adrs/ADR-010-decomposed-entity-model.md), [ADR-011](../../adrs/ADR-011-shape-c-response-envelope.md), [ADR-012](../../adrs/ADR-012-draw-mechanics-sibling-entity.md), [ADR-013](../../adrs/ADR-013-server-returns-structure-client-composes-presentation.md), [ADR-017](../../adrs/ADR-017-confidence-calibration.md).

**The four E08-resolved decisions are baked into the ACs below** (decided 2026-06-30): (1) `data_freshness` nullable + `sources` empty on `coverage:"none"`; (2) relax the empty-`tools/list` bootstrap + an exact-tool-set lock; (3) builder-owned warning accumulation + per-tool gating test; (4) audit Rec 5 — a single-read-path guard.

## Validation triad notes (2026-06-30)

Three serving reviewers (Source-Faithfulness/Citation · Shape-C Fidelity · Query-Correctness/Coverage-Signal) validated the draft; all returned **LAND-WITH-EDITS** (no NEEDS-REVISION). All findings applied. The load-bearing corrections, each verified against the live code/schema:

- **`species` → `species_group` was mischaracterized (Query reviewer MUST-FIX).** There is **no `deer`/`antelope` row in the DB** — the DEA fan-out/rename happens at *ingestion*; both MT and CO store granular `species_group` (`mule_deer`/`whitetail`/`elk`/`pronghorn`/`bear`). So the tools are already schema-generic with no serving-time mapping/branch; the `species` enum accepts granular values. `architecture.md:161`'s `species_group` comment is itself stale vs the corpus (forward-note flagged).
- **🚩 `additional_rules` has no envelope slot (Source-Faithfulness MUST-FIX).** `regulation_record.additional_rules` (HD-wide verbatim `NOTE:` lines, the Bear ID Test prose) isn't in `GetRegulationsResponse` → would silently drop verbatim regulatory text (ADR-001/008 gap). Surfacing it is an `architecture.md` §"Response shape" addition (a human-approved amendment). **Flagged to the human** + a Stage-1 flag-and-discuss AC in S09.1 (PM lean: surface). A "no-silent-authority-loss" guard AC added to catch the next such gap.
- **`buildStructuredToolResult` generalization (Shape-C MUST-FIX ×2).** It must parameterize **`renderThinText`** too (currently `GetRegulationsResponse`-typed, reads fields the narrower envelopes lack) → `(payload, schema, renderText)`; and each tool needs a **negative test that validation still bites** (throws on a schema-violating payload). `gateBySchemaVersion`'s return-and-append contract is replaced by the builder-owned sink (JSDoc updated).
- **Byte-identity must assert vs a direct DSN read** (Source-Faithfulness MUST-FIX), not the tool's own output (else tautology); include the ADR-015 `--- COMMENTS ---` separator + newline case. `confidence` on `Resolved*` rows is **inherited from the parent `regulation_record`** (child tables have no confidence column, ADR-017 §3) — copied, not re-derived.
- **Narrower-tool envelopes pinned (Shape-C SHOULD-FIX).** `list_seasons`/`get_tag_requirements` define their own `.strict()` envelope with `meta` = the jurisdiction-axis-free shape (`coverage` is a **single** `Coverage` value, not the 3-field object); reuse `ResolvedSeasonWindow`/`ResolvedTag` + their schemas **verbatim**; the `data_freshness`-null-iff-`sources`-empty invariant re-locked per envelope.
- **Query correctness (Query reviewer).** `ST_Covers(polygon_geom, point)` with `POINT(lng lat)` (lng first); the resolution predicate needs **no WKT round-trip** (only ancillary geometry-only selects do); `state` is an *output* of resolution; link tables join on the **4-col composite**; the species filter keys on `regulation_record.species_group` (not the denormalized `license_tag.species`); `draw_spec_key` is a jsonb soft FK (LEFT JOIN on composite); `residency` is a containment match. "One bounded query" relaxed to "a small constant 1–3, no per-binding fan-out" (a single flat seasons×tags×reporting join row-multiplies — worse). The read path is GiST + composite-PK equijoins — no ingestion-time `statement_timeout` risk.
- **Coverage boundary added:** jurisdiction-resolved-but-species-absent → `jurisdiction:"full"`, `species:"none"`, `overall:"partial"`, **non-empty `sources` + non-null `data_freshness`** (not collapsed to `none`).

**One item routed to the human (see forward notes): the `additional_rules` envelope decision** — surface (PM drafts the `architecture.md` amendment for approval) vs defer. The PM did not silently choose.

## Grounding — the E08 foundation E09 reuses (cite by symbol)

(From discovery 2026-06-30; the repo has a documented line-citation-drift failure mode, so cite by symbol.)

- **`mcp-server/src/db.ts`** — `createDbClient(dsn): DbClient`; `DbClient.query<T>(text, params): Promise<T[]>` (positional `$1…` bindings, fixed SQL strings only — `postgres.js` `prepare:false`, `max:1`, connect-per-request) + `.close(): Promise<void>`. The **single read path** (Rec 5 guard). `READ_SMOKE_SQL` / `ST_VALID_SMOKE_SQL` model the Supabase idioms tools inherit: **every PostGIS call `extensions.`-prefixed**, and **geometry-only functions round-trip via WKT** `extensions.ST_GeomFromText(extensions.ST_AsText(geom), 4326)`.
- **`mcp-server/src/response-builder.ts`** — `buildStructuredToolResult(...)` (currently typed to `GetRegulationsResponse`; validates against `getRegulationsResponseSchema`, throws on violation, emits `structuredContent` + thin `content[0].text`) → **E09/S09.1 generalizes it to be schema-parameterized** so the narrower tools reuse it; `gateBySchemaVersion<T>(rows, supported?) → {included, warnings}`; `buildDataFreshness(sources, generatedAt)` (**currently throws on empty — S09.1 changes it to return `null` on empty** per decision 1); `renderThinText`; `READ_ONLY_TOOL_ANNOTATIONS` (`readOnlyHint`/`idempotentHint`/`openWorldHint`); `SUPPORTED_SCHEMA_VERSIONS=[2]`; `STALE_THRESHOLD_DAYS=180`.
- **`mcp-server/src/output-schema.ts`** — `getRegulationsResponseSchema` (zod, `.strict()` envelope; **currently `sources.min(1)` + non-nullable `data_freshness` — S09.1 changes both** per decision 1). Each E09 tool declares its **own** `outputSchema` here (or a sibling) for `registerTool`.
- **`mcp-server/src/server.ts`** — `createMcpServer()` + the `initializeEmptyToolRegistry` register-then-remove bootstrap (**S09.1 deletes it when the first real tool registers** — decision 2); tools register via `server.registerTool(name, { description, inputSchema, outputSchema, annotations }, handler)`.
- **`mcp-server/src/types/response.ts`** (+ barrel `types/index.ts`) — `GetRegulationsResponse` + section/`Resolved*` types; E09 adds the `list_seasons` / `get_tag_requirements` output types here (serving-composition, **not** the 6-entity `schema.ts` sync). `types/schema.ts` holds the 6 entity types + `DrawSpec` (reused, not redefined).
- **DB join (from `supabase/migrations/` + `schema.py`):** `regulation_record` composite PK `(state, jurisdiction_code, species_group, license_year)`; the link tables `regulation_season` / `regulation_license` / `regulation_reporting` join back on **all four** PK columns (not a single id) + their entity id; `license_season` (`license_tag_id`↔`season_definition_id`, per ADR-018); `jurisdiction_binding` flat FK cols (`regulation_record_state`/`_jurisdiction_code`/`_species_group`/`_license_year`) + `geometry_id` + `role`; `geometry.geom geography(MultiPolygon,4326)` (GiST-indexed) + `geometry.state`; `draw_spec` composite PK `(state, hunt_code, year)` (+ `point_system`/`pools`); `license_tag.draw_spec_key` is a **jsonb soft FK** `{state, hunt_code, year}` (LEFT-JOIN `draw_spec` on its composite PK; non-draw tags → `draw_spec: null`).
- **`species` → `species_group` (no serving-time mapping; identical for MT + CO):** the DB `species_group` column stores **granular** values only — `mule_deer`, `whitetail`, `elk`, `pronghorn`, `bear` — for **both** states; the DEA `deer`→(`mule_deer`,`whitetail`) and `antelope`→`pronghorn` fan-out/rename happens **at ingestion** (`load_regulation_records.py` `_DEA_SPECIES_FANOUT`), so there is **no `deer`/`antelope` row in the DB** and no MT-specific serving branch. (Note: `license_tag.species` is a **separate denormalized text column** — the species filter keys on `regulation_record.species_group` reached via the link tables, NOT on `license_tag.species`.) `architecture.md:161`'s `species_group` comment (`"deer" | … | "antelope"`) is stale vs the loaded corpus — do not inherit it (forward-note flagged).
- **Jurisdiction resolution (`get_regulations`):** point → `geometry` via `extensions.ST_Covers(geom, extensions.ST_GeogFromText('SRID=4326;POINT(<lng> <lat>)'))` → `jurisdiction_binding` (on `geometry_id`) → `regulation_record` (via the flat FK cols) → seasons/tags/reporting via the link tables. **`ST_Covers(polygon_geom, point)` = polygon-covers-point** (geom is arg 1); WKT is **`POINT(longitude latitude)` — lng first** (lat/lng transposition is the classic silent bug → wrong/empty jurisdiction). `ST_Covers` runs **natively on geography** — the point is built with `ST_GeogFromText`, so the resolution predicate needs **no WKT round-trip / no `::geometry` cast**; the round-trip is required only for ancillary *geometry-only* selects (e.g. `ST_AsText`/`ST_NumGeometries` of a boundary). **`state` is an *output* of resolution** (read `geometry.state` from the covering row), not a precondition — the GiST `ST_Covers` lookup need not be pre-filtered by state; **all *downstream* link-table queries carry the resolved `state`** (no cross-state fan-out).
- **Test gating:** live-DB tests use `describe.skipIf(!process.env.TEST_READONLY_DSN)` + a top-level CI guard asserting the DSN is set in CI; they run in CI against the dev corpus. **Baseline floor: 133 passed + 6 skipped (local) / 139 (CI)** — E09 grows it additively.

## Stories

### S09.1: `get_regulations` — flagship Shape C composite + generalized response foundation

**Status:** Not started

**As an** agentic client asking "I'm hunting {species} at {lat,lng} on {date} — what applies?"
**I want** `get_regulations(lat, lng, species, date)` to resolve the jurisdiction spatially and return the full Shape C `GetRegulationsResponse` — resolved jurisdiction + all null-bearing sections + `sources[]` + `meta` (coverage tri-state, freshness, warnings) — in a bounded query, with out-of-scope queries returning `coverage:"none"`
**So that** the canonical interface's headline tool proves the whole regulation-stack pattern, and the shared layer (jurisdiction resolution, bounded join, coverage signals, generalized response mechanisms) exists for `list_seasons`/`get_tag_requirements` to reuse

**UAT: yes** (Group A static/unit/CI-live closes at-merge; PM-run faithfulness spot-check is the UAT sign-off, non-blocking)

**Context:**

The flagship. Resolves `(lat, lng)` → `geometry` (PostGIS `extensions.ST_Covers` + `ST_GeogFromText('SRID=4326;POINT(<lng> <lat>)')`, **state-filtered first**) → `jurisdiction_binding` (on `geometry_id`) → `regulation_record(s)` (flat FK cols) → seasons/tags/methods/reporting/contacts via the link tables, composed into the Shape C envelope. Registers the **first** real MCP tool — so this story **deletes the `initializeEmptyToolRegistry` bootstrap** (decision 2) and **replaces the empty-registry lock with an exact-registered-tool-set assertion** (`tools/list` is exactly the V1 tools registered so far, no internal/health tools — the UAT #1 guard). It also lands the cross-cutting foundation the other two reuse:

- **Decision 1:** make `meta.data_freshness` nullable in `types/response.ts` + `output-schema.ts`; drop `sources.min(1)`; `buildDataFreshness` returns `null` on empty (the throw becomes a "non-empty-when-called" guard). A `coverage:"none"` response carries `sources: []` + `data_freshness: null` — invariant **`data_freshness` null iff `sources` empty**.
- **Decision 3:** the response builder **owns warning accumulation** — gating across sections writes into one shared sink the builder drains into `meta.warnings`, so a tool can't forget to wire them. (Note: `gateBySchemaVersion` today *returns* `{included, warnings}` for the caller to append — Decision 3 **replaces that contract** with the builder-owned sink; update its JSDoc accordingly.) Generalize `buildStructuredToolResult` to be **schema-parameterized** (`buildStructuredToolResult(payload, schema, renderText)`) so S09.2/S09.3 reuse it with their own `outputSchema`. **The current `buildStructuredToolResult` hard-calls `renderThinText`, which is typed to `GetRegulationsResponse` and reads `meta`/section fields the narrower envelopes lack — so thin-text rendering must also be parameterized** (a per-tool `renderText(payload): string`). Every tool still emits `content[0].text` as a **mechanical, field-derived** derivative (presence/coverage/warning inventory) — never a summarizing/paraphrasing "overview" (ADR-013/ADR-008). (`architecture.md`'s "overview text assembled from the structured sections" prose is loose — the impl is mechanical, not an overview.)
- **Rec 5:** a single-read-path guard (lint/test that only `db.ts` opens a DB connection).

**`additional_rules` surfacing (🚩 flag-and-discuss at Stage-1 — may need an `architecture.md` amendment):** `regulation_record.additional_rules: VerbatimRule[]` (HD-wide `NOTE:` lines, the Bear ID Test prose — verbatim text with per-rule `confidence`, one of only **two** confidence carriers per ADR-017 §3) has **no slot in the current `GetRegulationsResponse` envelope** (`architecture.md` §"Response shape" has resolved/seasons/tags/methods/reporting/contacts/sources/meta — nowhere for `additional_rules`). Surfacing it would drop verbatim regulatory text the corpus loaded → an ADR-001/ADR-008 gap. **PM lean: surface it** (add a section/field to `GetRegulationsResponse` — a serving-composition response-type addition that touches `architecture.md` §"Response shape", so it's a human-approved amendment like the `data_freshness` one). Resolve at Stage-1 *before S09.1 ships*: surface (PM-drafts the amendment for approval) or defer with a documented reason. Do **not** silently omit it.

**Coverage tri-state boundary cases (lock all three):** total gap → all sections null, `sources: []`, `data_freshness: null`, `coverage.overall:"none"`. **Mixed: point resolves to a known HD but the species has no `regulation_record` there → `jurisdiction:"full"`, `species:"none"`, `overall:"partial"`, and `sources` is NON-empty + `data_freshness` NON-null** (the jurisdiction resolved and is cited) with the season/tag sections null — do **not** collapse species-absent-under-a-resolved-jurisdiction into `coverage:"none"`/`sources:[]`. Populated-but-empty section (e.g. `reporting.obligations: []`) ↔ `coverage:"full"` ("not required" vs "no data").

**Coverage tri-state (the heart of Shape C — get it exactly right):** `meta.coverage.{jurisdiction, species, overall}`. Jurisdiction `none` = no geometry covers the point (or no binding for the resolved state); species `none` = species not in the dataset for the resolved jurisdiction; `overall` aggregates. A null section pairs with `coverage:"none"`; a populated-but-empty section (e.g., `reporting.obligations: []`) pairs with `coverage:"full"` ("not required" vs "no data"). A total gap → all sections null, `sources: []`, `data_freshness: null`, `coverage.overall:"none"` — structured, not empty, not error.

**Bounded query:** one query (target) — no per-`jurisdiction_binding` fan-out. The dense-area resolution sizing (median ~3 parents/child, max 16 overlays) must resolve in a bounded round-trip; validate on the largest MT district. Connect-per-request; `close()` via `ctx.waitUntil(client.close().catch(…))` after the `Response` returns (never `await` inline — a teardown error must not become a 500).

**Relevant ADRs:** [ADR-011](../../adrs/ADR-011-shape-c-response-envelope.md), [ADR-013](../../adrs/ADR-013-server-returns-structure-client-composes-presentation.md), [ADR-001](../../adrs/ADR-001-authority-preserved.md)/[ADR-008](../../adrs/ADR-008-verbatim-regulation-text.md), [ADR-006](../../adrs/ADR-006-schema-versioned-from-day-one.md), [ADR-012](../../adrs/ADR-012-draw-mechanics-sibling-entity.md) (embedded `draw_spec`). PRD 003 SC #2, #3, #8, #14, #15.

Arg order is **(polygon `geom`, point)** — polygon-covers-point; WKT is **`POINT(longitude latitude)`** (lng first — a lat/lng transposition test guards against the silent wrong-jurisdiction bug).
