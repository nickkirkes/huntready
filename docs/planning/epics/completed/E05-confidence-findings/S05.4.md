### 3. Pre-write cross-check (live ArcGIS probe)

A `returnCountOnly=true` request against the PAD-US FeatureServer with the V1 WHERE clause should return **11 features** (10 NPS units matching `Des_Tp IN ('NP','NM','NRA')` + 1 DOD AFA matching the MIL filter). The loader then drops Curecanti NRA (the lone `NRA` row) → **10 rows written**.

Verify the raw count before committing if in doubt, using the loader's exact V1 WHERE clause:

```bash
curl -s "https://services.arcgis.com/v01gqwM5QqNysAAi/arcgis/rest/services/Federal_Fee_Managers_Authoritative_PADUS/FeatureServer/0/query?where=State_Nm%3D%27CO%27%20AND%20%28%28Mang_Name%3D%27NPS%27%20AND%20Des_Tp%20IN%20%28%27NP%27%2C%27NM%27%2C%27NRA%27%29%29%20OR%20%28Mang_Name%3D%27DOD%27%20AND%20Unit_Nm%3D%27United%20States%20Air%20Force%20Academy%27%29%29&returnCountOnly=true&f=json"
# Expected: {"count": 11} (4 NP + 5 NM + Curecanti NRA = 10 NPS, + 1 DOD AFA)
```

Total pre-drop = 11; the loader drops Curecanti NRA post-fetch → post-drop = 10 written.

### 4. Post-write verification queries

Run against the production Supabase project using the service-role DSN. PostGIS lives in the `extensions` schema in Supabase — prefix `ST_*` calls with `extensions.` accordingly.

**Q1 — Row count**

```sql
SELECT count(*)
FROM geometry
WHERE kind = 'restricted_area'
  AND state = 'US-CO';
-- Expected: 10
```

**Q2 — PostGIS validity**

```sql
SELECT id, extensions.ST_IsValid(geom::geometry) AS is_valid
FROM geometry
WHERE kind = 'restricted_area'
  AND state = 'US-CO';
-- Expected: all rows is_valid = true
```

**Q3 — Acreage sanity (±10% of PAD-US GIS_Acres)**

```sql
SELECT id, extensions.ST_Area(geom::geography) / 4046.86 AS acres
FROM geometry
WHERE kind = 'restricted_area'
  AND state = 'US-CO'
ORDER BY id;
```

Expected values (from `docs/research/colorado-restricted-areas-evaluation.md` GIS_Acres table; tolerance ±10%):

| Zone | PAD-US GIS_Acres | ±10% range |
|---|---|---|
| Rocky Mountain National Park | 265,475 | 238,928 – 292,023 |
| Great Sand Dunes National Park | 104,902 | 94,412 – 115,392 |
| Dinosaur National Monument | 150,671 | 135,604 – 165,738 |
| Mesa Verde National Park | 53,445 | 48,101 – 58,790 |
| Black Canyon of the Gunnison National Park | 30,272 | 27,245 – 33,299 |
| Colorado National Monument | 20,448 | 18,403 – 22,493 |
| Florissant Fossil Beds National Monument | 6,264 | 5,638 – 6,890 |
| Hovenweep National Monument | 369 | 332 – 406 |
| Yucca House National Monument | 36 | 32 – 40 |
| United States Air Force Academy | 18,519 | 16,667 – 20,371 |

**Important note on Dinosaur and Hovenweep:** both units straddle the CO/UT state line. PAD-US returns the **full unit polygon** (attribute filter, not a spatial clip to CO). The acreage sanity check above is therefore against the full-unit GIS_Acres from PAD-US, not a CO-only fraction. Do not expect a CO-clipped sub-area value.

**No binding/role verification in S05.4 scope.** S05.4 writes `geometry` rows only — there is no `role` to confirm here. `jurisdiction_binding` rows (where `role='no_hunt_zone'` will be assigned per ADR-021) are written by **E06's** binding loader, not by E05. Binding verification belongs to E06's Group B, not this runbook.

## `verbatim_rule` follow-up

The CPW Big Game brochure URL must be resolved before `verbatim_rule` is populated. The brochure's role is to supply CPW's regulatory citation that "no hunting" applies within each named NP/NM/AFA boundary — the geometry itself comes from PAD-US. The single-`source`-field provenance question (PAD-US geometry vs CPW brochure text) is resolved at that time.

Until then, `geometry.verbatim_rule` is **NULL for all 10 rows** (the loader writes `verbatim_rule=None` explicitly). This is intentional V1 behavior — the PAD-US geometry is authoritative for the boundary; the verbatim CPW text citation is E06 / future-story scope.

The brochure URL resolution is a prerequisite for any future story that populates `verbatim_rule` on these rows. Operator must navigate to `https://cpw.state.co.us/regulations/hunting/regulations-brochures` in a browser to find the 2026 brochure's actual URL (WebFetch returned 404 for all four URL guesses on 2026-06-03).
