## Decisions recorded

- **Pure-module + thin-shim architecture** (`cors`/`auth`/`router` import no `agents/mcp`) is the serving routing/middleware pattern E09/E10 inherit.
- **`/healthz` dispatched before the auth seam** — latent-P1 fix: both read the single `Authorization` header but expect different tokens; sequencing `/healthz` behind the seam made it unreachable when both were enabled with distinct tokens.
- **CORS policy is 3-case** (trimmed): unset/empty/whitespace-only → permissive (open-V1 default); `*` as any entry → permissive; botched non-empty list with zero valid origins → fail-closed deny. CORS applied on **every** response path (incl. MCP-handler response + try/catch'd platform-500s).
- **New config/secret surface (E11 deploy runbook):** `CORS_ALLOWED_ORIGINS` (var, default `*`), `AUTH_SEAM_ENABLED` (toggle; unset = deployed-V1 disabled), `AUTH_SEAM_TOKEN` (Workers Secret, test/staging only) — alongside `SUPABASE_READONLY_DSN` + `HEALTHCHECK_TOKEN`.
- **Documented deferral (cubic conflict):** whitespace-only `CORS_ALLOWED_ORIGINS` stays **permissive** (explicit human finding + open-V1 default), conflicting with a cubic P2 that wanted fail-closed. Documented in `cors.ts`.
- **No new ADRs.** Refines ADR-023 (CORS = no-BFF/Q5 consequence + wired-unenforced seam; ADR-023 progresses toward `Accepted` — finalization + deployed verification is E11) and ADR-003/005.
