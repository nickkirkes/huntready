## Gap Analysis

Four patterns are missing from the current `.gitignore`. None are blocking ‚Äî all relate to tooling that will be introduced in later milestones.

### 1. `.pnpm-store/` ‚Äî pnpm cache directory

pnpm stores its content-addressable package cache in `.pnpm-store/` when using `pnpm install --store-dir .`. The story context mentions "npm, possibly pnpm" as a package manager. If pnpm is adopted, this directory would be generated locally.

### 2. `pnpm-debug.log*` ‚Äî pnpm debug logs

Analogous to `npm-debug.log*` and `yarn-debug.log*` already covered on lines 43‚Äì44. pnpm writes debug logs with this prefix.

### 3. `coverage/` ‚Äî test coverage output

Standard output directory for test coverage tools (Jest, Vitest, c8, pytest-cov). The roadmap references quality gates and testing from M1 onward. Coverage output will be generated once testing begins.

### 4. `.env.*` broadening with `!.env.example` ‚Äî broader environment variable blocking

The current `.gitignore` blocks `.env`, `.env.local`, and `.env.*.local`, but does NOT block `.env.production`, `.env.staging`, `.env.development`, or other non-`.local` variants. These are common in Next.js projects (web/ scaffold in S0.6).

Meanwhile, S0.3 creates `.env.example` which MUST be tracked. The recommended pattern `.env.*` with negation `!.env.example` closes this gap while preserving the example file.

**Note:** When applying this change, the existing `.env.local` line (line 10) and `.env.*.local` line (line 11) should both be removed because `.env.*` is a strict superset of both patterns. `.env` on line 9 must stay ‚Äî it is NOT matched by `.env.*` (the wildcard requires at least one character after the dot). Keeping the redundant lines is harmless but inconsistent.
