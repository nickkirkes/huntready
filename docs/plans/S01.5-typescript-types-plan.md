## Tasks

### T1: Create schema.ts with all entity interfaces and supporting types (~4 min)

**Files:** `mcp-server/src/types/schema.ts`
**Action:** Create the directory `mcp-server/src/types/` and the file `schema.ts` containing all exported types from `docs/architecture.md` lines 151–344.
**Details:**

Order: type aliases first (referenced by interfaces), then supporting interfaces, then entity interfaces.

**Type aliases** (`export type`):
- `WeaponType` — union of 9 string literals: `"any_legal_weapon" | "archery" | "rifle" | "muzzleloader" | "shotgun" | "handgun" | "crossbow" | "traditional_handgun" | "heritage_muzzleloader"`
- `Residency` — `"resident" | "nonresident" | "both"`
- `GeometryRole` — union of 7 string literals: `"primary_unit" | "portion" | "restricted_area" | "cwd_management_zone" | "bear_management_unit" | "block_management_area" | "other_overlay"`

**Supporting types** — use `export type` for `PointSystem`, `ResidencyCap`, `ChoiceConfig`, `AllocationPool` (these are `type = {...}` in architecture.md, not `interface`). Use `export interface` for `SourceCitation`, `VerbatimRule`, `ClosurePredicate`, `ReservedPool`:
- `SourceCitation` — 8 fields (`id`, `agency`, `title`, `url`, `publication_date`, `document_type` union, `supersedes: string | null`, `page_reference: string | null`)
- `VerbatimRule` — 4 fields (`text`, `page_reference: string | null`, `confidence` union, `source: SourceCitation`)
- `ClosurePredicate` — 6 fields (`kind` union, `threshold_percent: number | null`, `threshold_sex: "male" | "female" | null`, `notification_channel` union, `observation_channel` union `| null`, `verbatim_rule: string`)
- `ReservedPool` — 5 fields (`share: number`, `eligibility` inline object with `kind` required + 5 optional fields using `?:`, `applies_to_residency: Residency | null`, `nonresident_subcap: number | null`, `verbatim_rule: string`)
- `PointSystem` — 5 fields (`kind` union, `accrual` union, `reset_on_success: boolean`, `purchase_only_code: string | null`, `inactive_forfeit_years: number | null`)
- `ResidencyCap` — 1 field (`nonresident_max_share: number`)
- `ChoiceConfig` — 2 fields (`count: number`, `points_used_in_choices: number[]`)
- `AllocationPool` — 4 fields (`share: number`, `selection` union, `eligibility` required field containing an inline object whose 3 inner properties are optional (`min_points?`, `residency?`, `guided?`), `tie_break?: "random" | "rank_ordered"`)

**Entity interfaces** (`export interface`):
- `RegulationRecord` — 13 fields. Includes 4 `_ids` arrays (`season_definition_ids`, `license_tag_ids`, `reporting_obligation_ids`, `jurisdiction_binding_ids`) — these are the logical API view, not DDL columns. Populated by JOINing link tables at query time.
- `SeasonDefinition` — 10 fields
- `LicenseTag` — 14 fields. `draw_spec_key` is an inline `{ state: string; hunt_code: string; year: number } | null` (not a named type — matches architecture.md exactly)
- `DrawSpec` — 14 fields. `parameters: Record<string, unknown> | null` (no `any`). `successor_hunt_code_key` same inline shape as `draw_spec_key`. `pools: AllocationPool[]` (no default; empty is invalid).
- `ReportingObligation` — 11 fields
- `Geometry` — 7 fields. `geom: "MultiPolygon"` literal type per architecture.md.
- `JurisdictionBinding` — 6 fields. `regulation_record_key` is a nested inline object `{ state: string; jurisdiction_code: string; species_group: string; license_year: number }` — this is the API-facing shape; the DDL and Python use flat columns.

**Key rules:**
- Copy faithfully from architecture.md — do not rename fields, change nullability, or restructure
- Use `| null` for nullable fields (not `?:`), except where architecture.md uses `?:` (only in `ReservedPool.eligibility` inner fields, `AllocationPool.eligibility` inner fields, and `AllocationPool.tie_break`)
- No `any` types — `Record<string, unknown>` for the parameters escape hatch
- No runtime code, no imports from outside the types directory
- Add a brief file header comment noting this is the TS leg of the three-place schema sync

**Verify:** `cd mcp-server && npx tsc --noEmit`
**UI:** no

- Per ADR-005: TypeScript for serving, Python for ingestion, three-place manual sync
- Per ADR-006: `schema_version` on `RegulationRecord` and `DrawSpec`
- Per ADR-008: `verbatim_rule` is always `string` (not nullable on entities)
- Per ADR-010: Decomposed entity model — 7 entity interfaces
- Per ADR-012: `DrawSpec` is a sibling entity; `parameters` escape hatch uses `Record<string, unknown>`
- Per CLAUDE.md: No `any` types
