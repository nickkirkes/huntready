## Strategy

S02.5 is investigation-driven. The plan executes a runtime investigation FIRST (T1–T2), then takes a runtime decision branch (T3) into one of three implementation paths:

| Path | Trigger | Implementation tasks |
|---|---|---|
| **A** — standalone CWD layer found in MT FWP catalog or Hub | T1 returns a layer URL | T4 (no schema change), T6, T7 (skip T5), T8–T11 |
| **B** — exclusion-filter over layer #2 | T2 returns ≥1 CWD-matching row | T4 (no schema change), T6 (Path B branch), T7 (skip T5), T8–T11 |
| **C** — hand-traced GeoJSON (most likely per S02.4 empirical) | T1 = no layer AND T2 = 0 rows | T4 (schema change), T5, T6, T7, T8–T11 |
| **D** — escalate as M1 blocker | Path C blocked (no PDF / no zones traceable) | Stop after T3, escalate to PM |

The plan covers all three implementation paths. The decision in T3 is what determines which downstream tasks fire and in what configuration.

**Schema decision (resolved in plan):** If Path C is chosen, add an optional `notes: dict[str, Any] | None = None` field to `SourceCitation` in both Pydantic (`ingestion/lib/schema.py`) and TypeScript (`mcp-server/src/types/`). The existing `extra="forbid"` config rejects ad-hoc keys, so an explicit additive field is required. Document the change in `docs/architecture.md` per ADR-006. If Path A or B wins, this task is skipped — no schema change is needed for either.

### T5: Hand-traced CWD zones GeoJSON (~5 min, plus offline tracing time)

**Files:** `ingestion/states/montana/cwd-zones-manual.geojson`

**Depends on:** T3 (only run if Path C; SKIP for Path A or B)

**Action:** Create the hand-traced GeoJSON FeatureCollection with three CWD Management Zone polygons.

**Details:**

The hand-tracing itself is offline work — using a tool like geojson.io, the human draws polygon boundaries from the source PDF descriptions. The build pipeline checks that the file exists and validates against schema. Each Feature must have:

- `geometry.type` ∈ {`Polygon`, `MultiPolygon`}
- `geometry.coordinates` in WGS84 (EPSG:4326), longitude-then-latitude per GeoJSON spec
- `properties.name` (string, e.g., "Libby CWD Management Zone")
- `properties.regulation_year` (integer, e.g., 2026)
- `properties.source_pdf_page` (string, e.g., "Legal Descriptions 2026, p. 14")
- `properties.zone_slug` (string, kebab-case for ID derivation, e.g., `libby`, `south-central-mt`, `northeast-mt`) — added so the loader doesn't need to slugify `name` (avoids ambiguity)
- `properties.validated_by` (string — the human who validated the polygon against the source PDF, e.g., `nick@rowdycloud.io`)
- `properties.source_url` (string — URL to the regulation page or PDF, used as `SourceCitation.url` for this feature)

Example skeleton:
```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "properties": {
        "name": "Libby CWD Management Zone",
        "regulation_year": 2026,
        "source_pdf_page": "Legal Descriptions 2026, p. 14",
        "zone_slug": "libby",
        "validated_by": "nick@rowdycloud.io",
        "source_url": "https://fwp.mt.gov/hunt/regulations"
      },
      "geometry": {
        "type": "Polygon",
        "coordinates": [[[<lng>, <lat>], ...]]
      }
    },
    ...
  ]
}
```

**Three zones required.** If the live booklet names different zones than Libby / South-Central / Northeast, substitute and document in the discovery report (T3).

### T6: Implement `load_cwd_zones.py` (~10 min)

**Files:** `ingestion/states/montana/load_cwd_zones.py`

**Depends on:** T3, and (Path C only: T4 + T5)

**Action:** Create the CWD zone loader. Mirror the structure of `load_restricted_areas.py` for path consistency. Branch internally based on the chosen path.

**Details:**

The loader must support whichever path was chosen in T3. The simplest design carries all three branches but only the chosen one runs at execution time, gated by a CLI flag or by inspecting which inputs exist (auto-detect).

**Recommended structure (auto-detect):**

```python
"""Montana CWD zone ingestion adapter (S02.5).

Loads CWD Management Zones into the geometry table with kind='cwd_zone'.

Source path is auto-detected:
  Path A: --layer-id and --service-url provided    → fetch from ArcGIS
  Path B: --filter-layer-2                         → fetch layer #2 + apply is_cwd_feature filter
  Path C: --geojson-path or default file present   → load hand-traced GeoJSON

See ingestion/states/montana/cwd-source-discovery.md for which path applies
to this run.
"""

# Imports mirror load_restricted_areas.py.
# Reuse: arcgis.{fetch_layer_metadata, fetch_features, geojson_to_multipolygon_wkt, build_source_citation},
#        db.upsert_geometries,
#        states.montana.cwd_discriminator.is_cwd_feature.

# Constants:
MT_HAND_TRACED_GEOJSON = Path(__file__).parent / "cwd-zones-manual.geojson"
MT_FWP_REGULATIONS_URL = "https://fwp.mt.gov/hunt/regulations"  # only used as Path C citation URL fallback
ID_PREFIX = "MT-CWD-zone"

def _path_c_feature_to_geometry(feature, fetch_year) -> Geometry:
    """Build Geometry for a hand-traced FeatureCollection feature."""
    props = feature["properties"]
    zone_slug = props["zone_slug"]
    geometry_wkt = arcgis.geojson_to_multipolygon_wkt(feature)
    citation = SourceCitation(
        id=f"mt-fwp-cwd-manual-{props['regulation_year']}",
        agency="Montana Fish, Wildlife & Parks",
        title="CWD Management Zones (manually traced from Legal Descriptions PDF)",
        url=props.get("source_url", MT_FWP_REGULATIONS_URL),
        publication_date=f"{props['regulation_year']:04d}-01-01",
        document_type="annual_regulations",
        page_reference=props["source_pdf_page"],
        notes={
            "manually_traced": True,
            "validated_against": props["source_pdf_page"],
            "validated_by": props["validated_by"],
        },
    )
    return Geometry(
        id=f"{ID_PREFIX}-{zone_slug}-geom",
        name=props["name"],
        kind="cwd_zone",
        geom=geometry_wkt,
        state="US-MT",
        license_year=props["regulation_year"],
        verbatim_rule=None,  # hand-traced from PDF; verbatim text is not embedded in the GeoJSON
        source=citation,
    )

def _path_c_load(conn, fetch_year) -> list[Geometry]:
    if not MT_HAND_TRACED_GEOJSON.exists():
        raise FileNotFoundError(f"hand-traced GeoJSON missing: {MT_HAND_TRACED_GEOJSON}")
    data = json.loads(MT_HAND_TRACED_GEOJSON.read_text())
    if data.get("type") != "FeatureCollection" or not data.get("features"):
        raise ValueError("cwd-zones-manual.geojson is not a non-empty FeatureCollection")
    geoms = [_path_c_feature_to_geometry(f, fetch_year) for f in data["features"]]
    if _duplicate_ids(geoms):
        raise ArcGISError(f"duplicate cwd_zone ids: {_duplicate_ids(geoms)}")
    db.upsert_geometries(conn, geoms)
    return geoms

def _path_a_load(conn, service_url, layer_id, fixture_dir, fetch_year, *, session):
    metadata = arcgis.fetch_layer_metadata(service_url, layer_id, fixture_dir,
                                           layer_slug="cwd-zones", session=session)
    features = arcgis.fetch_features(service_url, layer_id, metadata, fixture_dir,
                                     layer_slug="cwd-zones", session=session)
    if not features:
        raise ArcGISError(f"layer {layer_id} returned zero features (S02.5 path A)")
    geoms = [_path_a_feature_to_geometry(f, layer_id, metadata, service_url, fetch_year)
             for f in features]
    db.upsert_geometries(conn, geoms)
    return geoms

def _path_b_load(conn, fixture_dir, fetch_year, *, session):
    metadata = arcgis.fetch_layer_metadata(MT_FWP_HUNTING_DISTRICTS_URL, 2, fixture_dir,
                                           layer_slug="huntingDistricts", session=session)
    features = arcgis.fetch_features(MT_FWP_HUNTING_DISTRICTS_URL, 2, metadata, fixture_dir,
                                     layer_slug="huntingDistricts", session=session)
    matches = [f for f in features if is_cwd_feature(f.get("properties") or {})]
    if not matches:
        raise ArcGISError(
            "Path B: layer #2 returned zero CWD-matching rows. "
            "Switch to Path C (hand-traced) per cwd-source-discovery.md."
        )
    geoms = [_path_b_feature_to_geometry(f, metadata, fetch_year) for f in matches]
    db.upsert_geometries(conn, geoms)
    return geoms

def main(argv=None) -> int:
    parser = argparse.ArgumentParser(...)
    parser.add_argument("--path", choices=["A", "B", "C"], required=True,
                        help="Which path per cwd-source-discovery.md")
    parser.add_argument("--service-url", default=None, help="Path A only")
    parser.add_argument("--layer-id", type=int, default=None, help="Path A only")
    parser.add_argument("--fetch-year", type=int, default=datetime.now(timezone.utc).year)
    args = parser.parse_args(argv)

    with arcgis._build_session() as session, db.connect() as conn:
        if args.path == "A":
            geoms = _path_a_load(conn, args.service_url, args.layer_id,
                                 MT_FIXTURE_DIR, args.fetch_year, session=session)
        elif args.path == "B":
            geoms = _path_b_load(conn, MT_FIXTURE_DIR, args.fetch_year, session=session)
        else:
            geoms = _path_c_load(conn, args.fetch_year)
        # Single atomic commit — same pattern as load_restricted_areas.py.
        conn.commit()
    return 0

if __name__ == "__main__":
    raise SystemExit(main())
```

**Multi-path implementation is the chosen design.** Implement all three path branches as shown in the pseudocode above. The `--path` CLI argument selects which branch runs at execution time. Rationale: keeps the loader self-documenting, makes T3's path decision a runtime input rather than a code-shape input, and avoids re-editing the file if a future re-run picks a different path (e.g., FWP publishes a CWD MapServer next year and we move from Path C to Path A). The unused branches add ~30 lines and have unit tests in T7 — proportional cost.

**Constraints to honor:**
- Single atomic commit at end (per `.ruckus/known-pitfalls.md` and S02.4 precedent at line 271)
- Fail loud on zero features (S02.3 lesson — see `load_restricted_areas.py:197–202`)
- `is_cwd_feature` imported from `states.montana.cwd_discriminator`, NOT redefined (S02.4 precedent)
- ID slug strategy: `MT-CWD-zone-{zone_slug}-geom`
- For Path A: `arcgis.build_source_citation` (yields `document_type='gis_layer'`)
- For Path B: same as Path A's citation (it's still GIS-sourced)
- For Path C: hand-built `SourceCitation` with `document_type='annual_regulations'` and `notes={"manually_traced": True, "validated_against": props["source_pdf_page"]}`
- `verbatim_rule`: Path A may use `arcgis`-derived REG/COMMENTS if the new layer has those fields; Path B inlines a local copy of `_extract_verbatim_rule_combined` in `load_cwd_zones.py` (state-adapter-internal logic, not a cross-story contract — duplication acceptable per the same precedent that keeps `_slugify` local in each adapter); Path C is `None` (hand-traced from PDF, no inline regulation text)
