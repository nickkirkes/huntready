# Implementation Plan: S02.7 Spatial Query Verification + Epic Exit

**Source:** [docs/planning/epics/E02-geometry-ingestion.md:572-624](../planning/epics/E02-geometry-ingestion.md)

## Summary

Final E02 story with two workstreams:

1. **Workstream A — Manifest writer extension** to `arcgis.fetch_features` (~30 LOC + tests). Resolves known-issues #6: each `*-features-*.geojson` fetch now drops a paired `*-manifest-*.json` (committed, ~5 KB) with `features_count`, `layer_hash`, `hash_distribution`, `fetched_at`, `source_url`, `source_layer_max_record_count`, `source_layer_object_id_field`. Cross-operator drift signal that metadata fixtures can't catch.

2. **Workstream B — Verification artifacts + runbook**. Spot-check fixture per `kind`, runbook mirroring E01's structure, README updates, `.gitignore` policy comment, and a one-shot backfill script that generates manifests for the 10 layers ingested in S02.2-S02.5 from the local features GeoJSON files.

No new schema. No new geometry rows. No migrations.

### T2: Wire manifest computation + write into `fetch_features` (~5 min)

**Files:** `ingestion/ingestion/lib/arcgis.py`
**Action:** After `_write_features_fixture` (called around line 799 in `fetch_features`), compute per-feature hashes, the layer hash, and the 256-bucket histogram, then call `_write_manifest_fixture`.
**Details:**
- Use the validated post-dedup `checked` list (the same list passed to `_write_features_fixture`) — this is the count that passes the cross-check invariant.
- For each feature in `checked`, compute its hash via `compute_feature_hash(objectid=feature["properties"][metadata.object_id_field], geometry_wkt=geojson_to_multipolygon_wkt(feature), attributes=feature["properties"])`. Sort hashes lexicographically before combining.
- `layer_hash`: `hashlib.sha256("\n".join(sorted_per_feature_hashes).encode("utf-8")).hexdigest()`. Use a deterministic separator. (Document the separator and ordering choice with a one-line comment so future-you can reproduce the hash externally.)
- `hash_distribution`: dict with all 256 keys `"00"`–`"ff"` always present (zeros included), values = count of per-feature hashes whose `[:2]` prefix equals that key. Always-present keys ensures byte-identical re-fetch manifests; missing-key omission would cause spurious diffs. Use `{f"{i:02x}": 0 for i in range(256)}` as the seed.
- `source_url`: `f"{service_url}/{layer_id}"` (concrete service URL + layer ID).
- `source_layer_max_record_count`: `metadata.max_record_count` (the parsed field, not raw).
- `source_layer_object_id_field`: `metadata.object_id_field` (the parsed/fallback-corrected field — known pitfall: layer #11 has `null` in raw JSON, parsed value is `"OBJECTID"`).
- `fetched_at`: derive from the `timestamp` parameter that already drives the features filename. Convert `"%Y%m%dT%H%M%SZ"` → ISO 8601 with `Z` suffix. Use `datetime.strptime(timestamp, "%Y%m%dT%H%M%SZ").replace(tzinfo=timezone.utc).isoformat()`. The features fixture and manifest must share the exact same instant.
- `features_count`: `len(checked)`.
- Call `_write_manifest_fixture(fixture_dir, layer_slug, layer_id, timestamp, manifest)`.
- Return value of `fetch_features` is unchanged.
**Verify:** `cd ingestion && .venv/bin/ruff check ingestion/lib/arcgis.py && .venv/bin/mypy ingestion/lib/`

**Depends on:** T1

  - `## 3. Multi-part HD verification (named)`: `SELECT id, ST_NumGeometries(ST_GeomFromText(ST_AsText(geom), 4326)) FROM geometry WHERE id = 'MT-HD-deer-elk-lion-690-geom';` — expected `12`. Inline `-- Expected:` comment.

EXPLAIN ANALYZE plan documentation`: Run `EXPLAIN ANALYZE SELECT id FROM geometry WHERE ST_Covers(geom, ST_GeogFromText('SRID=4326;POINT(-114.320 48.310)'));` — paste the resulting plan inline as a code block.   - `## 5. Reproducibility (topological, not byte-level)`: Document the wipe + re-ingest pattern with the `ST_Equals` predicate. SQL: `SELECT a.id FROM geometry a JOIN <reloaded> b USING (id) WHERE NOT ST_Equals(a.geom, b.geom);` — expected zero rows. Or hash variant: `md5(ST_AsBinary(ST_Normalize(ST_GeomFromText(ST_AsText(geom), 4326))))`. **Critical note (E02-only):** "This wipe pattern works in E02 because nothing yet FK-references `geometry.id`. Once E03 lands and `jurisdiction_binding.geometry_id` references it, the wipe step requires a coordinated DELETE from `jurisdiction_binding` first (or `ON DELETE CASCADE`)." Italicize or call-out box this paragraph.
  - `## 6. Manifest-diff workflow`: Explain that `*-manifest-*.json` files (one per ArcGIS fetch) are the cross-operator drift signal. The check is `git diff` on the manifest files after a re-fetch. Re-fetch against unchanged source produces a manifest where every CONTENT field (`features_count`, `layer_hash`, `hash_distribution`, `source_url`, `source_layer_*`) matches the prior manifest exactly; only `fetched_at` advances to reflect the new fetch instant — that field always differs across runs and is not part of the drift signal. Any diff in the content fields indicates real upstream drift in feature counts, geometry shapes, or attribute values that the metadata fixture alone cannot catch. Note: backfill re-runs are stricter — `fetched_at` is parsed from the embedded filename timestamp during backfill (not sampled at runtime), so backfill against the same on-disk features files produces fully byte-identical manifests including `fetched_at`. **Hash determinism caveat:** A shapely version bump can change WKT output and therefore all per-feature hashes; the resulting "every manifest changed" delta is a tooling-version event, not a real source drift.
  - `## 7. Overlay architecture (informational)`: Document that overlay computation moved off SQL during S02.6 (Supabase's role-locked 2-min `statement_timeout` aborts the cross-join on real data). Implementation: local shapely + STRtree, ~5 sec end-to-end, see `ingestion/states/montana/build_overlay_fixture.py`. ## Conventions

- **Atomic deterministic JSON write:** `json.dumps(..., indent=2, sort_keys=True, ensure_ascii=False) + "\n"` then write to `path.with_suffix(path.suffix + ".tmp")` and `Path.replace(path)`. Established in S02.6's `_write_outputs`.
- **Filename timestamps:** `%Y%m%dT%H%M%SZ` (e.g., `20260501T141944Z`). Manifest's `fetched_at` ISO is derived from this, not regenerated.
- **Hash function:** `compute_feature_hash` (already in arcgis.py) for per-feature hashes; SHA-256 directly for `layer_hash` over sorted per-feature hashes joined with `\n`.
- **Hash distribution:** Always-present 256 keys (`"00".."ff"`), zero-included — required for byte-identical re-runs.
- **Pre-commit hooks** (`.pre-commit-config.yaml`) run on commit: ruff, mypy, detect-secrets. All new code passes these before commit.
- **Test conventions:** `monkeypatch` over libraries like `responses`; `tmp_fixture_dir` conftest fixture for filesystem; class-per-public-function organization in `test_arcgis.py`.
- **No `any` types** in `.tsx` files (CLAUDE.md). Not relevant here — all code is Python.
- **Read-only fail-loud** (per known-pitfalls.md): Backfill script must error explicitly when an expected features file is missing, not silently skip.
- **ADR cross-links:** Use relative paths `../adrs/ADR-XXX-*.md` from the runbook.
- **No paraphrasing of source authority** (ADR-001/008): N/A — this story doesn't write regulation text.
