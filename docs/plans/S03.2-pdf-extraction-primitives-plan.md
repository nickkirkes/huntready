### T3: `ConfidenceTier` enum + `min_tier` + `demote_one_tier` helpers (~5 min)
**Files:** `ingestion/ingestion/lib/pdf.py`
**Depends on:** T1
**Action:** Implement the ADR-017 confidence-calibration framework primitives.
**Details:**
- Define the enum with **rank-based** ordering (NOT lexicographic). Use a `str, Enum` mixin so `.value` round-trips with `schema.Confidence = Literal["high","medium","low"]`:
  ```python
  class ConfidenceTier(str, Enum):
      """Three-tier confidence per ADR-017.

      Rank ordering (most-uncertain-wins for MIN aggregation):
        LOW (rank=0) < MEDIUM (rank=1) < HIGH (rank=2)

      The string values match ``schema.Confidence`` so a tier instance can be
      written directly to ``regulation_record.confidence`` without conversion.
      """

      LOW = "low"
      MEDIUM = "medium"
      HIGH = "high"

      @property
      def rank(self) -> int:
          return _TIER_RANKS[self]

      def __lt__(self, other: object) -> bool:
          if not isinstance(other, ConfidenceTier):
              return NotImplemented
          return self.rank < other.rank


  _TIER_RANKS: dict[ConfidenceTier, int] = {
      ConfidenceTier.LOW: 0,
      ConfidenceTier.MEDIUM: 1,
      ConfidenceTier.HIGH: 2,
  }
  ```
  Notes:
  - `str, Enum` mixin: instances ARE strings (`ConfidenceTier.HIGH == "high"` is `True`), which is what the round-trip with `Confidence` literal requires.
  - With `class ConfidenceTier(str, Enum)` and a custom `__lt__`, the MRO is `[ConfidenceTier, str, Enum, object]` and our `__lt__` IS found first — so bare `min(tiers)` on enum instances would actually be rank-correct at runtime. We use an explicit `key=lambda t: t.rank` in `min_tier` anyway because it is unambiguous and does not require any reader to reason about MRO. The `__lt__` is implemented for general comparability of `ConfidenceTier` instances; `min_tier` does not depend on it.
- Define `min_tier`:
  ```python
  def min_tier(tiers: Iterable[ConfidenceTier]) -> ConfidenceTier:
      """Most-uncertain-wins MIN over confidence tiers (ADR-017 §"Aggregation").

      Implementation note: uses an explicit ``key=lambda t: t.rank`` rather than
      bare ``min()`` because Python's default min on string-mixin enums would
      compare lexicographically (``"high" < "low" < "medium"`` because h < l < m)
      — the wrong answer. See trap-case unit tests in test_pdf.py.

      Raises:
          PdfExtractionError: if ``tiers`` is empty (signals an upstream bug).
      """
      tiers_list = list(tiers)
      if not tiers_list:
          raise PdfExtractionError(
              "min_tier received an empty iterable — caller must supply at least one tier"
          )
      return min(tiers_list, key=lambda t: t.rank)
  ```
- Define `demote_one_tier` (per ADR-017 correction-touched rule):
  ```python
  def demote_one_tier(tier: ConfidenceTier) -> ConfidenceTier:
      """Demote one tier per ADR-017 §"Correction-touched demote rule".

      ``HIGH -> MEDIUM``, ``MEDIUM -> LOW``, ``LOW -> LOW`` (clamped at floor).
      Used when a record is touched by a correction PDF whose authority-class
      is below the parent's.
      """
      if tier is ConfidenceTier.HIGH:
          return ConfidenceTier.MEDIUM
      if tier is ConfidenceTier.MEDIUM:
          return ConfidenceTier.LOW
      return ConfidenceTier.LOW
  ```

### T4: `PdfDocument` context-manager wrapper + `open_pdf` + `iter_pages` (~5 min)
**Files:** `ingestion/ingestion/lib/pdf.py`
**Depends on:** T1
**Action:** Add the file-level helpers that wrap `pdfplumber.open()` while capturing filename/path traceability.
**Details:**
- Define `PdfDocument` as a frozen-ish wrapper. Use a regular class (not dataclass) because it needs `__enter__`/`__exit__` semantics and holds a live pdfplumber handle:
  ```python
  class PdfDocument:
      """Wraps a pdfplumber PDF handle with filename/path traceability.

      Use as a context manager:

          with open_pdf(path) as pdf:
              for page_num, page in iter_pages(pdf, 1, 5):
                  ...
      """

      def __init__(self, pdfplumber_pdf: pdfplumber.PDF, path: Path) -> None:
          self._pdf = pdfplumber_pdf
          self.path = path
          self.filename = path.name

      @property
      def pages(self) -> list[Page]:
          return self._pdf.pages

      def __enter__(self) -> PdfDocument:
          return self

      def __exit__(self, exc_type: object, exc_val: object, exc_tb: object) -> None:
          self._pdf.close()
  ```
  - `__exit__` returns `None` (do not suppress exceptions).
  - Type the `__exit__` params permissively as `object` to match Python's protocol while remaining mypy-clean without `Any`.
  - The `@property` for `pages` exposes the underlying list lazily; pdfplumber materializes pages on first access.

- Define `open_pdf`:
  ```python
  def open_pdf(path: Path) -> PdfDocument:
      """Open a PDF and return a `PdfDocument` wrapping the pdfplumber handle.

      Raises:
          PdfExtractionError: if the file does not exist or pdfplumber cannot
              parse it. Wraps any underlying exception so callers do not need
              to import pdfplumber's internal error types.
      """
      if not path.exists():
          raise PdfExtractionError(f"PDF not found at {path}")
      try:
          handle = pdfplumber.open(str(path))
      except Exception as exc:
          raise PdfExtractionError(f"failed to open PDF {path}: {exc}") from exc
      return PdfDocument(handle, path)
  ```
  - Wrap broad `Exception` here because pdfplumber raises a mix of `pdfminer` errors, `IOError`, etc. — callers should catch one error class.

- Define `iter_pages`:
  ```python
  def iter_pages(pdf: PdfDocument, start: int, end: int) -> Iterator[tuple[int, Page]]:
      """Iterate pages with 1-based page numbers (matches print convention).

      Args:
          pdf: an open `PdfDocument`
          start: first page number (1-based, inclusive)
          end: last page number (1-based, inclusive)

      Yields ``(page_num_1based, page)`` tuples.

      Raises:
          PdfExtractionError: if ``start < 1``, ``end < start``, or ``end`` exceeds
              the PDF's page count.
      """
      total = len(pdf.pages)
      if start < 1:
          raise PdfExtractionError(f"start page {start} must be >= 1 (1-based)")
      if end < start:
          raise PdfExtractionError(f"end page {end} must be >= start {start}")
      if end > total:
          raise PdfExtractionError(
              f"end page {end} exceeds PDF page count {total}"
          )
      for page_num in range(start, end + 1):
          yield (page_num, pdf.pages[page_num - 1])
  ```
  - These checks ARE at a system boundary (user-supplied page-range arguments), so per CLAUDE.md fail-loud guidance they belong here.

**Details:**
- Implementation:
  ```python
  def extract_text(
      page: Page, bbox: tuple[float, float, float, float] | None = None
  ) -> str:
      """Extract page text via pdfplumber's default word-grouping; optionally
      crop to a bounding box first.

      ADR-008 boundary: this primitive is a no-additional-normalization step
      on top of pdfplumber's default text-map. We do NOT call ``extract_text``
      with ``layout=True`` (which would inject synthetic spaces to preserve
      visual column alignment) and we do NOT do any post-processing
      (whitespace squash, Unicode normalize, line-break joining). Downstream
      extractors (S03.3–S03.5) are responsible for any cleanup needed for
      verbatim_rule storage.

      Platform note: pdfplumber's word-grouping reassembles glyphs into words
      with a single space between them, so a ``(hello  world)`` content stream
      with two literal spaces is returned as ``"hello world"`` (single space)
      — not byte-for-byte verbatim, but the closest representation pdfplumber
      offers. Newlines at glyph-line-breaks ARE preserved. Callers needing
      truly byte-exact source text would need to walk ``page.chars`` directly,
      which is not the V1 use case.
      """
      target = page.crop(bbox) if bbox is not None else page
      return target.extract_text() or ""
  ```
  - The `or ""` collapses pdfplumber's `None` return (rare — happens on pages with no text content) to a stable empty string. Callers always get `str`, never `None`.

### T6: `TableMatch` TypedDict + `extract_tables(page, settings=None)` (~5 min)
**Files:** `ingestion/ingestion/lib/pdf.py`
**Depends on:** T1
**Action:** Add the table-extraction primitive — uses pdfplumber's `find_tables()` (NOT `extract_tables()`) because only `find_tables` returns `Table` objects with `.bbox`.
**Details:**
- TypedDict:
  ```python
  class TableMatch(TypedDict):
      """A single table located on a page, with bbox and content.

      ``headers`` is the first row of the table (raw cell strings; may contain
      ``None`` cells for empty header positions). ``rows`` is the remaining rows.
      Both are returned verbatim — no normalization, no header inference.
      Callers are responsible for header interpretation.
      """
      bbox: tuple[float, float, float, float]
      headers: list[str | None]
      rows: list[list[str | None]]
  ```
- Function:
  ```python
  def extract_tables(
      page: Page,
      settings: dict[str, object] | None = None,
  ) -> list[TableMatch]:
      """Extract all tables on a page as typed `TableMatch` records.

      Wraps ``page.find_tables(table_settings)`` which (unlike pdfplumber's
      ``page.extract_tables()``) returns ``Table`` objects with ``.bbox``
      attached — required for `TableMatch.bbox`. The first row of each
      extracted table becomes ``headers``; remaining rows become ``rows``.
      An empty table (zero rows) yields ``headers=[]`` and ``rows=[]``.

      Args:
          page: the pdfplumber page to scan
          settings: optional pdfplumber ``TableSettings`` dict (see pdfplumber
              docs for keys like ``vertical_strategy``, ``horizontal_strategy``).
              ``None`` uses pdfplumber defaults.
      """
      matches: list[TableMatch] = []
      tables = page.find_tables(table_settings=settings) if settings else page.find_tables()
      for table in tables:
          bbox = cast(tuple[float, float, float, float], tuple(table.bbox))
          extracted = table.extract()
          if not extracted:
              matches.append(TableMatch(bbox=bbox, headers=[], rows=[]))
              continue
          headers = extracted[0]
          data_rows = extracted[1:]
          matches.append(TableMatch(bbox=bbox, headers=headers, rows=data_rows))
      return matches
  ```
  - Why `cast` instead of `# type: ignore`: `tuple(table.bbox)` is `tuple[float, ...]` from mypy's perspective (variadic), but `TableMatch.bbox` requires the fixed-length 4-tuple form. The runtime value is always exactly 4 floats per pdfplumber's `Table.bbox` contract. `cast` is the right tool here — `# type: ignore[typeddict-item]` is wrong because the actual mypy error code on a value-type mismatch is not `typeddict-item` (which is for missing/extra keys); using the wrong code would make the suppression silently no-op. The `cast` import is already added in T1.

### T7: `find_section(pdf, heading_pattern)` (~3 min)
**Files:** `ingestion/ingestion/lib/pdf.py`
**Depends on:** T4
**Action:** Add the heading-search helper using pdfplumber's `page.search()`.
**Details:**
- Implementation:
  ```python
  def find_section(
      pdf: PdfDocument, heading_pattern: str | re.Pattern[str]
  ) -> tuple[int, tuple[float, float, float, float]] | None:
      """Locate a heading by regex; return ``(page_num_1based, bbox)`` of the
      FIRST match, or ``None`` if not found.

      Iterates pages in order and calls ``page.search(pattern, regex=True)``.
      The bbox is the heading's character-level bounding box per pdfplumber
      semantics — usable downstream as a starting point for cropping the
      section content.
      """
      for page_num_1based, page in enumerate(pdf.pages, start=1):
          matches = page.search(heading_pattern, regex=True)
          if matches:
              first = matches[0]
              bbox = (
                  float(first["x0"]),
                  float(first["top"]),
                  float(first["x1"]),
                  float(first["bottom"]),
              )
              return (page_num_1based, bbox)
      return None
  ```
  - pdfplumber's `page.search` returns dicts with `x0`, `top`, `x1`, `bottom` (PDF coordinate convention: top is the y-of-top-edge measured from page top). Our bbox tuple uses the standard `(x0, y0, x1, y1)` ordering where `y0=top` and `y1=bottom`. This matches what `page.crop(bbox)` expects.
  - Returns `None` rather than raising — "not found" is a normal callsite branch, not an error.

  ```python
  def _blank_pdf_bytes() -> bytes:
      """A minimal valid 1-page PDF with no text content (via pypdf.PdfWriter)."""
      writer = pypdf.PdfWriter()
      writer.add_blank_page(width=612, height=792)
      buf = BytesIO()
      writer.write(buf)
      return buf.getvalue()


  def _text_bearing_pdf_bytes(text: str) -> bytes:
      """A minimal hand-crafted PDF with a single-page text content stream
      containing ``text``. Used to verify whitespace preservation in
      `extract_text` — pypdf has no convenient text-write API and adding
      `reportlab` as a test dep is overkill for one fixture.

      The text is embedded as a literal in a ``BT ... ET`` content stream;
      parens and backslashes in `text` are escaped per PDF spec.
      """
      escaped = text.replace("\\", "\\\\").replace("(", "\\(").replace(")", "\\)")
      # Newlines must split into separate ``Tj`` calls with ``Td`` line moves —
      # PDF text strings cannot contain literal newlines.
      lines = escaped.split("\n")
      content_ops = ["BT", "/F1 12 Tf", "50 750 Td"]
      for i, line in enumerate(lines):
          if i > 0:
              content_ops.append("0 -14 Td")
          content_ops.append(f"({line}) Tj")
      content_ops.append("ET")
      stream = "\n".join(content_ops).encode("latin-1")

      # Construct a minimal 4-object PDF (Catalog, Pages, Page, Content).
      objs: list[bytes] = []
      objs.append(b"1 0 obj\n<< /Type /Catalog /Pages 2 0 R >>\nendobj\n")
      objs.append(b"2 0 obj\n<< /Type /Pages /Kids [3 0 R] /Count 1 >>\nendobj\n")
      objs.append(
          b"3 0 obj\n<< /Type /Page /Parent 2 0 R /MediaBox [0 0 612 792] "
          b"/Resources << /Font << /F1 << /Type /Font /Subtype /Type1 "
          b"/BaseFont /Helvetica >> >> >> /Contents 4 0 R >>\nendobj\n"
      )
      objs.append(
          b"4 0 obj\n<< /Length "
          + str(len(stream)).encode("latin-1")
          + b" >>\nstream\n"
          + stream
          + b"\nendstream\nendobj\n"
      )

      out = BytesIO()
      out.write(b"%PDF-1.4\n")
      offsets: list[int] = []
      for obj in objs:
          offsets.append(out.tell())
          out.write(obj)
      xref_offset = out.tell()
      out.write(b"xref\n0 5\n")
      out.write(b"0000000000 65535 f\n")
      for off in offsets:
          out.write(f"{off:010d} 00000 n\n".encode("latin-1"))
      out.write(
          b"trailer\n<< /Size 5 /Root 1 0 R >>\nstartxref\n"
          + str(xref_offset).encode("latin-1")
          + b"\n%%EOF\n"
      )
      return out.getvalue()


  def _write_pdf(tmp_path: Path, name: str, data: bytes) -> Path:
      out = tmp_path / name
      out.write_bytes(data)
      return out
  ```
