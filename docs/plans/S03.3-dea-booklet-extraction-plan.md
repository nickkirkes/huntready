### T2: Cleanup utility helpers (~3 min)

**Files:** `ingestion/states/montana/extract_dea.py`

**Depends on:** T1

**Action:** Add two private helpers — `_normalize_cell` (cell-padding regex) and `_rejoin_hyphenated_linebreaks` (hyphenated line-break regex).

**Details:**
1. `def _normalize_cell(text: str | None) -> str | None`: returns `None` if `text` is `None` or whitespace-only after stripping; otherwise returns `re.sub(r"\s{3,}", " ", text.strip())`. The `\s` class includes spaces, tabs, and newlines per Python regex defaults — that is the intended behavior (collapses multi-line cell content). Document the empty-cell-becomes-None rule per ADR-001 fail-loud + spec line 318: "Empty cells: explicit `null`, not empty string."

Document inline that this is intentionally narrow: it does NOT rejoin `9/7-10/20` (slash before, digit after) nor `262-50` (digit before, digit after) — only lowercase-letter-bordered hyphens.
3. Both helpers are private (`_` prefix). They are unit-tested in T12.
4. NO additional cleanup beyond these two regexes is permitted at the structured-row level. Section-level `verbatim_text` is captured before any of this normalization runs.

### T4: HD section locator (~5 min)

**Files:** `ingestion/states/montana/extract_dea.py`

**Depends on:** T1, T2

**Action:** Add `_iter_hd_sections` that walks the deer/elk and antelope page ranges and yields each HD heading with its page number and section bbox.

**Details:**
1. Signature: `def _iter_hd_sections(pdf: PdfDocument, page_range: tuple[int, int], species_group: str) -> Iterator[tuple[str, str, str, int, tuple[float, float, float, float] | None]]`
   - Yields `(hd_number, hd_name, species_group, page_num_1based, section_bbox)`.
2. Implementation:
   - For each `(page_num, page)` from `iter_pages(pdf, *page_range)`:
     - Extract the page text via `extract_text(page)`.
     - Run `_HD_HEADING_REGEX.finditer(text)` to find all `HD <num> - <name>` matches on this page.
     - For each match, yield with `section_bbox=None` (we use the page-level bbox; per-section subcropping is handled in T5 if needed). The page-level bbox satisfies the AC's "every row carries a PageReference" — the HD is anchored to its starting page.
   - **Multi-page HDs:** A single HD section can span multiple pages (the regulation table flows across page breaks). The regex finds the heading on its starting page; the page reference points to the heading's page. T5's table extraction reads from that page forward until the next HD heading appears.
3. Species attribution:
   - Deer/elk pages 48–123 contain mixed deer + elk HDs — the same HD number appears for both species in separate subsections. The page-level structure has separate "DEER" and "ELK" species banners. **Handling**: rather than try to detect the species banner mid-page, pass `species_group` from the caller. The caller (T10's orchestrator) calls `_iter_hd_sections` twice for this range — once for deer with species sub-page-range derived from a banner-detection helper, once for elk — OR simpler: detect the species banner via `find_section(pdf, r"^(DEER|ELK)\s+REGULATIONS")` to compute deer-only and elk-only sub-ranges, then pass each.
   - **Practical strategy:** call `find_section` to locate the start of the `ELK REGULATIONS` (or `ELK HUNTING REGULATIONS`) banner inside the deer/elk range. Deer HDs run from 48 to (elk_banner_page - 1); elk HDs run from elk_banner_page to 123. If the banner regex doesn't match (PDF structure differs from research notes), raise `PdfExtractionError` with a clear message — fail-loud per ADR-001.
   - For T4's scope: `_iter_hd_sections` accepts a pre-resolved species page range and `species_group` string. The species-range resolution lives in T10's orchestrator.

### T7: Per-row season_windows extraction (~3 min)

**Files:** `ingestion/states/montana/extract_dea.py`

**Depends on:** T3, T6

> **Updated 2026-05-08 (UAT fix cycle).** The first iteration of this task built a section-level `_aggregate_section_season_windows` helper that picked the first non-empty cell per column and replicated that window across every row in the section. UAT against the live PDF surfaced this as a P0 ADR-008 violation: HD 124 deer has three distinct General Season windows (`Oct 24-Oct 30`, `Oct 24-Nov 29`, `Oct 31-Nov 29`) which the section-level aggregator collapsed to a single value. The corrected design is **per-row windows from each row's own column cells**; section-level aggregation has no place. See `docs/planning/epics/E03-confidence-findings/S03.3-uat-fixes.md` (Fix 1) and the "Section-level first-observation-wins is interpretive, not faithful" pitfall in `.roughly/known-pitfalls.md` for the full rationale.

**Action:** Compute `season_coverage` AND `season_windows` per row, both sourced from each row's own season-column cells. No section-level aggregator.

**Details:**
1. Inside `_rows_to_license_extractions` (T6), build a `season_col_map: dict[str, int]` once before the row loop (mapping normalized header → column index for each season column present in `header_row`).
2. For each data row, walk the season columns:
   ```python
   coverage = SeasonCoverage(early_season=False, archery_only=False, general=False, heritage_muzzleloader=False, late=False)
   row_windows: dict[str, SeasonWindow] = {}
   for season_col_name, col_idx in season_col_map.items():
       season_key = _SEASON_COLUMN_TO_KEY[season_col_name]
       try:
           raw_season_cell = row[col_idx]
       except IndexError:
           raw_season_cell = None
       if _is_season_cell_absent(raw_season_cell):
           coverage[season_key] = False
           continue
       coverage[season_key] = True
       cell_norm = _normalize_cell(raw_season_cell)
       if cell_norm is not None:
           row_windows[season_key] = SeasonWindow(
               window=cell_norm,
               weapon_type_override=_WEAPON_TYPE_OVERRIDE_BY_COLUMN[season_col_name],
           )
   ```
3. The pair `(coverage, row_windows)` is then attached to the row's `DeaRowExtraction` directly. `season_windows` only carries keys where `season_coverage[key] is True` (this falls out naturally from the loop above — no separate filter step).
4. **Divergence is the data, not an anomaly.** If two rows in the same section disagree on a season's date span, both values land verbatim in their respective rows. **Do NOT log a WARNING on divergence** — that would be noise. The HD-level interpretation (e.g., "use the earliest start date") is S03.6's job, not S03.3's.

**Verify:** `TestPerRowSeasonWindows` (5 tests) including `test_per_row_general_window_preserves_divergence_in_hd_124_deer`, which asserts each row's exact General window AND `len(set(...)) >= 3` so any regression to first-observation-wins fails loudly.

## Conventions

- **State-adapter file pattern:** match `fetch_pdfs.py` and `load_*.py` in this directory: `main(argv) -> int` + argparse, `_REPO_ROOT = Path(__file__).resolve().parents[3]`, module-level logger, fail-loud aggregating, invocation as `ingestion/.venv/bin/python ingestion/states/montana/extract_dea.py`.
- **PDF extraction primitives:** use the S03.2 shared library (`ingestion.lib.pdf`) only. Do NOT call `pdfplumber` directly — the wrappers preserve bbox info (`extract_tables` uses `find_tables()`) and emit the WARNING-on-empty-table signal that this story relies on.
- **Verbatim discipline (ADR-008):** section-level `verbatim_text` retains pdfplumber's word-grouped text without further normalization. Only the structured `rows` payload uses the cleanup regexes — and those regexes are documented in the module docstring.
- **Confidence assignment (ADR-017):** per-row `extraction_confidence` is assigned by S03.3; HD-level aggregation is S03.6's job (downstream MIN). Use `ConfidenceTier(str, Enum)` from `pdf.py` directly — no `.value` access at write sites.
- **Deterministic JSON output:** `json.dumps(data, indent=2, sort_keys=True, ensure_ascii=False) + "\n"`, atomic write via `.tmp` rename. Run twice + diff to verify (T13 step 3).
- **Test patterns:** synthesized PDF fixtures via `pypdf.PdfWriter`; tests in `ingestion/tests/`; fail loud on any column header that the mapping doesn't recognize (don't silently fall through).
- **Pitfalls already encoded** (`.roughly/known-pitfalls.md`):
  - `extract_text` collapses internal whitespace (not byte-exact); `page.chars` is the escape hatch we don't need.
  - `page.find_tables()` is the only way to get bboxes; `extract_tables` wrapper handles this.
  - State-adapter scripts must be invoked as `python <path>`, not `python -m`.
  - `expected_sha256` in `sources.yaml` is documentation; runtime drift uses the manifest.
- **Output JSON shape contract:** the TypedDicts defined in T1 are the contract S03.6 + S03.7 read. Changing them is a breaking change for those stories.
