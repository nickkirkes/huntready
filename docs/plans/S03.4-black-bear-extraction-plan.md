# Implementation Plan: S03.4 — Black Bear Booklet Extraction + Correction PDF Handling

**Branch:** `feat/S03.4-black-bear-extraction`

**Story spec:** `docs/planning/epics/E03-regulation-text-ingestion.md` lines 359–441.

**Discovery report (2026-05-10 session, ~12:26 AM EDT):** documented 12 spec discrepancies and structural surprises:

1. **BMU table is TRANSPOSED and PRINTED ROTATED.** pdfplumber returns 10 rows × N columns where each column = one BMU. All cell text is mirrored (line and char order reversed). `page.rotation == 0` — no PDF-level rotation hint.
2. **Table page range pp. 9–12** (not pp. 10–11 as spec claims) — 4 pages, not 2.
3. **8 quota-closure BMUs**, not 9. BMU 530 does NOT exist anywhere in the 2026 PDF. The 8 BMUs are: 411, 420, 440, 450, 510, 520, 600, 700.
4. **Correction PDF is prose-only** (no tables). It removes the entire "hound training season" column (Row 4 in transposed extraction) for all BMUs. Per-cell date-arbitration is honored via N synthesized correction operations (one per BMU) addressing `(bmu_number, "hound_training_season_dates")`.
5. **Regional map (p. 8) is image-only and unparseable** — but `hd_region` is directly in Row 0 of the BMU table as `"REGION N"` column markers. No `bmu_region_overrides.yaml` needed.
6. **Permit Managed sub-table on p. 11** (3 extra columns: BMUs 510-20, 520-20 with apply-by/permit-range/license-codes — different row semantics than main BMU columns). Detect via Row 0 `"Permit Managed Opportunities"` marker and skip from the main extraction (V1).
7. **Closure-prose page (p. 7) is two-column.** Use `page.crop((306, 0, 612, 792))` to isolate the right column.
8. **Quota cells embed subquota type** (e.g., `"= 4 Female subquota"` vs `"= 18 Harvest quota"`) — parse with regex, not `int()`.
9. **R1 reporting has TWO distinct obligations:** a 48-hour Mandatory Reporting (statewide-applicable) AND tooth submission (R1-only). Spec mentions both inline; emit as separate verbatim candidates.
10. **`hd_region` derives from Row 0 column markers** with left-to-right "current region" tracking, not from text-scan headings.
11. **Pre-existing schema:** `ClosurePredicate` and `ReportingObligation` Pydantic models exist at `ingestion/ingestion/lib/schema.py:97-117` and `:331-363`. S03.4 emits TypedDict candidates that S03.7/S03.9 will load and validate.
12. **Citation IDs locked in `sources.yaml`:** booklet = `mt-fwp-black-bear-2026-booklet` (publication_date `2026-04-27`); correction = `mt-fwp-black-bear-2026-correction-2026-03-18` (publication_date `2026-03-18`). Both have valid ISO dates; correction `pending` flag was removed 2026-05-07.

This plan bakes in those landmines so the implementation does not re-discover them.

2. **`_load_citation_from_sources_yaml(citation_id: str) -> SourceCitationDict`:**
   ```python
   def _load_citation_from_sources_yaml(citation_id: str) -> SourceCitationDict:
       """Load a single SourceCitation entry from sources.yaml by id.

       Fail-loud per ADR-001: if the id is missing, or publication_date is
       missing/unparseable as ISO-8601, raise PdfExtractionError. This is
       the gate referenced in spec § "Date-collision tiebreaker and
       missing-date handling": malformed dates never reach the merge stage.
       """
       with _SOURCES_YAML.open() as f:
           data = yaml.safe_load(f)
       entries = data.get("pdfs", [])
       for entry in entries:
           if entry.get("id") == citation_id:
               pub_date = entry.get("publication_date")
               if not pub_date or not isinstance(pub_date, str):
                   raise PdfExtractionError(
                       f"sources.yaml entry '{citation_id}' missing or non-string "
                       f"publication_date (got {pub_date!r})"
                   )
               # Validate ISO-8601 date.
               try:
                   import datetime
                   datetime.date.fromisoformat(pub_date)
               except ValueError as exc:
                   raise PdfExtractionError(
                       f"sources.yaml entry '{citation_id}' has unparseable "
                       f"publication_date {pub_date!r}: {exc}"
                   ) from exc
               return cast(SourceCitationDict, {
                   "id": entry["id"],
                   "agency": entry["agency"],
                   "title": entry["title"],
                   "url": entry["url"],
                   "publication_date": pub_date,
                   "document_type": entry["document_type"],
               })
       raise PdfExtractionError(
           f"sources.yaml has no entry with id={citation_id!r}"
       )
   ```

**Details:**
1. **`_reverse_cell_text(s: str | None) -> str | None`:**
   ```python
   def _reverse_cell_text(s: str | None) -> str | None:
       """Reverse line order then character order within each line.

       The Black Bear booklet prints the BMU regulation table physically rotated.
       pdfplumber returns each multi-line cell with both line and character order
       reversed (e.g., "Sep.15-Nov.29" extracts as "92 .voN-51 .peS"). This
       helper restores readable text BEFORE any normalization or regex matching.

       None and empty strings are passed through unchanged.
       """
       if s is None:
           return None
       if not s:
           return s
       lines = s.split("\n")
       lines.reverse()
       return "\n".join(line[::-1] for line in lines)
   ```
2. **`_normalize_cell(cell: str | None) -> str | None`:** identical to `extract_dea._normalize_cell` — `re.sub(r"\s{3,}", " ", cell.strip())`, return `None` for empty/whitespace-only/None. 3. **`_is_season_cell_absent(cell: str | None) -> bool`:** True if cell is `None` or `cell.strip() == "-"` (the dash sentinel). Identical semantics to `extract_dea._is_season_cell_absent`.
4. **`_rejoin_hyphenated_linebreaks(text: str) -> str`:** `re.sub(r"(?<=[a-z])-\n(?=[a-z])", "", text)`. Use ONLY for the closure-prose / reporting-prose `verbatim_rule` strings (the BMU-table cells are reversed first, so this regex would not apply meaningfully). Document this scope in the docstring.
