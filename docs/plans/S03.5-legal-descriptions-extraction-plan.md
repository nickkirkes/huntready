## Approach summary

The Legal Descriptions PDF (`mt-fwp-legal-descriptions-2026-2027-2026-02-03.pdf`, 56 pages, biennial citation id `mt-fwp-legal-descriptions-2026-2027`) is a three-column document. Each content page is split into three columns at `x â‰ˆ 36â€“195 / 210â€“355 / 390â€“555`. The TOC at page 2 enumerates nine species sections; only **three** of them have matching geometry rows in the V1 Montana DB:

| Section | PDF pages | V1 in scope? | Namespace |
|---|---|---|---|
| Region Legal Descriptions | 3â€“4 | No (no geometry rows) | â€” |
| Antelope HDs | 5â€“9 | **Yes** | `MT-HD-antelope-{N}-geom` |
| Bighorn Sheep HDs | 10â€“12 | No | â€” |
| Bison HDs | 13 | No | â€” |
| Black Bear MUs | 14â€“18 | **Yes** | `MT-HD-bear-{N}-geom` |
| Deer and Elk HDs | 19â€“35 | **Yes** | `MT-HD-deer-elk-lion-{N}-geom` |
| Moose HDs | 36â€“42 | No | â€” |
| Mountain Goat HDs | 43â€“45 | No | â€” |
| Mountain Lion HDs | 46â€“53 | No | â€” |
| Contacts | 54â€“56 | â€” (skip) | â€” |

**Scope decision (documented in module docstring + working note):** the extractor walks **V1 pages only** (5â€“9, 14â€“18, 19â€“35). Bighorn/Bison/Moose/Goat/Lion are explicitly out of V1 (no geometry rows to match against) and their headings are NOT surfaced as `unmatched`. They are simply not processed. This keeps the 10% fail-loud threshold meaningful: the rate is computed over V1 candidates only.

**Named-zone overlay pass:** two CWD zones (`Libby CWD Management Zone` p. 19, `Kalispell Area CWD Management Zone` p. 21) are matched against `MT-CWD-zone-{slug}-geom`. Headings appear on V1 pages already in scope. Slug mismatch on Kalispell ("Area" word dropped) â†’ hardcoded lookup.

**Out of scope for this story** (explicitly):
- 55 `portion` geometry rows (suffixes `elPt22` / `wtPt12` etc. are not derivable from PDF heading text). All 55 land in `unlinked`.
- 54 `restricted_area` geometry rows (no headed sections in this PDF â€” they appear only as prose modifiers inside HD descriptions). All 54 land in `unlinked`.
- `MT-STATEWIDE-geom` (kind=state, no boundary prose in this booklet). Lands in `unlinked`.
- "Portion of HD NNN" sub-headings inside the PDF (~50 occurrences). Captured but surfaced as `unmatched` (no automatic resolution to opaque `Pt{N}` suffixes).

**Matching algorithm:**

1. Load geometry IDs from `fixtures/geometry-overlays.json` â†’ build `_HD_LOOKUP: dict[(species_namespace, hd_number_int), geometry_id]` and `_CWD_LOOKUP: dict[str, geometry_id]`. Fail loud if either is empty.
2. For each V1 species section page range, set `species_namespace` and walk pages.
3. Per page: filter to upright chars (excludes rotated sidebar banners), crop each of three column bboxes, run `extract_text()`, concatenate columns leftâ†’right into one stream per page.
4. Split column stream into `(heading, body)` blocks via `_HD_HEADING_RE` (`^\s*(\d{2,3})\s+([A-Za-z][^:]{3,60}):\s+(?:Those portions?|That portion)`). Also recognize `_CWD_HEADING_RE` for named zones.
5. For multi-column / multi-page descriptions (most common case): the column stream of body text naturally continues across columns and into the next page until the next heading regex match. This is "first-match-then-accumulate" semantics â€” the body of heading H is everything between H and the next H'.
6. CWD zones: same accumulation rule. The Libby zone heading does NOT appear three times; verify during T3 whether the discovery's "three-occurrence" claim was a stream-reading artifact. If genuinely three occurrences, dedupe and assemble bodies in order.
7. Match each heading â†’ geometry_id by species namespace + HD number. CWD via lookup. Multiple plausible candidates (should be impossible inside a single namespace) â†’ `low` confidence. Unique â†’ `high`.
8. Build artifact envelope with `source_id`, `extracted_at`, `matched[]`, `unmatched[]`, `unlinked[]`.
9. Compute `unmatched_rate = len(unmatched) / (len(matched) + len(unmatched))`. If > 0.10 raise `PdfExtractionError` and `SystemExit(1)`. Below: WARN log only.

**Confidence per ADR-017:**
- Clean unambiguous match: `high`
- Fuzzy (would only happen if two candidates inside one namespace, which shouldn't occur but guard anyway): `low`
- Unmatched: no tier (the entry is in `unmatched`, not `matched`)
- `medium` is NOT used for heading-to-geometry matching (spec lines 491-494).

### T4: Heading regex + column-stream block splitter (~5 min)
**Files:** `ingestion/states/montana/extract_legal_descriptions.py`
**Depends on:** T1
**Action:** Define the regex constants and a splitter:
- `_HD_HEADING_RE = re.compile(r"^\s*(\d{2,3})\s+([^:]{3,80}?):\s+(?:Those portions?|That portion)\b", re.MULTILINE)` â€” captures `(hd_number_str, hd_name)`; the trailing keyword phrase (`Those portion(s)|That portion`) is part of the anchor but NOT captured (it's consumed; body starts AFTER it).
- `_CWD_HEADING_RE = re.compile(r"^\s*(Libby CWD Management Zone|Kalispell Area CWD Management Zone):\s+(?:That portion|Those portions?)\b", re.MULTILINE)` â€” captures `(zone_name,)`.
- `_PORTION_HEADING_RE = re.compile(r"^\s*Portion of HD\s+(\d{2,3})\b", re.MULTILINE)` â€” captured for `unmatched` surfacing; not a match target.
- Dataclass `HeadedBlock`: `kind: Literal["hd", "cwd", "portion", "continuation"]`, `heading_text: str`, `hd_number: int | None`, `cwd_name: str | None`, `body_raw: str`, `page_num_1based: int`. (`body_raw` will be filled by the page-walk in T5. The `"continuation"` kind is a sentinel emitted in T5 when a page's column stream begins with prose before any heading â€” T8's `_apply_continuation_merges` folds these into the prior block's body, so they never reach the matcher.)
- Function `_split_column_stream_into_blocks(text: str, page_num_1based: int) -> list[HeadedBlock]`:
  - Iterates over `_HD_HEADING_RE.finditer(text)` and `_CWD_HEADING_RE.finditer(text)` and `_PORTION_HEADING_RE.finditer(text)`. Combine into one sorted-by-`.start()` list.
  - For each match: heading is the match string; body is `text[match.end():next_match.start()]` (or `text[match.end():]` for the last one).
  - Returns the list of `HeadedBlock` records.
**Details:**
- The body for an HD heading starts immediately after the `Those portions of`/`That portion` keyword â€” that's where the prose "Beginning at..." starts. The regex is anchored on the keyword (not capturing it) so `match.end()` is at the keyword's end; the body extracted from `text[match.end():...]` will start with the rest of the boundary description.
- **Important nuance:** in a single column's text stream, the body of a heading on this page may continue into the *next* column or *next* page. T5 handles cross-column / cross-page accumulation. T4 only splits within a single concatenated stream.
- The regex `[^:]{3,80}?` is non-greedy on the name part to avoid swallowing punctuation; the 80-char cap guards against runaway matches in malformed text.
**Verify:** unit tests in T10 `TestHeadingRegex` cover: 2-digit HD, 3-digit HD, name with apostrophe ("O'Brien"), name with hyphen ("Sun-River"), name with parenthetical, name containing digits ("North Fork 116"), CWD zone heading, portion sub-heading, malformed line (no match).

### T8: Artifact builder + atomic deterministic JSON writer (~5 min)
**Files:** `ingestion/states/montana/extract_legal_descriptions.py`
**Depends on:** T2, T5, T6, T7
**Action:**
- `_apply_continuation_merges(blocks: list[HeadedBlock]) -> list[HeadedBlock]`: walk the block list; whenever a `kind="continuation"` block follows a non-continuation block, append its body to the prior block's body separated by `" "`, and drop the continuation entry. Continuation at the start of a section (no prior block) is dropped with a `_logger.debug` line (this would only happen if a section's first page begins mid-description, which shouldn't occur for V1 pages).
- `_build_artifact(merged_blocks, citation, hd_lookup, cwd_lookup, all_v1_ids) -> LegalDescriptionsArtifact`:

  - For each: collapse body whitespace via `re.sub(r"\s+", " ", body_raw).strip()` (Cleanup Rule A); call `_match_block_to_geometry` (T7).
  - On match: append to `matched[]` with `{"geometry_id", "geometry_kind", "verbatim_description": <collapsed body>, "page_reference": <T5 page_reference>, "extraction_confidence": <ConfidenceTier value>}`.
  - On unmatched: append to `unmatched[]` with `{"heading_text": <reconstructed e.g. "262 Description name">, "page_reference": <T5 page_reference>, "reason": <from T7>}`.
  - After both passes: compute `matched_ids = {entry["geometry_id"] for entry in matched}`; for each `(gid, kind)` in `all_v1_ids` that is NOT in matched_ids, append to `unlinked[]` as `{"geometry_id": gid, "geometry_kind": kind, "reason": "no_heading_in_pdf"}`.
  - Sort `matched` by `(geometry_kind, geometry_id)`, `unmatched` by `(page_num_1based, heading_text)`, `unlinked` by `(geometry_kind, geometry_id)` for deterministic output.
  - Build envelope: `{"source_id": _CITATION_ID, "extracted_at": <from manifest helper>, "matched": [...], "unmatched": [...], "unlinked": [...]}`.
- `_write_deterministic_json(path, payload)`: `json.dumps(payload, indent=2, sort_keys=True, ensure_ascii=False) + "\n"` written to `path.with_suffix(".json.tmp")` then `replace()` to `path` (mirror extract_dea.py's pattern lines 1644-1648).

## Conventions

- **State-agnostic boundary** (ADR-005): adapter scripts live in `ingestion/states/montana/`. Shared code in `ingestion/ingestion/lib/`. AST guard test mandatory.
- **Verbatim text discipline** (ADR-008): `verbatim_description` is the prose body for the heading, preserving all source words. The whitespace-collapse rule (Rule A) is documented and applied consistently; no further normalization.
- **Confidence calibration** (ADR-017): only `high` is emitted for matched S03.5 entries in V1 scope (per spec Â§491-494). `medium` not used. `low` reserved for the unreachable multi-candidate branch.
- **Schema additions** (ADR-018): no schema work in this story (S03.0 already added `geometry.legal_description` column). S03.5 produces JSON; S03.6 writes to the DB column.
- **Documentation discipline** (ADR-009, S03.3 UAT lesson): every text-normalization rule applied to extracted output must appear in the module docstring with regex, scope, rationale, and a pointer to the locking test. Tested as a discipline (no automated checker), but reviewers flag if missing.
- **Fail-loud invariant** (CLAUDE.md "Architectural commitments"): no silent failures. Empty input â†’ raise. Above-threshold unmatched rate â†’ raise + non-zero exit. Below-threshold unmatched â†’ WARN log + working-note flag.
- **Single-writer contract** (S03.5 spec Â§487): NO database writes. S03.5 produces only the JSON artifact. Lock this via the AST guard test (no DB imports â€” no `from ingestion.lib.db import ...`).
- **Determinism** (CLAUDE.md): atomic write, sorted keys, sorted arrays. Re-running with identical PDF + manifest produces byte-identical artifact.
- **Working artifact lifecycle** (ADR-017 Â§6): `docs/planning/epics/E03-confidence-findings/S03.5.md` deletes at the m1 tag.
