### T2: Add `_extract_statewide_rules` function (~4 min)
**Files:** `ingestion/states/montana/extract_black_bear.py`
**Action:** Add a new extraction function that reads page-2 right-column verbatim text and returns `list[StatewideRuleCandidate]`. Insert immediately after the existing `_extract_reporting_obligations` function (the related closure-prose extractor) so the page-N extractors are grouped.
**Details:**
- Add module-level constant `_STATEWIDE_RULES_PAGE = 2` near the existing `_CLOSURE_PROSE_PAGE = 7` constant (line 151).
- Reuse the existing `_RIGHT_COLUMN_BBOX` constant (line 154) — page 2's "Obtain a License" content is in the right column on the same letter-page geometry.
- Function signature: `def _extract_statewide_rules(pdf: PdfDocument, pdf_filename: str, source_id: str, source_publication_date: str, extracted_at: str) -> list[StatewideRuleCandidate]:` — matches the parameter convention of the sibling `_extract_reporting_obligations` (and the rest of the page-N extractors in this module). **Do NOT derive `pdf_filename` synthetically from `source_id + source_publication_date`** — that would silently override the real filename in custom `--booklet-pdf` operator runs. The orchestrator (`_extract_base`) is responsible for threading `booklet_pdf.filename` through. Similarly, `extracted_at` MUST be threaded explicitly from the orchestrator (manifest timestamp), NOT substituted with `source_publication_date` as a fallback.
- Use two regex anchors to locate the Bear ID Test paragraph:
  - `_BEAR_ID_START_RE = re.compile(r"A hunter may purchase only one Black Bear License per year\.")`
  - `_BEAR_ID_END_RE = re.compile(r"fwp\.mt\.gov/hunt/education/bear-identification")`
- Algorithm:
  1. Open page 2 via `iter_pages(pdf, 2, 2)`.
  2. Extract right-column text via `extract_text(page, bbox=_RIGHT_COLUMN_BBOX)`.
  3. Locate the start anchor; if absent → return `[]` (fail-loud is operator's responsibility via the row-count guard; an empty list will trip the `[1, 1]` band).
  4. Locate the end anchor AFTER the start anchor; if absent → raise `RuntimeError(f"_extract_statewide_rules: end anchor missing on p{_STATEWIDE_RULES_PAGE}. Source PDF may have changed; investigate before re-running.")`.
  5. Take the substring from start of match to end of match (inclusive of both anchors). Collapse internal `\s+` to single space (S03.3 extras-only whitespace convention; document in module docstring cleanup-rules section).
  6. Construct one `StatewideRuleCandidate` with `rule_hint="pre_purchase_prerequisite"`, the cleaned verbatim text, page_reference for page 2, source_id, source_publication_date, `extraction_confidence="medium"` (heading-anchored prose per ADR-017).
  7. Return `[candidate]`.
- Update the module-level "Cleanup rules" docstring section (S03.3 discipline reminder — locked by `TestStatewideOverlayColumnFaithful`-style tests): document the new `\s+ → " "` collapse applied to the statewide rule body, the exact regex, the scope (page-2 right-column paragraph between the two anchors only), and a pointer to the tests that lock it.
- Module-level note: AC #936 requires recording in the working note that the antelope binding to `MT-STATEWIDE-geom` does NOT yet exist. No code change needed for this — recorded in T13.
**Verify:** `cd ingestion && .venv/bin/ruff check ingestion/ tests/ && .venv/bin/mypy ingestion/lib/ states/montana/extract_black_bear.py`
