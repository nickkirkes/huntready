## Architectural decisions baked into this plan

These resolve discovery-stage open questions. Surface in review if any is contested:

1. **OQ-S7-1 / OQ-S7-2 (species granularity of license_tag and season_definition):** Use artifact-level species label (`"deer" | "elk" | "antelope" | "bear"`) in the `id` and `species` field. For deer, the species fan-out happens only at the link-table layer — both `mule_deer` and `whitetail` regulation_records reference the same `license_tag.id` via two `regulation_license` link rows, and the same `season_definition.id` via two `regulation_season` link rows. This avoids duplicating the same physical license/season into two near-identical DB rows.
2. **OQ-S7-3 (UAT HD substitution):** Spec line 637 says `HD 262 elk` but HD 262 only exists for deer in the artifact. **Substitute HD 170 (Flathead River) elk** — confirmed asymmetric: General Elk License covers `archery_only/general/heritage_muzzleloader`; Elk B License 170-00 covers `archery_only/general/late`. T12 includes the epic + closure-note doc edits.
3. **OQ-S7-4 (season name rendering):** Module-level constant `_SEASON_NAME_BY_KEY` maps artifact keys to title-case FWP column-header names: `{"archery_only": "Archery Only", "general": "General", "heritage_muzzleloader": "Heritage Muzzleloader", "late": "Late", "early_season": "Early Season"}`.
4. **OQ-S7-5/-6 (bear closure_predicate attachment):** `sex_threshold` (spring closure) attaches to `spring_season` only. `quota_threshold` (fall closure) attaches to both `general_season` and `archery_only_season`. The match is by BMU number against the `closures[]` arrays.
5. **OQ-S7-7 (license_tag.kind heuristic):** Evaluated in order, first match wins. (a) `section["hd_number"] == "STATEWIDE"` → `'statewide'`; (b) `license_code.startswith("General ")` → `'general'`; (c) `"B License"` substring in license_code → `'limited_draw'`; (d) `"Permit:"` substring (covers `"Deer Permit: …"`, `"Elk Permit: …"`) → `'limited_draw'`; (e) `license_code.startswith("Antelope License:")` with non-STATEWIDE hd_number → `'limited_draw'` (per-HD antelope A-licenses like `"Antelope License: 471-20"`); (f) bear rows (T9) → `'general'`. Anything else → fail loud with the unknown code (V1 closed-set discipline; surfaces drift loudly). The full DEA artifact has been profiled against this heuristic with no unmatched rows.
6. **OQ-S7-8 (window parsing):** Module-level helper `_parse_window(window_str, license_year)` handles three formats: dotless (`"Sep 05-Oct 18"`), dotted (`"Aug. 15-Nov. 08"`), and bear-style with embedded newlines (`"Sep.\n15-Nov.\n29"`). Year-wrap rule: if `closes_month < opens_month`, `closes_year = license_year + 1`.
7. **OQ-S7-9 (bear quota_range):** Bear has only `fall_quota.count` / `spring_quota.count` (single integers, no range). Set `license_tag.quota_range = None` for bear (the BMU's `fall_quota.count` populates `license_tag.quota`). The `quota_range` field is reserved for ranges from the DEA's `quota_range` cell (e.g., 900-20's `"1-7,500"`).
8. **OQ-S7-10 (statewide 900-20 quota_range parsing):** Module-level helper `_parse_quota_range("1-7,500")` strips commas and splits on `-` → `(1, 7500)`. Same helper handles None.
9. **verbatim_rule cell-precision (refines spec lines 567-568):** Spec wants cell-precision. For V1 we use a HD-section-scoped fallback for `season_definition.verbatim_rule`: prefer the General/A license row's `extras` cell when populated; if all candidates are null, use `section["verbatim_text"]` (broader scope, still source-faithful, no paraphrase). For `license_tag.verbatim_rule` use `section["verbatim_text"]` for the row's enclosing section (full-row reconstruction is interpretive; section-scope is faithful and Pydantic-validator-safe). This refines spec lines 567-568 toward what's faithfully extractable in V1; M2 may revisit if a downstream consumer needs per-cell precision.
10. **license_tag.residency default:** DEA artifact carries no residency-by-license signal. Default `residency='both'` for all DEA rows (matches FWP's de facto behavior — Montana A/B licenses are sold to residents and non-residents alike, with `weapon_types` and `quota` governing eligibility). Same for bear license_tags.
11. **license_tag.purchase_url default:** Use `"https://fwp.mt.gov/hunt/licensing/buy"` (Montana FWP's evergreen license-purchase entry point). Single constant; survives downstream URL rotations because it points at the routing page, not a deep link.

### T4: load_seasons_and_licenses.py — window parser + quota_range parser (~4 min)

**Files:** `ingestion/states/montana/load_seasons_and_licenses.py`
**Depends on:** T3
**Action:** Add the date-window parser and the quota_range parser as private module helpers, plus their licence-code-to-slug helper.
**Details:**
- `_parse_window(window_str: str, license_year: int) -> tuple[date, date]` — handles three formats: dotless `"Sep 05-Oct 18"`, dotted `"Aug. 15-Nov. 08"`, bear-newline `"Sep.\n15-Nov.\n29"`. Normalize by `re.sub(r"\s+", " ", window_str.replace(".", ""))` then split on `-`. Use `datetime.datetime.strptime("Sep 05 2026", "%b %d %Y").date()` for parsing each side. Year-wrap rule: if `closes.replace(year=opens.year) < opens`, set `closes_year = opens.year + 1`. Returns `(opens_date, closes_date)`.
- `_parse_quota_range(raw: str | None) -> tuple[int, int] | None` — returns None for None or whitespace; else `int(part.replace(',', '').strip())` for each side of `-`. Raises ValueError on un-parseable input (fail-loud).
- `_license_code_slug(license_code: str) -> str` — extracts the numeric component from license codes: `"Elk B License: 262-50"` → `"262-50"`, `"Antelope License: 900-20"` → `"900-20"`, `"General Elk License"` → `"general"`, `"General Antelope License"` → `"general"`. Regex: `r"\b\d+(-\d+)?\b"` for numeric forms; else lowercase first-word.

## Conventions

- **ADR-008 (verbatim discipline):** All `verbatim_rule` strings are source-faithful, no paraphrase. When cell-level precision isn't available (most license_tag rows), default to section-scoped `verbatim_text` rather than constructing a row summary. Surface in plan-review.
- **ADR-010 (decomposed entities):** S03.7 is the canonical story that operationalizes the 4-layer decomposition (regulation_record → season_definition / license_tag via link tables). All future state adapters must mirror this entity ↔ link-table separation.
- **ADR-012 (soft FK):** `license_tag.draw_spec_key = None` for every row written here; S03.8 backfills. Validated by a single AC-item.
- **ADR-017 (confidence inheritance):** Child entities (season_definition, license_tag) carry their own `verbatim_rule.source.confidence` *via* the embedded SourceCitation's document_type and ADR-017's parent-inheritance interpretation, but the DB schema has NO `confidence` column on these tables. Don't try to add one.
- **ADR-018 (license_season + statewide geometry):** Same migration that added `license_season` also added `geometry.legal_description` and `geometry.kind='state'` — already shipped in S03.0.
- **Module duplication:** Per the in-CLAUDE.md note about state-adapter independence, `_load_citation_from_sources_yaml`, `_dea_jurisdiction_code`, and `_DEA_SPECIES_FANOUT` are duplicated from `load_regulation_records.py` rather than cross-imported. This keeps each state-adapter file self-contained and revision-safe.
- **Per-row source citations:** Bear path constructs per-row citation (matches S03.6 lines 322-356). DEA path uses section-scoped citation with row-level page_reference override (matches S03.6's `_build_dea_records` pattern).
- **Pydantic marshalling:** All `model_dump(exclude_none=True)` for jsonb columns; `Json(...)` wrapping in psycopg3.
- **No commit in db.py helpers; main() owns the single atomic commit.** Mirrors `load_regulation_records.py` exactly.
