## What this story does

Writes `draw_spec` rows for the 388 (post-dedup, post-OTC-corrected) limited_draw DEA license_tag identities from S03.7, then backfills `license_tag.draw_spec_key` for those rows. Atomic single-transaction, idempotent.

## Locked design decisions (PM-confirmed; do not relitigate)

- **OTC discrimination via apply_by cross-row AND cross-HD OTC-wins**: any `license_code` string with at least one artifact row (in ANY HD section) whose `apply_by` contains the substring `"OTC"` classifies as `kind='over_the_counter'`. The OTC set is keyed by `license_code` alone (not by `(species, hd, license_code)` 3-tuple) ‚Äî same physical license appearing in different HD sections shares the same kind. All other "B License" rows ‚Üí `kind='limited_draw'`. Permit/Antelope License rows ‚Üí `kind='limited_draw'` unchanged. STATEWIDE/General branches unchanged.
- **Hardcoded V1 draw defaults** (no data source in any ingested artifact; documented in T11's working note):
  - `choices = ChoiceConfig(count=1, points_used_in_choices=[1])` ‚Äî MT has no preference-point system
  - `pools = [AllocationPool(share=1.0, selection="unweighted_random")]` ‚Äî single pool; NR-cap structure (MCA Title 87-2-106's 10% rule) elided to V1; logged in `draw-mechanics.md` (T10)
  - `draw_phase = "primary"` ‚Äî Surplus Drawing leftover phase not extracted from S03.3
  - `parameters = None` ‚Äî Q12 deferral; any future case requiring it triggers flag-and-defer
- **`hunt_code`**: `license_code` verbatim (e.g., `"Deer B License: 262-50"`). NOT just the numeric stub. This matches the `license_tag.license_code` column exactly ‚Äî the soft-FK lookup is by full string.
- **`application_deadline`**: parsed from per-row `apply_by` (post-dedup, OTC-wins). All 390 limited_draw identities have at least one non-null non-OTC `apply_by` across their duplicate rows (verified ‚Äî zero hits expected on front-matter fallback).
- **Front-matter lookup** as defensive safety net only. Built from DEA pp. 5/9/10/11 chart values. If invoked at runtime, WARN log + flag-and-defer entry (drift signal ‚Äî implies S03.3 extraction regression).
- **`900-20` skip**: falls out cleanly via S03.7's STATEWIDE branch (`kind='statewide'`, not `limited_draw`).
- **Bear rows excluded**: closure mechanics, not draws (spec line 740).
- **Source-of-truth**: rebuild from `dea-2026.json` artifact (S03.6/S03.7 precedent).
- **Row-count guard band**: 388 ¬± 30% = [271, 504] (int-truncation matching S03.7's pattern). Guard fires BEFORE `db.connect()`.
- **Re-upsert scope inside S03.8 transaction**: only DEA license_tag rows from `_build_dea_license_tags`. Bear license_tags untouched (no classification change there).
- **DB helpers**: 2 new helpers added to `db.py` (T2 + T3); both no-commit (caller-controlled txn); `update_license_tag_draw_spec_key` fails loud on `cur.rowcount == 0` mirroring [`db.update_legal_description`](ingestion/ingestion/lib/db.py#L249-L283).

## Plan-realignment notes (post-implementation)

Per the project's plan-realignment discipline (cf. S03.3 Phase 6 + S03.6 closure), this plan was updated AFTER implementation to reflect the as-built code. Key divergences from the original Stage 3 plan:

1. **OTC keying broadened** (commit d68b548): `_otc_identities` 3-tuple keyed by `(species, hd, license_code)` ‚Üí `_otc_license_codes` string keyed by `license_code` alone. Baseline correction: 390/160 ‚Üí 388/162 limited_draw/over_the_counter.

2. **`_row_has_otc` helper** (commit 73c4500): the pre-pass apply_by check is now a separate helper with explicit `isinstance(str)` guard and `"apply_by" not in row` fail-loud, replacing the original silent `(row.get("apply_by") or "")` pattern that would silently return False for missing keys or non-string values.

3. **`parameters` JSON-safe serialization** (commit 73c4500): `upsert_draw_spec` uses `_PARAMETERS_TYPE_ADAPTER.dump_python(spec.parameters, mode="json")` to convert date/UUID/Decimal to JSON-safe Python types before wrapping in `Json`. The original Stage 3 plan specified raw `Json(spec.parameters)` which would raise `TypeError` at `cursor.execute` time for ADR-012 escape-hatch parameters carrying non-stdlib-JSON values.

4. **`typing.Literal` import** (commit 73c4500): T4 scaffold required `from typing import Literal` for the `_DEFAULT_DRAW_PHASE: Literal["primary"]` annotation; omitted from the original T4 imports list.

5. **Cross-listing override field-by-field checks** (commit d68b548): if a structural field disagrees and the override doesn't cover that specific key, raise RuntimeError. The original Stage 6 add specified `.get(...) is None` for absence checks, which cannot distinguish "key absent" from "key present with explicit None value".

6. **Override key-presence vs value-None** (this cubic-P2 fix): override entries can now specify `{"key": None}` to force a canonical SQL-NULL value; the validator uses `"key" in override` to distinguish from "key absent" (which fails loud). The original implementation collapsed these two cases via `.get(...)`.

7. **Cross-listing override symmetry** (commit cc7f2bb): the override now rewrites BOTH `LicenseTag.quota` AND `DrawSpec.quota` (plus propagates back to Phase 1's license_tag list in `main()` via Step 4b) so both tables agree on the canonical value. The original plan described rewriting only the draw_spec side.
