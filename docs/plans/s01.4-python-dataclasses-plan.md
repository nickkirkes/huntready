# Implementation Plan: S01.4 — Python Pydantic Models Matching DDL

## Context

S01.4 is the fourth story in the E01 (Schema Migrations) epic. S01.2 created the DDL (10 tables) and S01.3 added RLS. This story creates the Python-side mirror: Pydantic v2 models in `ingestion/ingestion/lib/schema.py` that match the DDL column-for-column, giving the ingestion pipeline type-safe validation before writing to Postgres.

The canonical type definitions live in `docs/architecture.md` § "Schema types". The DDL in `supabase/migrations/20260425000000_initial_schema.sql` is the physical truth. Python models are DB-facing (flat FK columns, no `_ids` arrays — those are query-time JOINs in the MCP server).

### T2: Create jsonb sub-models in `schema.py` (~5 min)

**Files:** `ingestion/ingestion/lib/schema.py`
**Action:** Create the file with all 11 jsonb sub-models that represent embedded Postgres jsonb structures.
**Details:**

File header:
```python
"""Pydantic v2 models mirroring the HuntReady DDL one-to-one.

These models are the Python leg of the three-place schema sync
(DDL → Python → TypeScript). Field names match DDL column names exactly.
Canonical type definitions: docs/architecture.md § "Schema types".

See ADR-005, ADR-006, ADR-008, ADR-010, ADR-012.
"""

from __future__ import annotations

import datetime
from typing import Annotated, Any, Literal

from pydantic import BaseModel, ConfigDict, Field, field_validator

import shapely
from shapely import from_wkt, make_valid
```

Define reusable type aliases before models:
```python
WeaponType = Literal[
    "any_legal_weapon", "archery", "rifle", "muzzleloader",
    "shotgun", "handgun", "crossbow", "traditional_handgun",
    "heritage_muzzleloader",
]

Residency = Literal["resident", "nonresident", "both"]
Confidence = Literal["high", "medium", "low"]
```

Create these 11 models in dependency order:

1. **`SourceCitation`** — `id: str`, `agency: str`, `title: str`, `url: str`, `publication_date: str`, `document_type: Literal["annual_regulations", "rule_change", "emergency_order", "correction"]`, `supersedes: str | None = None`, `page_reference: str | None = None`. Config: `model_config = ConfigDict(frozen=True, extra="forbid")`

2. **`VerbatimRule`** — `text: str` (validated non-empty via `field_validator`), `page_reference: str | None = None`, `confidence: Literal["high", "medium", "low"]`, `source: SourceCitation`. Config: frozen, extra=forbid.

3. **`ClosurePredicate`** — `kind: Literal["quota_threshold", "sex_threshold", "emergency_order"]`, `threshold_percent: float | None = None`, `threshold_sex: Literal["male", "female"] | None = None`, `notification_channel: Literal["agency_website", "agency_phone", "email_alert", "other"]`, `observation_channel: Literal["mandatory_reporting", "check_station", "harvest_survey"] | None = None`, `verbatim_rule: str` (validated non-empty). Config: frozen, extra=forbid.

4. **`ReservedPoolEligibility`** — `kind: Literal["landowner", "youth", "hunter_education_recent", "other"]`, `min_acres: int | None = None`, `min_acres_contiguous: bool | None = None`, `min_age: int | None = None`, `max_age: int | None = None`, `notes: str | None = None`. Config: frozen, extra=forbid. **Use `model_config = ConfigDict(frozen=True, extra="forbid")` — exclude_none is handled at serialization call site, not globally in config** (Pydantic v2 does not have a `ConfigDict(exclude_none=True)` — you pass `exclude_none=True` to `model_dump()`).

5. **`ReservedPool`** — `share: float`, `eligibility: ReservedPoolEligibility`, `applies_to_residency: Literal["resident", "nonresident", "both"] | None = None`, `nonresident_subcap: float | None = None`, `verbatim_rule: str` (validated non-empty). Config: frozen, extra=forbid.

6. **`PointSystem`** — `kind: Literal["preference_linear", "bonus_squared", "bonus_weighted"]`, `accrual: Literal["annual_on_apply", "annual_if_purchased"]`, `reset_on_success: bool`, `purchase_only_code: str | None = None`, `inactive_forfeit_years: int | None = None`. Config: frozen, extra=forbid.

7. **`ResidencyCap`** — `nonresident_max_share: float`. Config: frozen, extra=forbid.

8. **`ChoiceConfig`** — `count: int`, `points_used_in_choices: list[int]`. Config: frozen, extra=forbid.

9. **`AllocationPoolEligibility`** — `min_points: int | None = None`, `residency: Literal["resident", "nonresident", "both"] | None = None`, `guided: bool | None = None`. Config: frozen, extra=forbid.

10. **`AllocationPool`** — `share: float`, `selection: Literal["rank_ordered_by_points", "unweighted_random", "squared_weighted_random", "linear_weighted_random"]`, `eligibility: AllocationPoolEligibility = Field(default_factory=AllocationPoolEligibility)` — **always present** per canonical TS type (all sub-fields optional but the object itself is required), `tie_break: Literal["random", "rank_ordered"] | None = None`. Config: frozen, extra=forbid.

11. **`DrawSpecKey`** — `state: str`, `hunt_code: str`, `year: int`. Config: frozen, extra=forbid. (Used for soft FK references in `license_tag.draw_spec_key` and `draw_spec.successor_hunt_code_key`.)

**Validation rules for this task:**
- All `verbatim_rule: str` fields must have a `@field_validator` ensuring non-empty (`if not v.strip(): raise ValueError(...)`)
- `VerbatimRule.text` must also be validated non-empty
- All models use `frozen=True` (immutability) and `extra="forbid"` (reject unknown fields)

## Blast Radius

- **Do NOT modify:** `supabase/migrations/`, `docs/architecture.md`, `mcp-server/`, `web/`, `plugin/`
- **Watch for:** `from __future__ import annotations` turns all annotations into strings at runtime. Pydantic v2 handles this correctly via deferred evaluation, but if a `@field_validator` ever inspects `__annotations__` directly (don't do this), it will see string forms. Verify with import test.
- **Loader note:** `LicenseTag.quota_range` is stored as `tuple[int, int] | None` in the model. The insert loader (future story) must convert this to `psycopg.types.range.Int4Range(lower, upper)` before passing to psycopg3 — raw tuples do not map to Postgres range types automatically.

## Conventions

- ADR-005: Python for ingestion, TypeScript for serving — no cross-imports
- ADR-006: `schema_version` default=2 on RegulationRecord and DrawSpec
- ADR-008: All `verbatim_rule` fields NOT NULL and non-empty (except JurisdictionBinding where it's nullable per DDL)
- ADR-010: Decomposed entity model — 6 entities + link tables
- ADR-012: `DrawSpec.parameters` escape hatch — shared code must never read it
- Pydantic v2 API only (`ConfigDict`, `field_validator`, `Field`) — no v1 patterns
- Field names match DDL column names exactly (all snake_case)
- `frozen=True` on all models (immutable after construction)
- `extra="forbid"` on all models (reject unknown fields)
